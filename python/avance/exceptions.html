Introduction

Pourquoi gérer les exceptions

Dans tout programme réel, des erreurs surviennent : un fichier introuvable, une division par zéro, une connexion réseau interrompue. Python utilise un mécanisme d'exceptions pour signaler et traiter ces situations anormales de manière élégante, sans faire planter le programme.

Plutôt que de vérifier chaque condition possible avec des if, Python encourage le style EAFP (Easier to Ask Forgiveness than Permission) : on essaie d'exécuter le code, et on gère l'erreur si elle survient. Cette approche produit un code plus lisible et plus pythonique.

Principe EAFP : en Python, il est préférable de tenter une opération et de gérer l'exception plutôt que de tester toutes les préconditions. C'est le contraire du style LBYL (Look Before You Leap) courant dans d'autres langages.

Le cours

try, except, finally, raise et exceptions personnalisées

Le bloc try / except

Le mécanisme de base consiste à placer le code susceptible de provoquer une erreur dans un bloc try, puis à intercepter l'exception dans un bloc except.

Python
# Exemple basique : division par zéro
try:
    resultat = 10 / 0
except ZeroDivisionError:
    print("Erreur : division par zéro impossible")

# Capturer le message d'erreur
try:
    nombre = int("abc")
except ValueError as e:
    print(f"Conversion impossible : {e}")
    # Conversion impossible : invalid literal for int() with base 10: 'abc'

# Capturer plusieurs types d'exceptions
try:
    valeur = [1, 2, 3][10]
except (IndexError, KeyError) as e:
    print(f"Erreur d'accès : {e}")

Blocs except multiples et hiérarchie

Python parcourt les blocs except de haut en bas et exécute le premier qui correspond. L'ordre est important : les exceptions les plus spécifiques doivent venir en premier.

Python
def diviser(a, b):
    try:
        resultat = a / b
    except ZeroDivisionError:
        print("Division par zéro !")
    except TypeError:
        print("Types incompatibles pour la division")
    except Exception as e:
        print(f"Erreur inattendue : {e}")
    else:
        print(f"Résultat : {resultat}")

diviser(10, 3)     # Résultat : 3.333...
diviser(10, 0)     # Division par zéro !
diviser(10, "a")   # Types incompatibles pour la division

# Hiérarchie des exceptions (extrait) :
# BaseException
#   ├── KeyboardInterrupt
#   ├── SystemExit
#   └── Exception
#       ├── ValueError
#       ├── TypeError
#       ├── KeyError
#       ├── IndexError
#       ├── FileNotFoundError (sous-classe d'OSError)
#       └── ...

Le bloc else et finally

Le bloc else s'exécute uniquement si aucune exception n'a été levée. Le bloc finally s'exécute dans tous les cas, qu'il y ait eu une exception ou non. Il est idéal pour libérer des ressources.

Python
def lire_fichier(chemin):
    fichier = None
    try:
        fichier = open(chemin, "r")
        contenu = fichier.read()
    except FileNotFoundError:
        print(f"Fichier '{chemin}' introuvable")
    except PermissionError:
        print(f"Permission refusée pour '{chemin}'")
    else:
        # Exécuté seulement si aucune exception
        print(f"Fichier lu avec succès ({len(contenu)} caractères)")
        return contenu
    finally:
        # Toujours exécuté — nettoyage garanti
        if fichier is not None:
            fichier.close()
            print("Fichier fermé")

# En pratique, on préfère le context manager (with)
try:
    with open("data.txt") as f:
        contenu = f.read()
except FileNotFoundError:
    print("Fichier introuvable")

Lever des exceptions avec raise

Le mot-clé raise permet de lever une exception explicitement. C'est utile pour signaler des erreurs dans vos propres fonctions lorsque les arguments sont invalides ou qu'une condition métier n'est pas respectée.

Python
def retirer(solde, montant):
    """Retire un montant du solde."""
    if montant <= 0:
        raise ValueError("Le montant doit être positif")
    if montant > solde:
        raise ValueError(f"Solde insuffisant ({solde}€ disponibles)")
    return solde - montant

# Utilisation
try:
    nouveau_solde = retirer(100, 150)
except ValueError as e:
    print(f"Erreur : {e}")
    # Erreur : Solde insuffisant (100€ disponibles)

# Re-lever une exception (propagation)
def traiter_donnees(data):
    try:
        resultat = data["valeur"] * 2
    except KeyError:
        print("Clé manquante, on re-lève l'exception")
        raise  # Re-lève l'exception originale

Exceptions personnalisées

Pour des projets complexes, créez vos propres classes d'exceptions en héritant de Exception. Cela permet de distinguer les erreurs spécifiques à votre application.

Python
# Définir des exceptions personnalisées
class ErreurBancaire(Exception):
    """Exception de base pour les opérations bancaires."""
    pass

class SoldeInsuffisant(ErreurBancaire):
    """Levée quand le solde est insuffisant."""
    def __init__(self, solde, montant):
        self.solde = solde
        self.montant = montant
        super().__init__(
            f"Solde insuffisant : {solde}€ disponibles, "
            f"{montant}€ demandés"
        )

class CompteBloque(ErreurBancaire):
    """Levée quand le compte est bloqué."""
    pass

# Utilisation
class Compte:
    def __init__(self, solde=0, bloque=False):
        self.solde = solde
        self.bloque = bloque

    def retirer(self, montant):
        if self.bloque:
            raise CompteBloque("Ce compte est bloqué")
        if montant > self.solde:
            raise SoldeInsuffisant(self.solde, montant)
        self.solde -= montant

try:
    compte = Compte(100)
    compte.retirer(200)
except SoldeInsuffisant as e:
    print(e)          # Solde insuffisant : 100€ disponibles, 200€ demandés
    print(e.solde)    # 100
    print(e.montant)  # 200
except ErreurBancaire as e:
    print(f"Erreur bancaire : {e}")

Chaînage d'exceptions (raise from)

Python 3 permet de chaîner les exceptions avec raise ... from ... pour conserver la cause originale d'une erreur tout en levant une exception plus appropriée.

Python
class ErreurConfiguration(Exception):
    pass

def charger_config(chemin):
    try:
        with open(chemin) as f:
            config = f.read()
    except FileNotFoundError as e:
        # Chaîner : l'exception originale est conservée
        raise ErreurConfiguration(
            f"Fichier de config manquant : {chemin}"
        ) from e

try:
    charger_config("config.yaml")
except ErreurConfiguration as e:
    print(e)            # Fichier de config manquant : config.yaml
    print(e.__cause__)  # [Errno 2] No such file or directory: 'config.yaml'

# Supprimer le chaînage explicitement
try:
    x = int("abc")
except ValueError:
    raise RuntimeError("Valeur invalide") from None

Bonnes pratiques

  • Ne jamais utiliser un except nu : except: sans type attrape tout, y compris KeyboardInterrupt et SystemExit. Utilisez au minimum except Exception:.
  • Cibler les exceptions spécifiques : attrapez ValueError plutôt que Exception quand c'est possible.
  • Ne pas ignorer les exceptions : évitez les blocs except: pass qui masquent les erreurs.
  • Utiliser le context manager (with) : préférez with open(...) à try/finally pour la gestion de fichiers.
  • Documenter les exceptions : indiquez dans la docstring quelles exceptions une fonction peut lever.
  • Créer une hiérarchie : pour un projet, créez une exception de base puis des sous-classes spécifiques.
Python
# ❌ Mauvais — attrape tout et ignore
try:
    faire_quelque_chose()
except:
    pass

# ✅ Bon — spécifique et informatif
try:
    faire_quelque_chose()
except ValueError as e:
    logger.error(f"Valeur invalide : {e}")
    raise

# ✅ Utiliser logging pour les erreurs en production
import logging

logger = logging.getLogger(__name__)

try:
    resultat = operation_critique()
except Exception as e:
    logger.exception("Erreur lors de l'opération critique")
    raise

Quiz

🐍 Exceptions — Quiz Score : 0 / 6

Question 1 / 6

Quel bloc s'exécute uniquement si aucune exception n'est levée dans le try ?

Le bloc else après try/except ne s'exécute que si aucune exception n'a été levée. finally s'exécute toujours.

Question 2 / 6

Pourquoi éviter except: sans spécifier de type ?

Un except: nu attrape toutes les exceptions, y compris KeyboardInterrupt (Ctrl+C) et SystemExit. Utilisez au minimum except Exception:.

Question 3 / 6

Que fait raise sans argument dans un bloc except ?

raise sans argument dans un bloc except propage l'exception originale, conservant le traceback complet.

Question 4 / 6

Comment créer une exception personnalisée ?

Une exception personnalisée est une classe qui hérite de Exception (ou d'une de ses sous-classes). On utilise class, pas def.

Question 5 / 6

Quel est le rôle de raise ... from ... ?

raise X from Y lève l'exception X en indiquant que Y en est la cause. L'attribut __cause__ pointe vers l'exception originale.

Question 6 / 6

Dans quel ordre doit-on placer les blocs except ?

Les except les plus spécifiques doivent venir en premier. Un except Exception en premier attraperait toutes les exceptions, rendant les blocs suivants inaccessibles.

Exercice 1 — Conversion sécurisée

Écrivez une fonction convertir_en_nombre(texte) qui tente de convertir une chaîne en int, puis en float si l'entier échoue. Si les deux échouent, retournez None.

Voir la solution 
def convertir_en_nombre(texte):
    """Convertit une chaîne en int ou float, ou retourne None."""
    try:
        return int(texte)
    except ValueError:
        try:
            return float(texte)
        except ValueError:
            return None

# Tests
print(convertir_en_nombre("42"))      # 42 (int)
print(convertir_en_nombre("3.14"))    # 3.14 (float)
print(convertir_en_nombre("abc"))     # None
print(convertir_en_nombre("  7  "))   # 7 (int)

Exercice 2 — Validation avec exceptions

Créez une fonction valider_age(age) qui lève un TypeError si l'argument n'est pas un entier, un ValueError si l'âge est négatif ou supérieur à 150. Sinon, retournez l'âge.

Voir la solution 
def valider_age(age):
    """Valide et retourne un âge."""
    if not isinstance(age, int):
        raise TypeError(f"L'âge doit être un entier, reçu : {type(age).__name__}")
    if age < 0:
        raise ValueError(f"L'âge ne peut pas être négatif : {age}")
    if age > 150:
        raise ValueError(f"L'âge semble irréaliste : {age}")
    return age

# Tests
try:
    print(valider_age(25))       # 25
    print(valider_age(-5))       # ValueError
except (TypeError, ValueError) as e:
    print(f"Erreur : {e}")

try:
    print(valider_age("vingt"))  # TypeError
except (TypeError, ValueError) as e:
    print(f"Erreur : {e}")

Exercice 3 — Exceptions personnalisées

Créez une hiérarchie d'exceptions pour un système de gestion de stock : ErreurStock (base), StockInsuffisant et ProduitInexistant. Implémentez une classe Inventaire avec les méthodes ajouter(), retirer() et consulter().

Voir la solution 
class ErreurStock(Exception):
    """Exception de base pour le stock."""
    pass

class StockInsuffisant(ErreurStock):
    def __init__(self, produit, disponible, demande):
        self.produit = produit
        self.disponible = disponible
        self.demande = demande
        super().__init__(
            f"Stock insuffisant pour '{produit}' : "
            f"{disponible} dispo, {demande} demandés"
        )

class ProduitInexistant(ErreurStock):
    def __init__(self, produit):
        self.produit = produit
        super().__init__(f"Produit inexistant : '{produit}'")

class Inventaire:
    def __init__(self):
        self.stock = {}

    def ajouter(self, produit, quantite):
        self.stock[produit] = self.stock.get(produit, 0) + quantite

    def retirer(self, produit, quantite):
        if produit not in self.stock:
            raise ProduitInexistant(produit)
        if self.stock[produit] < quantite:
            raise StockInsuffisant(produit, self.stock[produit], quantite)
        self.stock[produit] -= quantite

    def consulter(self, produit):
        if produit not in self.stock:
            raise ProduitInexistant(produit)
        return self.stock[produit]

# Tests
inv = Inventaire()
inv.ajouter("pommes", 50)

try:
    inv.retirer("pommes", 100)
except StockInsuffisant as e:
    print(e)  # Stock insuffisant pour 'pommes' : 50 dispo, 100 demandés

try:
    inv.consulter("bananes")
except ProduitInexistant as e:
    print(e)  # Produit inexistant : 'bananes'

Exercice 4 — Gestionnaire de contexte avec finally

Écrivez une fonction lire_json(chemin) qui lit un fichier JSON et retourne le contenu sous forme de dictionnaire. Gérez les exceptions FileNotFoundError, json.JSONDecodeError et PermissionError avec des messages clairs.

Voir la solution 
import json

def lire_json(chemin):
    """Lit un fichier JSON et retourne son contenu."""
    try:
        with open(chemin, "r", encoding="utf-8") as f:
            return json.load(f)
    except FileNotFoundError:
        print(f"Erreur : le fichier '{chemin}' n'existe pas")
    except json.JSONDecodeError as e:
        print(f"Erreur : JSON invalide dans '{chemin}' — {e.msg}")
    except PermissionError:
        print(f"Erreur : permission refusée pour '{chemin}'")
    return None

# Tests
data = lire_json("config.json")
if data is not None:
    print(f"Configuration chargée : {data}")
else:
    print("Impossible de charger la configuration")

Exercice 5 — Décorateur de gestion d'erreur

Créez un décorateur @gerer_erreurs qui intercepte toute exception dans la fonction décorée, affiche un message d'erreur formaté et retourne une valeur par défaut (passée en argument du décorateur).

Voir la solution 
import functools

def gerer_erreurs(valeur_defaut=None):
    """Décorateur qui intercepte les exceptions et retourne une valeur par défaut."""
    def decorateur(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            try:
                return func(*args, **kwargs)
            except Exception as e:
                print(f"[ERREUR] {func.__name__}: {type(e).__name__} — {e}")
                return valeur_defaut
        return wrapper
    return decorateur

@gerer_erreurs(valeur_defaut=0)
def diviser(a, b):
    return a / b

@gerer_erreurs(valeur_defaut=[])
def charger_donnees(chemin):
    with open(chemin) as f:
        return f.readlines()

# Tests
print(diviser(10, 2))     # 5.0
print(diviser(10, 0))     # [ERREUR] diviser: ZeroDivisionError — ...  → 0
print(charger_donnees("inexistant.txt"))  # [ERREUR] ...  → []
