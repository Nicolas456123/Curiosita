Introduction

Structurer son code Python

Un module est simplement un fichier .py contenant du code Python. Un package est un dossier contenant des modules et un fichier __init__.py. Ensemble, ils permettent de découper un projet en composants logiques, réutilisables et maintenables.

Python dispose d'une bibliothèque standard riche (plus de 200 modules) et d'un écosystème de packages tiers immense via PyPI (Python Package Index). L'outil pip permet d'installer ces packages, et les environnements virtuels isolent les dépendances de chaque projet.

Philosophie Python : « Batteries included » (piles incluses). La bibliothèque standard couvre la plupart des besoins courants : fichiers, réseau, JSON, bases de données, tests, expressions régulières, etc.

Le système d'import

Python offre plusieurs façons d'importer des modules. Chaque forme a ses avantages et ses conventions.

Python
# Import standard — le module entier
import math
print(math.sqrt(16))  # 4.0
print(math.pi)         # 3.141592653589793

# Import avec alias
import datetime as dt
maintenant = dt.datetime.now()

# Import spécifique — seulement ce dont on a besoin
from math import sqrt, pi
print(sqrt(25))  # 5.0 — pas besoin de math.sqrt

# Import avec renommage
from collections import defaultdict as dd

# ❌ À éviter : import étoile (pollue le namespace)
# from math import *

# Vérifier ce qu'un module contient
import json
print(dir(json))        # Liste tous les attributs
print(json.__file__)    # Chemin du fichier source
print(json.__version__) # Version (si disponible)

Créer ses propres modules

N'importe quel fichier .py est un module importable. Le nom du module est le nom du fichier sans l'extension.

Python
# ─── fichier : utils.py ───
"""Module utilitaire avec des fonctions réutilisables."""

PI = 3.14159

def aire_cercle(rayon):
    """Calcule l'aire d'un cercle."""
    return PI * rayon ** 2

def perimetre_cercle(rayon):
    """Calcule le périmètre d'un cercle."""
    return 2 * PI * rayon

# Code exécuté uniquement si on lance ce fichier directement
if __name__ == "__main__":
    print(f"Aire (r=5) : {aire_cercle(5):.2f}")
    print(f"Périmètre (r=5) : {perimetre_cercle(5):.2f}")

# ─── fichier : main.py ───
import utils

print(utils.PI)                    # 3.14159
print(utils.aire_cercle(10))       # 314.159

# Ou import spécifique
from utils import aire_cercle
print(aire_cercle(3))              # 28.274...

Le garde if __name__ == "__main__" : quand Python exécute un fichier directement, __name__ vaut "__main__". Quand il est importé, __name__ vaut le nom du module. Ce test empêche l'exécution du code de test lors d'un import.

Packages et __init__.py

Un package est un dossier contenant un fichier __init__.py (même vide). Il permet de regrouper des modules liés dans une hiérarchie.

Python
# Structure d'un package
# geometrie/
#   __init__.py
#   cercle.py
#   rectangle.py
#   triangle.py

# ─── geometrie/__init__.py ───
"""Package de calculs géométriques."""
from .cercle import aire_cercle, perimetre_cercle
from .rectangle import aire_rectangle

# __all__ contrôle ce que 'from geometrie import *' exporte
__all__ = ["aire_cercle", "perimetre_cercle", "aire_rectangle"]

# ─── geometrie/cercle.py ───
import math

def aire_cercle(rayon):
    return math.pi * rayon ** 2

def perimetre_cercle(rayon):
    return 2 * math.pi * rayon

# ─── Utilisation depuis l'extérieur ───
# Grâce à __init__.py, on peut importer directement :
from geometrie import aire_cercle
print(aire_cercle(5))

# Ou accéder au sous-module :
from geometrie.cercle import perimetre_cercle

# Imports relatifs (à l'intérieur du package)
# from .cercle import aire_cercle     # même niveau
# from ..utils import helper           # niveau parent

pip et PyPI

pip est le gestionnaire de packages de Python. Il télécharge et installe des packages depuis PyPI (Python Package Index), le dépôt officiel qui héberge plus de 400 000 packages.

Terminal
# Installer un package
pip install requests

# Installer une version spécifique
pip install requests==2.31.0

# Installer avec contrainte de version
pip install "requests>=2.28,<3.0"

# Mettre à jour un package
pip install --upgrade requests

# Désinstaller
pip uninstall requests

# Lister les packages installés
pip list

# Afficher les infos d'un package
pip show requests

# Sauvegarder les dépendances
pip freeze > requirements.txt

# Installer depuis un fichier requirements
pip install -r requirements.txt
requirements.txt
# Fichier requirements.txt typique
requests==2.31.0
flask>=3.0
numpy~=1.26.0
python-dotenv
pytest>=7.0  # dépendance de développement

Environnements virtuels

Un environnement virtuel isole les dépendances de chaque projet. Sans cela, tous les projets partagent les mêmes packages, ce qui cause des conflits de versions.

Terminal
# Créer un environnement virtuel
python -m venv mon_env

# Activer l'environnement
# Sur Linux/macOS :
source mon_env/bin/activate
# Sur Windows :
mon_env\Scripts\activate

# Le prompt change : (mon_env) $

# Installer des packages (isolés dans l'env)
pip install requests flask

# Vérifier l'isolation
which python    # → mon_env/bin/python
pip list        # Seulement les packages de cet env

# Sauvegarder les dépendances
pip freeze > requirements.txt

# Désactiver l'environnement
deactivate

# Recréer l'environnement sur une autre machine
python -m venv mon_env
source mon_env/bin/activate
pip install -r requirements.txt

Structure de projet professionnelle

Un projet Python bien structuré suit des conventions qui facilitent la collaboration, les tests et la distribution.

Structure
# Structure recommandée pour un projet Python
mon_projet/
├── README.md              # Description du projet
├── LICENSE                # Licence open source
├── pyproject.toml         # Configuration du projet (moderne)
├── requirements.txt       # Dépendances de production
├── requirements-dev.txt   # Dépendances de développement
├── .gitignore             # Fichiers à ignorer par git
├── .env                   # Variables d'environnement (pas dans git !)
│
├── src/                   # Code source
│   └── mon_app/
│       ├── __init__.py
│       ├── __main__.py    # Point d'entrée (python -m mon_app)
│       ├── config.py      # Configuration
│       ├── models.py      # Modèles de données
│       └── utils.py       # Fonctions utilitaires
│
├── tests/                 # Tests unitaires
│   ├── __init__.py
│   ├── test_models.py
│   └── test_utils.py
│
└── docs/                  # Documentation
    └── index.md
Python
# ─── .gitignore typique pour Python ───
# __pycache__/
# *.pyc

# venv/
# *.egg-info/

"""Point d'entrée : python -m mon_app"""

    config = charger_config()

Modules de la bibliothèque standard

Python offre une bibliothèque standard extrêmement riche. Voici les modules les plus utiles à connaître.

Python
# os et pathlib — manipulation de fichiers et chemins
from pathlib import Path

projet = Path("mon_projet")
projet.mkdir(exist_ok=True)
(projet / "config.json").write_text('{"debug": true}')

for fichier in Path(".").glob("**/*.py"):
    print(fichier)

# json — sérialisation JSON
import json

data = {"nom": "Alice", "notes": [15, 18, 12]}
texte_json = json.dumps(data, indent=2, ensure_ascii=False)
print(texte_json)

# collections — structures de données avancées

mots = "le chat mange le poisson du chat".split()

print(demain.strftime("%d/%m/%Y %H:%M"))

Exercice 1 — Créer un module utilitaire

Créez un module maths_utils.py contenant les fonctions factorielle(n), est_premier(n) et pgcd(a, b). Ajoutez un bloc if __name__ == "__main__" avec des tests.

Voir la solution 
# ─── maths_utils.py ───
"""Module de fonctions mathématiques utilitaires."""

def factorielle(n):
    """Calcule n! (factorielle de n)."""
    if n < 0:
        raise ValueError("n doit être positif")
    if n <= 1:
        return 1
    return n * factorielle(n - 1)

def est_premier(n):
    """Vérifie si n est un nombre premier."""
    if n < 2:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False
    return True

def pgcd(a, b):
    """Calcule le PGCD par l'algorithme d'Euclide."""
    while b:
        a, b = b, a % b
    return a

if __name__ == "__main__":
    # Tests quand le module est lancé directement
    print(f"5! = {factorielle(5)}")        # 120
    print(f"17 premier ? {est_premier(17)}") # True
    print(f"PGCD(48, 18) = {pgcd(48, 18)}")  # 6

Exercice 2 — Créer un package

from .rectangle import perimetre as perimetre_rectangle
from .triangle import aire as aire_triangle

    "aire_cercle", "perimetre_cercle",
    "aire_rectangle", "perimetre_rectangle",

# ─── geometrie/cercle.py ───
import math

def aire(rayon):
    return math.pi * rayon ** 2

def perimetre(rayon):
    return 2 * math.pi * rayon

# ─── geometrie/rectangle.py ───

Exercice 4 — Module de configuration

Créez un module config.py qui charge la configuration depuis un fichier JSON. S'il n'existe pas, créez-le avec des valeurs par défaut. Implémentez des fonctions charger(), sauvegarder() et obtenir(cle, defaut).

Voir la solution 
# ─── config.py ───
"""Module de gestion de configuration JSON."""
import json
from pathlib import Path

CHEMIN_CONFIG = Path("config.json")
DEFAUTS = {
    "debug": False,
    "langue": "fr",
    "port": 8080,
    "max_connexions": 100,
}

_config = None  # Cache en mémoire

def charger(chemin=None):
    """Charge la configuration depuis le fichier JSON."""
    global _config
    chemin = Path(chemin) if chemin else CHEMIN_CONFIG

    if not chemin.exists():
        _config = DEFAUTS.copy()
        sauvegarder(chemin)
        print(f"Config créée avec les défauts : {chemin}")
    else:
        with open(chemin, encoding="utf-8") as f:
            _config = json.load(f)
    return _config

def sauvegarder(chemin=None):
    """Sauvegarde la configuration dans le fichier JSON."""
    chemin = Path(chemin) if chemin else CHEMIN_CONFIG
    with open(chemin, "w", encoding="utf-8") as f:
        json.dump(_config, f, indent=2, ensure_ascii=False)

def obtenir(cle, defaut=None):
    """Retourne une valeur de configuration."""
    if _config is None:
        charger()
    return _config.get(cle, defaut)

def definir(cle, valeur, persister=True):
    """Modifie une valeur de configuration."""
    if _config is None:
        charger()
    _config[cle] = valeur
    if persister:
        sauvegarder()

if __name__ == "__main__":
    config = charger()
    print(f"Debug : {obtenir('debug')}")
    print(f"Port : {obtenir('port')}")

Exercice 5 — Explorer la bibliothèque standard

Utilisez les modules collections, itertools et functools pour : (1) compter les mots d'un texte avec Counter, (2) créer un dictionnaire qui auto-initialise les listes avec defaultdict, (3) utiliser functools.reduce pour calculer le produit d'une liste.

Voir la solution 
from collections import Counter, defaultdict
from functools import reduce
import operator

# 1. Counter — compter les mots
texte = "le chat noir mange le poisson rouge du chat noir"
mots = texte.split()
compteur = Counter(mots)
print("Mots les plus fréquents :")
for mot, nb in compteur.most_common(3):
    print(f"  {mot}: {nb} fois")
# chat: 2, le: 2, noir: 2

# 2. defaultdict — grouper par catégorie
eleves = [
    ("math", "Alice"), ("français", "Bob"),
    ("math", "Clara"), ("français", "Alice"),
    ("science", "Bob"), ("math", "David"),
]

par_matiere = defaultdict(list)
for matiere, eleve in eleves:
    par_matiere[matiere].append(eleve)

for matiere, noms in par_matiere.items():
    print(f"{matiere}: {', '.join(noms)}")

# 3. reduce — produit des éléments
nombres = [2, 3, 4, 5]
produit = reduce(operator.mul, nombres)
print(f"Produit de {nombres} = {produit}")  # 120

# Équivalent avec lambda
produit = reduce(lambda a, b: a * b, nombres)
print(f"Produit (lambda) = {produit}")  # 120
