Introduction

Qu'est-ce qu'un décorateur

Un décorateur est une fonction qui prend une autre fonction en argument et retourne une nouvelle fonction enrichie. C'est un patron de conception puissant qui permet d'ajouter des comportements (logging, validation, cache, authentification) sans modifier le code original.

Les décorateurs reposent sur un concept fondamental de Python : les fonctions sont des objets de première classe. On peut les stocker dans des variables, les passer en argument, et les retourner depuis d'autres fonctions. La syntaxe @ n'est qu'un sucre syntaxique élégant pour cette mécanique.

En résumé : @mon_decorateur au-dessus d'une fonction def f() équivaut exactement à écrire f = mon_decorateur(f) après la définition. Le décorateur enveloppe la fonction originale.

Fonctions d'ordre supérieur

Avant de comprendre les décorateurs, il faut maîtriser le concept de closure (fermeture) : une fonction interne qui capture les variables de la fonction englobante.

Python
# Les fonctions sont des objets
def saluer(nom):
    return f"Bonjour {nom}"

# Assigner une fonction à une variable
dire_bonjour = saluer
print(dire_bonjour("Alice"))  # Bonjour Alice

# Passer une fonction en argument
def executer(func, valeur):
    return func(valeur)

print(executer(saluer, "Bob"))  # Bonjour Bob

# Retourner une fonction (closure)
def multiplicateur(facteur):
    def multiplier(x):
        return x * facteur  # facteur est capturé
    return multiplier

doubler = multiplicateur(2)
tripler = multiplicateur(3)
print(doubler(5))   # 10
print(tripler(5))   # 15

Premier décorateur

Un décorateur est simplement une fonction qui prend une fonction, crée un wrapper autour d'elle, et retourne ce wrapper.

Python
# Décorateur sans la syntaxe @
def en_majuscules(func):
    def wrapper(*args, **kwargs):
        resultat = func(*args, **kwargs)
        return resultat.upper()
    return wrapper

def saluer(nom):
    return f"Bonjour {nom}"

# Application manuelle du décorateur
saluer = en_majuscules(saluer)
print(saluer("Alice"))  # BONJOUR ALICE

# Avec la syntaxe @ (identique, mais plus lisible)
@en_majuscules
def saluer(nom):
    return f"Bonjour {nom}"

print(saluer("Bob"))  # BONJOUR BOB

functools.wraps — Préserver les métadonnées

Sans @functools.wraps, le wrapper remplace le nom et la docstring de la fonction originale. Ce décorateur standard copie les métadonnées de la fonction décorée vers le wrapper.

Python
import functools

# ❌ Sans wraps — les métadonnées sont perdues
def mon_deco_sans(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@mon_deco_sans
def ma_fonction():
    """Ma super docstring."""
    pass

print(ma_fonction.__name__)  # wrapper (mauvais !)
print(ma_fonction.__doc__)   # None (perdu !)

# ✅ Avec wraps — les métadonnées sont conservées
def mon_deco_avec(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@mon_deco_avec
def ma_fonction():
    """Ma super docstring."""
    pass

print(ma_fonction.__name__)  # ma_fonction ✓
print(ma_fonction.__doc__)   # Ma super docstring. ✓

Décorateurs pratiques

Les décorateurs sont omniprésents en Python : mesure de performance, mise en cache, contrôle d'accès, retry automatique, etc.

Python
import functools
import time

# Décorateur de chronométrage
def chrono(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        debut = time.perf_counter()
        resultat = func(*args, **kwargs)
        duree = time.perf_counter() - debut
        print(f"{func.__name__} exécutée en {duree:.4f}s")
        return resultat
    return wrapper

@chrono
def calcul_lourd(n):
    return sum(i**2 for i in range(n))

calcul_lourd(1_000_000)
# calcul_lourd exécutée en 0.1234s

# Décorateur de logging
def log_appel(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"→ Appel de {func.__name__}({args}, {kwargs})")
        resultat = func(*args, **kwargs)
        print(f"← {func.__name__} retourne {resultat}")
        return resultat
    return wrapper

@log_appel
def additionner(a, b):
    return a + b

additionner(3, 5)
# → Appel de additionner((3, 5), {})
# ← additionner retourne 8

Décorateurs avec arguments

Pour passer des arguments à un décorateur, il faut ajouter un niveau d'imbrication supplémentaire : une fonction qui retourne le décorateur.

Python
import functools

# Décorateur avec arguments — triple imbrication
def repeter(n=2):
    """Décorateur qui répète l'appel n fois."""
    def decorateur(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                resultat = func(*args, **kwargs)
            return resultat
        return wrapper
    return decorateur

@repeter(n=3)
def dire_bonjour(nom):
    print(f"Bonjour {nom} !")

dire_bonjour("Alice")
# Bonjour Alice !
# Bonjour Alice !
# Bonjour Alice !

# Décorateur de retry avec délai
def retry(max_tentatives=3, delai=1):
    def decorateur(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for tentative in range(1, max_tentatives + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"Tentative {tentative}/{max_tentatives} échouée : {e}")
                    if tentative < max_tentatives:
                        time.sleep(delai)
            raise RuntimeError(f"Échec après {max_tentatives} tentatives")
        return wrapper
    return decorateur

@retry(max_tentatives=3, delai=0.5)
def operation_fragile():
    # Simule une opération qui peut échouer
    import random
    if random.random() < 0.7:
        raise ConnectionError("Connexion perdue")
    return "Succès !"

Empiler des décorateurs

On peut appliquer plusieurs décorateurs à une même fonction. Ils s'appliquent de bas en haut (le plus proche de def s'exécute en premier).

Python
# Empiler des décorateurs
@chrono
@log_appel
def traiter(x):
    return x * 2

# Équivalent à : traiter = chrono(log_appel(traiter))
# Ordre d'exécution :
# 1. chrono démarre le timer
# 2. log_appel affiche l'appel
# 3. traiter s'exécute
# 4. log_appel affiche le retour
# 5. chrono affiche la durée

traiter(21)
# → Appel de traiter((21,), {})
# ← traiter retourne 42
# traiter exécutée en 0.0001s

Décorateurs de classe et décorateurs intégrés

Python fournit plusieurs décorateurs intégrés. On peut aussi créer des décorateurs sous forme de classes avec la méthode __call__.

Python
# Décorateurs intégrés de Python
class Cercle:
    def __init__(self, rayon):
        self._rayon = rayon

    @property
    def rayon(self):
        return self._rayon

    @rayon.setter
    def rayon(self, valeur):
        if valeur < 0:
            raise ValueError("Le rayon doit être positif")
        self._rayon = valeur

    @staticmethod
    def est_valide(rayon):
        return rayon > 0

    @classmethod
    def depuis_diametre(cls, diametre):
        return cls(diametre / 2)

# functools.lru_cache — mise en cache automatique
from functools import lru_cache

@lru_cache(maxsize=128)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(100))  # Instantané grâce au cache

# Décorateur sous forme de classe
class Compteur:
    def __init__(self, func):
        functools.update_wrapper(self, func)
        self.func = func
        self.appels = 0

    def __call__(self, *args, **kwargs):
        self.appels += 1
        print(f"{self.func.__name__} appelée {self.appels} fois")
        return self.func(*args, **kwargs)

@Compteur
def dire_hello():
    print("Hello !")

dire_hello()  # dire_hello appelée 1 fois → Hello !
dire_hello()  # dire_hello appelée 2 fois → Hello !

Exercice 1 — Décorateur de validation

Créez un décorateur @valider_positifs qui vérifie que tous les arguments numériques passés à une fonction sont positifs. Si ce n'est pas le cas, levez un ValueError.

Voir la solution 
import functools

def valider_positifs(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        for arg in args:
            if isinstance(arg, (int, float)) and arg < 0:
                raise ValueError(f"Argument négatif interdit : {arg}")
        for cle, val in kwargs.items():
            if isinstance(val, (int, float)) and val < 0:
                raise ValueError(f"Argument '{cle}' négatif interdit : {val}")
        return func(*args, **kwargs)
    return wrapper

@valider_positifs
def calculer_aire(largeur, hauteur):
    return largeur * hauteur

print(calculer_aire(5, 3))    # 15
# calculer_aire(-2, 3)        # ValueError: Argument négatif interdit : -2

Exercice 2 — Décorateur de cache simple

Implémentez votre propre décorateur @cache_simple qui mémorise les résultats des appels précédents dans un dictionnaire. Affichez « [CACHE] » quand un résultat est retourné depuis le cache.

Voir la solution 
import functools

def cache_simple(func):
    cache = {}

    @functools.wraps(func)
    def wrapper(*args):
        if args in cache:
            print(f"[CACHE] {func.__name__}{args}")
            return cache[args]
        resultat = func(*args)
        cache[args] = resultat
        return resultat

    wrapper.cache = cache  # Accès au cache depuis l'extérieur
    wrapper.vider_cache = lambda: cache.clear()
    return wrapper

@cache_simple
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(10))  # 55
print(fibonacci(10))  # [CACHE] fibonacci((10,)) → 55
print(len(fibonacci.cache))  # 11 entrées en cache

Exercice 3 — Décorateur avec argument : limitation de fréquence

Créez un décorateur @limiter(intervalle) qui empêche d'appeler une fonction plus d'une fois toutes les intervalle secondes. Si on appelle trop vite, affichez un avertissement et retournez le dernier résultat.

Voir la solution 
import functools
import time

def limiter(intervalle):
    """Limite la fréquence d'appel d'une fonction."""
    def decorateur(func):
        dernier_appel = [0]
        dernier_resultat = [None]

        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            maintenant = time.time()
            ecart = maintenant - dernier_appel[0]
            if ecart < intervalle:
                print(f"[LIMITÉ] Attendez encore {intervalle - ecart:.1f}s")
                return dernier_resultat[0]
            dernier_appel[0] = maintenant
            dernier_resultat[0] = func(*args, **kwargs)
            return dernier_resultat[0]
        return wrapper
    return decorateur

@limiter(intervalle=2)
def requete_api(url):
    print(f"Requête vers {url}...")
    return {"status": 200}

print(requete_api("/api/data"))    # Requête vers /api/data... → {status: 200}
print(requete_api("/api/data"))    # [LIMITÉ] Attendez encore 1.9s → {status: 200}
time.sleep(2)
print(requete_api("/api/data"))    # Requête vers /api/data... → {status: 200}

Exercice 4 — Décorateur de typage

Créez un décorateur @verifier_types qui prend en argument les types attendus et vérifie les arguments de la fonction au moment de l'appel. Exemple : @verifier_types(int, int) pour une fonction à deux paramètres entiers.

Voir la solution 
import functools

def verifier_types(*types_attendus):
    """Vérifie les types des arguments positionnels."""
    def decorateur(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for i, (arg, type_attendu) in enumerate(zip(args, types_attendus)):
                if not isinstance(arg, type_attendu):
                    raise TypeError(
                        f"Argument {i} de {func.__name__} : "
                        f"attendu {type_attendu.__name__}, "
                        f"reçu {type(arg).__name__}"
                    )
            return func(*args, **kwargs)
        return wrapper
    return decorateur

@verifier_types(str, int)
def repeter_texte(texte, fois):
    return texte * fois

print(repeter_texte("abc", 3))    # abcabcabc
# repeter_texte(123, 3)           # TypeError: Argument 0 : attendu str, reçu int

Exercice 5 — Décorateur de classe (Singleton)

Implémentez un décorateur de classe @singleton qui garantit qu'une seule instance d'une classe peut exister. Si on tente de créer une deuxième instance, retournez la première.

Voir la solution 
import functools

def singleton(cls):
    """Décorateur qui transforme une classe en singleton."""
    instances = {}

    @functools.wraps(cls, updated=[])
    def get_instance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]

    get_instance._instances = instances
    return get_instance

@singleton
class Configuration:
    def __init__(self):
        self.parametres = {}
        print("Configuration créée")

    def set(self, cle, valeur):
        self.parametres[cle] = valeur

# Tests
config1 = Configuration()  # Configuration créée
config2 = Configuration()  # Rien (même instance retournée)
print(config1 is config2)  # True

config1.set("theme", "sombre")
print(config2.parametres)  # {'theme': 'sombre'}
