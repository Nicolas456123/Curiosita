01

Introduction

Protéger les données internes

L'encapsulation est le principe de la POO qui consiste à regrouper les données et les méthodes qui les manipulent à l'intérieur d'un objet, tout en contrôlant l'accès depuis l'extérieur. L'objectif est de cacher les détails d'implémentation et d'exposer uniquement une interface publique stable.

En Java ou C++, on utilise des mots-clés explicites (private, protected, public) pour restreindre l'accès aux attributs. Python adopte une approche différente, basée sur des conventions de nommage et la philosophie « nous sommes tous des adultes consentants » — la langue fait confiance au programmeur, mais fournit des mécanismes pour signaler et protéger les attributs internes.

Pourquoi encapsuler ? Sans encapsulation, n'importe quel code peut modifier directement les données d'un objet, potentiellement en les rendant incohérentes. Par exemple, assigner un âge négatif ou un solde bancaire invalide. L'encapsulation permet de valider les données avant modification.

02

Le cours

Contrôler l'accès en Python

Attributs publics

Par défaut, tous les attributs en Python sont publics. N'importe quel code peut y accéder et les modifier directement :

class Personne:
    def __init__(self, nom, age):
        self.nom = nom    # public
        self.age = age    # public

p = Personne("Alice", 25)
p.age = -5  # aucune erreur ! Mais cela n'a pas de sens
print(p.age)  # -5 — donnée incohérente

Ce comportement est problématique : rien n'empêche d'assigner des valeurs invalides.

Convention du simple underscore (_)

Un attribut précédé d'un simple underscore signale qu'il est destiné à un usage interne. C'est une convention sociale, pas une restriction technique :

class MoteurRecherche:
    def __init__(self):
        self._index = {}       # usage interne — ne pas accéder directement
        self._nb_requetes = 0  # compteur interne

    def indexer(self, mot, document):
        if mot not in self._index:
            self._index[mot] = []
        self._index[mot].append(document)

    def rechercher(self, mot):
        self._nb_requetes += 1
        return self._index.get(mot, [])

m = MoteurRecherche()
m.indexer("python", "doc1.txt")
print(m.rechercher("python"))  # ['doc1.txt']

# Techniquement accessible, mais la convention dit "ne le faites pas"
print(m._index)  # {'python': ['doc1.txt']}  — fonctionne, mais déconseillé

Convention PEP 8 : un attribut ou une méthode commençant par _ est considéré comme « privé » par convention. Les IDE et linters peuvent émettre des avertissements si du code externe y accède. Lors d'un from module import *, les noms commençant par _ ne sont pas importés.

Name mangling avec double underscore (__)

Un attribut précédé de deux underscores (sans underscores finaux) déclenche le name mangling : Python renomme l'attribut pour rendre son accès direct plus difficile depuis l'extérieur :

class CompteBancaire:
    def __init__(self, titulaire, solde):
        self.__titulaire = titulaire  # name mangling
        self.__solde = solde          # name mangling

    def get_solde(self):
        return self.__solde

    def deposer(self, montant):
        if montant > 0:
            self.__solde += montant

compte = CompteBancaire("Alice", 1000)

# Accès direct impossible avec le nom original
# print(compte.__solde)  # AttributeError !

# Python a renommé l'attribut en _CompteBancaire__solde
print(compte._CompteBancaire__solde)  # 1000 — contournable mais déconseillé

# Accès via la méthode publique (bonne pratique)
print(compte.get_solde())  # 1000

Le name mangling transforme __attribut en _NomClasse__attribut. Ce n'est pas une vraie protection de sécurité, mais cela évite les collisions de noms dans les hiérarchies d'héritage.

Getters et setters classiques

On peut créer des méthodes explicites pour lire et modifier un attribut :

class Temperature:
    def __init__(self, celsius):
        self._celsius = celsius

    def get_celsius(self):
        return self._celsius

    def set_celsius(self, valeur):
        if valeur < -273.15:
            raise ValueError("Température en dessous du zéro absolu !")
        self._celsius = valeur

    def get_fahrenheit(self):
        return self._celsius * 9/5 + 32

t = Temperature(25)
print(t.get_celsius())      # 25
print(t.get_fahrenheit())   # 77.0
t.set_celsius(100)
# t.set_celsius(-300)       # ValueError !

Cette approche fonctionne mais n'est pas « pythonique ». Python offre un mécanisme plus élégant : @property.

Le décorateur @property

@property permet de définir des attributs « intelligents » : le code extérieur utilise une syntaxe d'attribut simple (obj.attr), mais en coulisse, des méthodes getter/setter/deleter sont appelées :

class Temperature:
    def __init__(self, celsius):
        self.celsius = celsius  # appelle le setter !

    @property
    def celsius(self):
        """Getter — appelé quand on lit obj.celsius"""
        return self._celsius

    @celsius.setter
    def celsius(self, valeur):
        """Setter — appelé quand on écrit obj.celsius = x"""
        if valeur < -273.15:
            raise ValueError("Température en dessous du zéro absolu !")
        self._celsius = valeur

    @property
    def fahrenheit(self):
        """Propriété en lecture seule (pas de setter)"""
        return self._celsius * 9/5 + 32

t = Temperature(25)
print(t.celsius)      # 25     — appelle le getter
print(t.fahrenheit)   # 77.0   — propriété calculée

t.celsius = 100       # appelle le setter avec validation
print(t.celsius)      # 100

# t.celsius = -300    # ValueError: Température en dessous du zéro absolu !
# t.fahrenheit = 50   # AttributeError: can't set attribute (lecture seule)

Avantage majeur : @property permet de commencer avec un attribut public simple, puis d'ajouter de la validation sans changer l'interface. Le code appelant écrit toujours obj.celsius = 100, que ce soit un attribut direct ou une property.

Propriétés calculées et lecture seule

Les propriétés sont très utiles pour exposer des valeurs calculées à partir d'autres attributs :

class Rectangle:
    def __init__(self, largeur, hauteur):
        self._largeur = largeur
        self._hauteur = hauteur

    @property
    def largeur(self):
        return self._largeur

    @largeur.setter
    def largeur(self, valeur):
        if valeur <= 0:
            raise ValueError("La largeur doit être positive")
        self._largeur = valeur

    @property
    def hauteur(self):
        return self._hauteur

    @hauteur.setter
    def hauteur(self, valeur):
        if valeur <= 0:
            raise ValueError("La hauteur doit être positive")
        self._hauteur = valeur

    @property
    def aire(self):
        """Propriété calculée — pas de setter"""
        return self._largeur * self._hauteur

    @property
    def perimetre(self):
        return 2 * (self._largeur + self._hauteur)

r = Rectangle(4, 6)
print(r.aire)       # 24
print(r.perimetre)  # 20
r.largeur = 10
print(r.aire)       # 60 — recalculé automatiquement

Le décorateur @property avec deleter

On peut aussi définir un deleter pour contrôler la suppression d'un attribut :

class Utilisateur:
    def __init__(self, nom, email):
        self.nom = nom
        self._email = email

    @property
    def email(self):
        return self._email

    @email.setter
    def email(self, valeur):
        if "@" not in valeur:
            raise ValueError("Email invalide : il faut un @")
        self._email = valeur

    @email.deleter
    def email(self):
        print(f"Suppression de l'email de {self.nom}")
        self._email = None

u = Utilisateur("Alice", "alice@example.com")
print(u.email)       # alice@example.com
u.email = "bob@mail.fr"
del u.email          # Suppression de l'email de Alice
print(u.email)       # None

Résumé des conventions Python

Voici un récapitulatif des niveaux d'accès en Python :

  • attribut — public, accessible par tous. Utilisé pour l'interface publique de la classe
  • _attribut — protégé par convention. Signale un détail d'implémentation interne. N'accédez pas depuis l'extérieur sauf si vous savez ce que vous faites
  • __attribut — name mangling (_Classe__attribut). Protège contre les collisions de noms dans l'héritage. Usage rare en pratique
  • @property — la solution pythonique pour l'encapsulation. Combinaison d'interface simple et de contrôle interne

Philosophie Python : « We're all consenting adults here. » Python ne force pas l'encapsulation stricte. Les conventions (_ et __) sont des signaux au programmeur, pas des verrous. La meilleure pratique : commencez par des attributs publics, ajoutez @property quand vous avez besoin de validation ou de logique supplémentaire.

Exercice 1 — Classe MotDePasse

Créez une classe MotDePasse avec un attribut _valeur. Utilisez @property pour que le setter vérifie que le mot de passe fait au moins 8 caractères, contient au moins un chiffre et une majuscule. La lecture retourne le mot de passe masqué ("****").

Voir la solution 
class MotDePasse:
    def __init__(self, valeur):
        self.valeur = valeur  # passe par le setter

    @property
    def valeur(self):
        return "****"  # ne révèle jamais le vrai mot de passe

    @valeur.setter
    def valeur(self, nouveau):
        if len(nouveau) < 8:
            raise ValueError("Au moins 8 caractères requis")
        if not any(c.isupper() for c in nouveau):
            raise ValueError("Au moins une majuscule requise")
        if not any(c.isdigit() for c in nouveau):
            raise ValueError("Au moins un chiffre requis")
        self._valeur = nouveau

    def verifier(self, tentative):
        return tentative == self._valeur

# Test
mdp = MotDePasse("Secret123")
print(mdp.valeur)              # ****
print(mdp.verifier("Secret123"))  # True
print(mdp.verifier("mauvais"))    # False

Exercice 2 — Thermostat avec @property

Créez une classe Thermostat avec une température en Celsius. Ajoutez des propriétés pour lire/écrire en Fahrenheit et en Kelvin. Le setter Celsius refuse les valeurs sous -273.15. Les propriétés Fahrenheit et Kelvin calculent et convertissent automatiquement.

Voir la solution 
class Thermostat:
    def __init__(self, celsius=20):
        self.celsius = celsius

    @property
    def celsius(self):
        return self._celsius

    @celsius.setter
    def celsius(self, valeur):
        if valeur < -273.15:
            raise ValueError("Impossible : sous le zéro absolu")
        self._celsius = valeur

    @property
    def fahrenheit(self):
        return self._celsius * 9/5 + 32

    @fahrenheit.setter
    def fahrenheit(self, valeur):
        self.celsius = (valeur - 32) * 5/9

    @property
    def kelvin(self):
        return self._celsius + 273.15

    @kelvin.setter
    def kelvin(self, valeur):
        self.celsius = valeur - 273.15

# Test
t = Thermostat(100)
print(t.fahrenheit)  # 212.0
print(t.kelvin)      # 373.15

t.fahrenheit = 32
print(t.celsius)     # 0.0

t.kelvin = 0
print(t.celsius)     # -273.15

Exercice 3 — Historique de modifications

Créez une classe Valeur dont l'attribut donnee est une property. Chaque modification enregistre l'ancienne valeur dans un historique. Ajoutez une méthode annuler() qui restaure la valeur précédente.

Voir la solution 
class Valeur:
    def __init__(self, initiale):
        self._historique = []
        self._donnee = initiale

    @property
    def donnee(self):
        return self._donnee

    @donnee.setter
    def donnee(self, nouvelle):
        self._historique.append(self._donnee)
        self._donnee = nouvelle

    def annuler(self):
        if self._historique:
            self._donnee = self._historique.pop()
        else:
            print("Aucune modification à annuler")

    @property
    def historique(self):
        return list(self._historique)  # copie défensive

# Test
v = Valeur(10)
v.donnee = 20
v.donnee = 30
v.donnee = 42
print(v.donnee)      # 42
print(v.historique)   # [10, 20, 30]
v.annuler()
print(v.donnee)      # 30
v.annuler()
print(v.donnee)      # 20

Exercice 4 — Classe Etudiant encapsulée

Créez une classe Etudiant avec nom (lecture seule après création), notes (liste privée) et une propriété moyenne calculée. Ajoutez ajouter_note(note) qui vérifie que la note est entre 0 et 20.

Voir la solution 
class Etudiant:
    def __init__(self, nom):
        self._nom = nom
        self._notes = []

    @property
    def nom(self):
        return self._nom
    # Pas de setter → lecture seule

    @property
    def notes(self):
        return list(self._notes)  # copie défensive

    @property
    def moyenne(self):
        if not self._notes:
            return 0.0
        return sum(self._notes) / len(self._notes)

    def ajouter_note(self, note):
        if not 0 <= note <= 20:
            raise ValueError("La note doit être entre 0 et 20")
        self._notes.append(note)

# Test
e = Etudiant("Alice")
e.ajouter_note(15)
e.ajouter_note(18)
e.ajouter_note(12)
print(f"{e.nom} — Moyenne : {e.moyenne:.1f}")  # Alice — Moyenne : 15.0
# e.nom = "Bob"  # AttributeError: can't set attribute

Exercice 5 — Classe Couleur avec validation

Créez une classe Couleur avec des propriétés r, g, b (rouge, vert, bleu) qui valident que chaque composante est un entier entre 0 et 255. Ajoutez une propriété calculée hex qui retourne le code hexadécimal (ex: "#FF8000").

Voir la solution 
class Couleur:
    def __init__(self, r, g, b):
        self.r = r
        self.g = g
        self.b = b

    @staticmethod
    def _valider(valeur, nom):
        if not isinstance(valeur, int) or not 0 <= valeur <= 255:
            raise ValueError(f"{nom} doit être un entier entre 0 et 255")

    @property
    def r(self):
        return self._r

    @r.setter
    def r(self, valeur):
        Couleur._valider(valeur, "Rouge")
        self._r = valeur

    @property
    def g(self):
        return self._g

    @g.setter
    def g(self, valeur):
        Couleur._valider(valeur, "Vert")
        self._g = valeur

    @property
    def b(self):
        return self._b

    @b.setter
    def b(self, valeur):
        Couleur._valider(valeur, "Bleu")
        self._b = valeur

    @property
    def hex(self):
        return f"#{self._r:02X}{self._g:02X}{self._b:02X}"

# Test
c = Couleur(255, 128, 0)
print(c.hex)     # #FF8000
c.r = 0
print(c.hex)     # #008000
# c.g = 300      # ValueError: Vert doit être un entier entre 0 et 255
