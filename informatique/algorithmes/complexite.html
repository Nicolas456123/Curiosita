Introduction

Pourquoi mesurer l'efficacité ?

Tous les algorithmes ne se valent pas. Pour une même tâche — trier une liste de 10 millions d'éléments, par exemple — un algorithme peut terminer en quelques millisecondes tandis qu'un autre mettrait des heures. La complexité algorithmique est l'outil mathématique qui permet de quantifier cette différence et de prédire le comportement d'un algorithme à mesure que la taille des données augmente.

Pourquoi c'est essentiel : un algorithme en O(n²) qui fonctionne bien sur 1 000 éléments deviendra inutilisable sur 1 million d'éléments. Comprendre la complexité, c'est savoir anticiper les limites de vos programmes avant même de les exécuter.

La notation Big-O (grand O) est le langage universel pour exprimer la complexité. Elle décrit le taux de croissance d'un algorithme en fonction de la taille de l'entrée, en ignorant les constantes et les termes de moindre importance. Ce cours vous apprend à lire, comprendre et calculer la complexité de n'importe quel algorithme.

Le cours

Notation Big-O et analyse d'algorithmes

Qu'est-ce que la complexité ?

La complexité d'un algorithme mesure la quantité de ressources (temps ou mémoire) nécessaires pour résoudre un problème en fonction de la taille de l'entrée (notée n). On distingue :

  • Complexité temporelle : combien d'opérations élémentaires l'algorithme effectue.
  • Complexité spatiale : combien de mémoire supplémentaire l'algorithme utilise.

On s'intéresse généralement au pire cas (worst case), mais aussi parfois au cas moyen (average case) et au meilleur cas (best case).

La notation Big-O

La notation Big-O décrit la borne supérieure asymptotique du taux de croissance. Elle répond à la question : « Quand n devient très grand, comment le temps d'exécution évolue-t-il ? »

Hiérarchie des complexités
O(1)       Constant      — Accès à un élément d'un tableau
O(log n)   Logarithmique — Recherche dichotomique
O(n)       Linéaire      — Parcours d'une liste
O(n log n) Quasi-linéaire — Tri fusion, tri rapide (moyen)
O(n²)      Quadratique   — Tri par insertion (pire cas)
O(2ⁿ)      Exponentiel   — Sous-ensembles, force brute
O(n!)      Factoriel     — Permutations, voyageur de commerce

Exemples concrets en Python

Python
# O(1) — Temps constant
def premier_element(liste):
    return liste[0]  # Une seule opération, quelle que soit la taille

# O(n) — Temps linéaire
def somme(liste):
    total = 0
    for x in liste:   # On parcourt chaque élément une fois
        total += x
    return total

# O(n²) — Temps quadratique
def doublons(liste):
    for i in range(len(liste)):
        for j in range(i + 1, len(liste)):  # Boucle imbriquée
            if liste[i] == liste[j]:
                return True
    return False

# O(log n) — Temps logarithmique
def recherche_dichotomique(liste, cible):
    gauche, droite = 0, len(liste) - 1
    while gauche <= droite:
        milieu = (gauche + droite) // 2
        if liste[milieu] == cible:
            return milieu
        elif liste[milieu] < cible:
            gauche = milieu + 1  # On élimine la moitié à chaque étape
        else:
            droite = milieu - 1
    return -1

Règles de calcul

Pour déterminer la complexité d'un algorithme, appliquez ces règles :

  • Ignorer les constantes : O(3n) = O(n), O(n/2) = O(n).
  • Garder le terme dominant : O(n² + n) = O(n²), O(n + log n) = O(n).
  • Boucles séquentielles : on additionne (et on garde le terme dominant).
  • Boucles imbriquées : on multiplie les complexités.
  • Appels récursifs : on utilise le théorème maître ou on dessine l'arbre de récurrence.
Python
# Boucles séquentielles : O(n) + O(n) = O(n)
for x in liste:     # O(n)
    print(x)
for x in liste:     # O(n)
    print(x * 2)

# Boucles imbriquées : O(n) × O(n) = O(n²)
for i in range(n):   # O(n)
    for j in range(n): # × O(n)
        print(i, j)

# Division récursive : O(log n)
def diviser(n):
    if n <= 1:
        return
    print(n)
    diviser(n // 2)  # On divise par 2 à chaque appel

Complexité spatiale

La complexité spatiale mesure la mémoire supplémentaire utilisée par l'algorithme (en plus de l'entrée). Un algorithme « en place » utilise O(1) de mémoire supplémentaire.

Python
# O(1) en espace — en place
def inverser_en_place(liste):
    i, j = 0, len(liste) - 1
    while i < j:
        liste[i], liste[j] = liste[j], liste[i]
        i += 1
        j -= 1

# O(n) en espace — crée une nouvelle liste
def inverser_copie(liste):
    return liste[::-1]  # Nouvelle liste de taille n

Comparaison pratique

Pour n = 1 000 000 éléments, voici le nombre approximatif d'opérations :

O(1) : 1 opération | O(log n) : 20 opérations | O(n) : 1 000 000 | O(n log n) : 20 000 000 | O(n²) : 1 000 000 000 000 (infaisable en temps raisonnable)

O(3n²)
O(n²)
O(n² + n)
O(100)
On ignore les constantes multiplicatives (3) et les termes de moindre ordre (5n, 100). Le terme dominant est n², donc la complexité est O(n²).

Question 3 / 5

Si un algorithme divise l'espace de recherche par 2 à chaque étape, quelle est sa complexité ?

O(n)
O(n/2)
O(log n)
O(1)
Diviser par 2 à chaque étape donne log₂(n) étapes au total. C'est le principe de la recherche dichotomique, en O(log n).

Question 4 / 5

Quelle est la complexité de deux boucles imbriquées de taille n ?

O(n)
O(2n)
O(n²)
O(n log n)
Deux boucles imbriquées, chacune de taille n, donnent n × n = n² itérations, soit une complexité O(n²).

Question 5 / 5

Parmi ces complexités, laquelle est la plus efficace pour de grandes entrées ?

O(n)
O(log n)
O(n log n)
O(n²)
O(log n) croît le plus lentement parmi ces options. Pour n = 1 000 000, log₂(n) ≈ 20, ce qui est considérablement plus rapide que O(n) = 1 000 000.

Exercice 1 — Identifier la complexité

Déterminez la complexité temporelle de chaque fonction :

def f1(n):
    for i in range(n):
        for j in range(n):
            for k in range(n):
                print(i, j, k)

def f2(n):
    i = n
    while i > 0:
        print(i)
        i = i // 3
Voir la solution

f1 : Trois boucles imbriquées de taille n → O(n³).
f2 : On divise i par 3 à chaque itération → O(log₃ n) = O(log n).

Exercice 2 — Optimiser un algorithme

La fonction suivante vérifie si une liste contient des doublons en O(n²). Réécrivez-la en O(n) en utilisant un ensemble (set).

def a_doublons(liste):
    for i in range(len(liste)):
        for j in range(i + 1, len(liste)):
            if liste[i] == liste[j]:
                return True
    return False
Voir la solution 
def a_doublons(liste):
    vus = set()
    for x in liste:
        if x in vus:      # O(1) en moyenne pour un set
            return True
        vus.add(x)
    return False
# Complexité : O(n) en temps, O(n) en espace

Exercice 3 — Comparer des approches

Vous devez trouver la paire d'éléments dans une liste triée dont la somme vaut une cible donnée. Comparez une approche en O(n²) (deux boucles) et une approche en O(n) (deux pointeurs). Implémentez les deux et expliquez pourquoi la seconde est meilleure.

Voir la solution 
# O(n²) — Force brute
def paire_somme_brute(liste, cible):
    for i in range(len(liste)):
        for j in range(i + 1, len(liste)):
            if liste[i] + liste[j] == cible:
                return (liste[i], liste[j])
    return None

# O(n) — Deux pointeurs (liste triée)
def paire_somme_rapide(liste, cible):
    g, d = 0, len(liste) - 1
    while g < d:
        s = liste[g] + liste[d]
        if s == cible:
            return (liste[g], liste[d])
        elif s < cible:
            g += 1
        else:
            d -= 1
    return None
