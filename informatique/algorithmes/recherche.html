Introduction

Chercher et explorer

La recherche consiste à trouver un élément particulier dans une collection de données. C'est une opération omniprésente : chercher un mot dans un dictionnaire, un contact dans un répertoire, un fichier sur un disque. Le choix de l'algorithme de recherche dépend de la structure des données et de leur organisation (triées ou non).

Recherche vs Parcours : la recherche vise à trouver un élément précis. Le parcours vise à visiter tous les éléments d'une structure (arbre, graphe) dans un ordre donné. Les deux sont fondamentaux en algorithmique.

Ce cours couvre la recherche séquentielle (linéaire), la recherche dichotomique (binaire) et les parcours de graphes en profondeur (DFS) et en largeur (BFS). Ces algorithmes constituent la boîte à outils essentielle de tout développeur.

Le cours

Algorithmes de recherche et de parcours

Recherche séquentielle (linéaire)

La recherche séquentielle parcourt la liste élément par élément jusqu'à trouver la cible ou atteindre la fin. Elle fonctionne sur n'importe quelle liste, triée ou non.

Python
def recherche_sequentielle(liste, cible):
    for i, elem in enumerate(liste):
        if elem == cible:
            return i      # Trouvé à l'indice i
    return -1            # Non trouvé

# Complexité : O(n) pire cas, O(1) meilleur cas

Recherche dichotomique (binaire)

La recherche dichotomique exploite le fait que la liste est triée pour éliminer la moitié des candidats à chaque étape. Elle est en O(log n), ce qui est considérablement plus rapide que O(n) pour de grandes listes.

Python
def recherche_dichotomique(liste, cible):
    gauche, droite = 0, len(liste) - 1
    while gauche <= droite:
        milieu = (gauche + droite) // 2
        if liste[milieu] == cible:
            return milieu
        elif liste[milieu] < cible:
            gauche = milieu + 1   # Chercher dans la moitié droite
        else:
            droite = milieu - 1   # Chercher dans la moitié gauche
    return -1

# Complexité : O(log n) — pour n = 1 000 000, seulement ~20 étapes

Parcours en profondeur (DFS)

Le DFS (Depth-First Search) explore un graphe en allant aussi loin que possible dans chaque branche avant de revenir en arrière. Il utilise une pile (ou la pile d'appels via la récursion).

Python
def dfs(graphe, depart, visites=None):
    if visites is None:
        visites = set()
    visites.add(depart)
    print(depart, end=" ")

    for voisin in graphe[depart]:
        if voisin not in visites:
            dfs(graphe, voisin, visites)
    return visites

# Exemple de graphe (liste d'adjacence)
graphe = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'F'],
    'D': ['B'], 'E': ['B'], 'F': ['C']
}
dfs(graphe, 'A')  # A B D E C F

Parcours en largeur (BFS)

Le BFS (Breadth-First Search) explore un graphe niveau par niveau, en visitant tous les voisins d'un sommet avant de passer au niveau suivant. Il utilise une file (queue).

Python
from collections import deque

def bfs(graphe, depart):
    visites = {depart}
    file = deque([depart])
    ordre = []

    while file:
        sommet = file.popleft()
        ordre.append(sommet)
        for voisin in graphe[sommet]:
            if voisin not in visites:
                visites.add(voisin)
                file.append(voisin)
    return ordre

bfs(graphe, 'A')  # ['A', 'B', 'C', 'D', 'E', 'F']

DFS vs BFS : DFS explore en profondeur (idéal pour détecter des cycles, parcourir un labyrinthe). BFS explore par niveaux (idéal pour trouver le plus court chemin dans un graphe non pondéré).

Exercice 1 — Recherche récursive

Implémentez la recherche dichotomique de manière récursive (au lieu de la version itérative avec while).

Voir la solution 
def recherche_dicho_recursive(liste, cible, g=0, d=None):
    if d is None:
        d = len(liste) - 1
    if g > d:
        return -1
    m = (g + d) // 2
    if liste[m] == cible:
        return m
    elif liste[m] < cible:
        return recherche_dicho_recursive(liste, cible, m + 1, d)
    else:
        return recherche_dicho_recursive(liste, cible, g, m - 1)

Exercice 2 — Plus court chemin BFS

Modifiez le BFS pour qu'il retourne le plus court chemin (liste de sommets) entre deux sommets d'un graphe non pondéré.

Voir la solution 
from collections import deque

def plus_court_chemin(graphe, depart, arrivee):
    file = deque([(depart, [depart])])
    visites = {depart}
    while file:
        sommet, chemin = file.popleft()
        if sommet == arrivee:
            return chemin
        for voisin in graphe[sommet]:
            if voisin not in visites:
                visites.add(voisin)
                file.append((voisin, chemin + [voisin]))
    return None  # Pas de chemin

Exercice 3 — Composantes connexes

Utilisez le DFS pour trouver toutes les composantes connexes d'un graphe non orienté. Retournez une liste de listes de sommets.

Voir la solution 
def composantes_connexes(graphe):
    visites = set()
    composantes = []
    for sommet in graphe:
        if sommet not in visites:
            composante = []
            pile = [sommet]
            while pile:
                s = pile.pop()
                if s not in visites:
                    visites.add(s)
                    composante.append(s)
                    for v in graphe[s]:
                        if v not in visites:
                            pile.append(v)
            composantes.append(composante)
    return composantes
