Introduction

Structures non linéaires

Les arbres et les graphes sont des structures de données non linéaires qui modélisent des relations hiérarchiques (arbres) ou des connexions arbitraires (graphes). Un arbre généalogique, l'organisation d'un système de fichiers, le réseau routier d'une ville — ces structures sont partout. Leur maîtrise est indispensable pour résoudre des problèmes complexes en informatique.

Arbre vs Graphe : un arbre est un graphe particulier — connexe, acyclique et avec une racine. Tout arbre est un graphe, mais tous les graphes ne sont pas des arbres. Un graphe peut contenir des cycles et ne pas avoir de hiérarchie.

Ce cours explore les arbres binaires de recherche (ABR), les tas (heaps), et les algorithmes fondamentaux sur les graphes, notamment l'algorithme de Dijkstra pour trouver le plus court chemin dans un graphe pondéré.

Arbres binaires et algorithmes de graphes

Arbre binaire de recherche (ABR)

Un ABR est un arbre binaire où, pour chaque nœud, tous les éléments du sous-arbre gauche sont inférieurs et tous ceux du sous-arbre droit sont supérieurs. Cela permet une recherche, insertion et suppression en O(log n) en moyenne.

Python
class Noeud:
    def __init__(self, valeur):
        self.valeur = valeur
        self.gauche = None
        self.droite = None

def inserer(racine, valeur):
    if racine is None:
        return Noeud(valeur)
    if valeur < racine.valeur:
        racine.gauche = inserer(racine.gauche, valeur)
    else:
        racine.droite = inserer(racine.droite, valeur)
    return racine

def rechercher(racine, valeur):
    if racine is None or racine.valeur == valeur:
        return racine
    if valeur < racine.valeur:
        return rechercher(racine.gauche, valeur)
    return rechercher(racine.droite, valeur)

# Parcours infixe (donne les éléments triés)
def infixe(racine):
    if racine:
        infixe(racine.gauche)
        print(racine.valeur, end=" ")
        infixe(racine.droite)

Le tas (Heap)

Un tas est un arbre binaire complet où chaque nœud parent est supérieur (tas-max) ou inférieur (tas-min) à ses enfants. Il est utilisé pour implémenter les files de priorité et le tri par tas.

Python
import heapq

# heapq implémente un tas-min en Python
tas = []
heapq.heappush(tas, 5)
heapq.heappush(tas, 1)
heapq.heappush(tas, 3)

print(heapq.heappop(tas))  # 1 (le minimum)
print(heapq.heappop(tas))  # 3
print(heapq.heappop(tas))  # 5

# Insertion et extraction en O(log n)

Représentation d'un graphe

Un graphe peut être représenté par une liste d'adjacence (efficace en mémoire) ou une matrice d'adjacence (accès rapide). Pour un graphe pondéré, on associe un poids à chaque arête.

Python
# Liste d'adjacence (graphe pondéré)
graphe = {
    'A': {'B': 4, 'C': 2},
    'B': {'A': 4, 'D': 3, 'C': 1},
    'C': {'A': 2, 'B': 1, 'D': 5},
    'D': {'B': 3, 'C': 5}
}

Algorithme de Dijkstra

L'algorithme de Dijkstra trouve le plus court chemin depuis un sommet source vers tous les autres sommets dans un graphe pondéré à poids positifs. Il utilise une file de priorité (tas-min).

Python
import heapq

def dijkstra(graphe, source):
    distances = {s: float('inf') for s in graphe}
    distances[source] = 0
    file = [(0, source)]  # (distance, sommet)

    while file:
        dist, u = heapq.heappop(file)
        if dist > distances[u]:
            continue
        for v, poids in graphe[u].items():
            d = dist + poids
            if d < distances[v]:
                distances[v] = d
                heapq.heappush(file, (d, v))
    return distances

print(dijkstra(graphe, 'A'))
# {'A': 0, 'B': 3, 'C': 2, 'D': 6}

Complexité de Dijkstra : O((V + E) log V) avec une file de priorité binaire, où V est le nombre de sommets et E le nombre d'arêtes. L'algorithme ne fonctionne pas avec des poids négatifs — pour cela, on utilise Bellman-Ford en O(V × E).

Exercice 1 — Hauteur d'un arbre

Écrivez une fonction récursive qui calcule la hauteur d'un arbre binaire (le nombre d'arêtes du plus long chemin de la racine à une feuille).

Voir la solution 
def hauteur(racine):
    if racine is None:
        return -1
    return 1 + max(hauteur(racine.gauche), hauteur(racine.droite))

Exercice 2 — Vérifier un ABR

Écrivez une fonction qui vérifie si un arbre binaire est un ABR valide (chaque nœud respecte la propriété d'ordre).

Voir la solution 
def est_abr(racine, mini=float('-inf'), maxi=float('inf')):
    if racine is None:
        return True
    if racine.valeur <= mini or racine.valeur >= maxi:
        return False
    return (est_abr(racine.gauche, mini, racine.valeur) and
            est_abr(racine.droite, racine.valeur, maxi))

Exercice 3 — Dijkstra avec chemin

Modifiez l'algorithme de Dijkstra pour qu'il retourne non seulement les distances, mais aussi le chemin le plus court vers chaque sommet.

Voir la solution 
import heapq

def dijkstra_chemin(graphe, source):
    distances = {s: float('inf') for s in graphe}
    predecesseurs = {s: None for s in graphe}
    distances[source] = 0
    file = [(0, source)]
    while file:
        dist, u = heapq.heappop(file)
        if dist > distances[u]:
            continue
        for v, poids in graphe[u].items():
            d = dist + poids
            if d < distances[v]:
                distances[v] = d
                predecesseurs[v] = u
                heapq.heappush(file, (d, v))
    return distances, predecesseurs

def reconstruire_chemin(predecesseurs, destination):
    chemin = []
    s = destination
    while s is not None:
        chemin.append(s)
        s = predecesseurs[s]
    return chemin[::-1]
