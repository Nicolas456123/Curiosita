Introduction

Quand les machines comprennent le langage humain

Le traitement du langage naturel (Natural Language Processing, NLP) est la branche de l'IA qui permet aux machines de comprendre, interpréter et générer du langage humain. Depuis la traduction automatique jusqu'aux assistants conversationnels, en passant par l'analyse de sentiments et le résumé automatique, le NLP est devenu omniprésent dans notre quotidien.

Le langage humain est extraordinairement complexe : polysémie (un mot peut avoir plusieurs sens), ambiguïté syntaxique, implicites culturels, ironie, métaphores. Pendant des décennies, les approches basées sur des règles linguistiques ont dominé. Aujourd'hui, les modèles neuronaux, et en particulier les Transformers, ont révolutionné le domaine en apprenant directement à partir de milliards de textes.

Révolution des LLM : les grands modèles de langue (Large Language Models) comme GPT, BERT ou LLaMA ont démontré des capacités émergentes impressionnantes : raisonnement, traduction, génération de code, résumé — le tout à partir d'un simple objectif d'entraînement : prédire le mot suivant.

Les concepts fondamentaux du NLP

Tokenisation

La tokenisation est la première étape de tout pipeline NLP : transformer un texte brut en une séquence de tokens (unités élémentaires). La tokenisation la plus simple découpe par mots, mais les approches modernes utilisent des sous-mots (subword tokenization) comme BPE (Byte Pair Encoding) ou WordPiece.

L'avantage des sous-mots est de gérer un vocabulaire de taille fixe tout en pouvant représenter n'importe quel mot. Les mots fréquents sont un seul token (« le », « est »), tandis que les mots rares sont découpés en morceaux (« anticonstitutionnellement » → « anti », « constitu », « tionnel », « lement »). Chaque token reçoit un identifiant numérique unique dans le vocabulaire.

Python
from transformers import AutoTokenizer

# Tokenizer de GPT-2 (BPE)
tokenizer = AutoTokenizer.from_pretrained("gpt2")

texte = "Le traitement du langage naturel est fascinant."
tokens = tokenizer.tokenize(texte)
ids = tokenizer.encode(texte)

print("Tokens :", tokens)
# ['Le', ' traitement', ' du', ' langage', ' naturel', ' est', ' fascinant', '.']
print("IDs    :", ids)
# [3123, 9147, 2566, 26210, 14288, 1556, 37811, 13]
print(f"Vocabulaire : {tokenizer.vocab_size} tokens")
# Vocabulaire : 50257 tokens

Embeddings de mots

Les embeddings (plongements lexicaux) représentent chaque mot comme un vecteur dense dans un espace de haute dimension (typiquement 100 à 768 dimensions). Contrairement au one-hot encoding (vecteurs creux de taille V), les embeddings capturent les relations sémantiques : les mots proches en sens sont proches dans l'espace vectoriel.

Word2Vec (2013) a popularisé cette idée avec deux architectures : CBOW (prédire un mot à partir de son contexte) et Skip-gram (prédire le contexte à partir d'un mot). La célèbre relation « roi - homme + femme ≈ reine » illustre les propriétés algébriques de ces vecteurs. GloVe et FastText sont des alternatives qui exploitent les statistiques globales de co-occurrence.

Python
import gensim.downloader as api

# Charger des embeddings pré-entraînés
modele = api.load("word2vec-relnews-300")

# Similarité sémantique
print(modele.similarity("roi", "reine"))    # ~0.65
print(modele.similarity("chat", "voiture"))  # ~0.10

# Analogie : roi - homme + femme ≈ ?
resultat = modele.most_similar(positive=["roi", "femme"], negative=["homme"])
print(resultat[0])  # ('reine', 0.71)

Mécanisme d'attention et Transformers

Le mécanisme d'attention permet au modèle de pondérer différemment les mots d'une phrase selon le contexte. Par exemple, dans « La banque du fleuve était verdoyante », le mot « banque » doit être interprété en fonction de « fleuve » et « verdoyante ». L'auto-attention calcule pour chaque mot un score d'importance par rapport à tous les autres mots de la phrase.

L'architecture Transformer empile des blocs d'auto-attention multi-têtes et de réseaux feed-forward. Le positional encoding ajoute l'information de position (sinusoïdal ou appris), car l'attention seule est invariante à l'ordre des mots. Les Transformers traitent toute la séquence en parallèle, ce qui les rend beaucoup plus rapides à entraîner que les RNN.

Encoder vs Decoder : BERT utilise uniquement l'encodeur (bidirectionnel, optimal pour la compréhension). GPT utilise uniquement le décodeur (auto-régressif, optimal pour la génération). T5 utilise les deux (encoder-decoder, polyvalent pour la traduction et le résumé).

Modèles de langue pré-entraînés

BERT (Bidirectional Encoder Representations from Transformers) est pré-entraîné sur deux tâches : le Masked Language Modeling (prédire un mot masqué dans une phrase) et la Next Sentence Prediction. Son approche bidirectionnelle lui permet de comprendre le contexte gauche et droit simultanément, ce qui en fait un excellent modèle pour la classification, l'extraction d'entités et la réponse aux questions.

GPT (Generative Pre-trained Transformer) est pré-entraîné comme modèle auto-régressif : il prédit le prochain token à partir de tous les tokens précédents. Cette simplicité d'objectif, combinée à une échelle massive (175 milliards de paramètres pour GPT-3), permet des capacités émergentes remarquables. Le fine-tuning avec RLHF (Reinforcement Learning from Human Feedback) aligne ensuite le modèle avec les préférences humaines.

Python — Hugging Face
from transformers import pipeline

# Classification de sentiments avec un modèle pré-entraîné
classifieur = pipeline("sentiment-analysis", model="nlptown/bert-base-multilingual-uncased-sentiment")
resultat = classifieur("Ce restaurant est excellent, j'y retournerai !")
print(resultat)  # [{'label': '5 stars', 'score': 0.73}]

# Génération de texte avec GPT-2
generateur = pipeline("text-generation", model="gpt2")
texte = generateur("L'intelligence artificielle va", max_length=50)
print(texte[0]['generated_text'])

Tâches fondamentales du NLP

Le NLP englobe de nombreuses tâches. La classification de texte attribue une catégorie à un document (spam, sentiment, thème). La reconnaissance d'entités nommées (NER) identifie les noms de personnes, lieux, organisations dans un texte. L'analyse syntaxique détermine la structure grammaticale d'une phrase (sujet, verbe, complément).

La traduction automatique convertit un texte d'une langue à une autre. Le résumé automatique condense un long texte en quelques phrases clés. La réponse aux questions (question answering) extrait une réponse précise à partir d'un contexte donné. Toutes ces tâches bénéficient aujourd'hui du fine-tuning de modèles pré-entraînés.

Python — NER et QA
from transformers import pipeline

# Reconnaissance d'entités nommées
ner = pipeline("ner", grouped_entities=True)
entites = ner("Marie Curie a travaillé à Paris avec Pierre Curie.")
for e in entites:
    print(f"  {e['word']:20s} → {e['entity_group']}")
# Marie Curie          → PER
# Paris                → LOC
# Pierre Curie         → PER

# Réponse aux questions
qa = pipeline("question-answering")
reponse = qa(question="Où travaillait Marie Curie ?",
             context="Marie Curie a travaillé à Paris au laboratoire de physique.")
print(reponse['answer'])  # "Paris"

Chatbots et agents conversationnels

Les chatbots modernes reposent sur des LLM fine-tunés pour le dialogue. L'entraînement se fait en plusieurs étapes : (1) pré-entraînement sur un large corpus, (2) fine-tuning supervisé sur des exemples de conversations de qualité, (3) alignement par RLHF pour que le modèle soit utile, honnête et inoffensif.

Le prompting est l'art de formuler des instructions pour guider le comportement du modèle. Le few-shot prompting fournit quelques exemples dans le prompt. Le chain-of-thought demande au modèle de raisonner étape par étape. Le RAG (Retrieval-Augmented Generation) enrichit le prompt avec des documents pertinents récupérés dans une base de connaissances, réduisant les hallucinations.

Embeddings contextuels et recherche sémantique

Contrairement à Word2Vec qui attribue un seul vecteur par mot, les modèles comme BERT produisent des embeddings contextuels : le même mot reçoit des vecteurs différents selon le contexte. « La banque du fleuve » et « la banque d'investissement » donnent des représentations différentes pour « banque ».

Les sentence embeddings représentent des phrases entières comme des vecteurs. Des modèles comme Sentence-BERT permettent la recherche sémantique : retrouver des documents par leur sens plutôt que par des mots-clés exacts. C'est la base des systèmes de RAG et des moteurs de recherche intelligents.

Exercice 1 — Tokenisation comparée

Comparez la tokenisation de la phrase « L'anticonstitutionnellement est un adverbe extraordinairement long » avec trois tokenizers différents : GPT-2 (BPE), BERT (WordPiece), et une tokenisation par mots simples (split sur les espaces). Comptez le nombre de tokens produits par chacun.

from transformers import AutoTokenizer

phrase = "L'anticonstitutionnellement est un adverbe extraordinairement long"

Exercice 3 — Classification de sentiments avec BERT

Fine-tunez un modèle BERT multilingue pour classifier des avis de films en français (positif/négatif). Utilisez la bibliothèque Hugging Face Transformers et un petit dataset de 100 avis. Évaluez la précision avec la validation croisée.

Voir la solution 
from transformers import AutoTokenizer, AutoModelForSequenceClassification
from transformers import Trainer, TrainingArguments
import torch
from torch.utils.data import Dataset

class AvisDataset(Dataset):
    def __init__(self, textes, labels, tokenizer, max_len=128):
        self.encodings = tokenizer(textes, truncation=True,
                                    padding=True, max_length=max_len)
        self.labels = labels

    def __len__(self): return len(self.labels)
    def __getitem__(self, idx):
        item = {k: torch.tensor(v[idx]) for k, v in self.encodings.items()}
        item['labels'] = torch.tensor(self.labels[idx])
        return item

# Données d'exemple
textes = ["Film excellent, à voir absolument !", "Très mauvais, une perte de temps",
          "Acteurs formidables et scénario captivant", "Ennuyeux et prévisible", ...]
labels = [1, 0, 1, 0, ...]  # 1=positif, 0=négatif

tokenizer = AutoTokenizer.from_pretrained("bert-base-multilingual-cased")
model = AutoModelForSequenceClassification.from_pretrained(
    "bert-base-multilingual-cased", num_labels=2)

dataset = AvisDataset(textes, labels, tokenizer)

args = TrainingArguments(output_dir="./results", num_train_epochs=3,
                          per_device_train_batch_size=16, learning_rate=2e-5)
trainer = Trainer(model=model, args=args, train_dataset=dataset)
trainer.train()
# Avec seulement 100 exemples, BERT atteint ~85-90% de précision
# grâce au transfer learning

Exercice 4 — Système de question-réponse

Construisez un système simple de question-réponse en utilisant le pipeline Hugging Face. Prenez un paragraphe de Wikipedia en français comme contexte et posez 5 questions différentes. Analysez la qualité des réponses et le score de confiance.

from transformers import pipeline

qa = pipeline("question-answering",
              model="etalab-ia/camembert-base-squadFR-fquad-piaf")

contexte = """
La Tour Eiffel est une tour de fer puddlé de 330 mètres de hauteur
située à Paris. Construite en deux ans par Gustave Eiffel et ses

    "Quelle est la hauteur de la Tour Eiffel ?",

Exercice 5 — Recherche sémantique avec Sentence-BERT

Implémentez un moteur de recherche sémantique simple. Créez une base de 10 phrases en français, encodez-les avec Sentence-BERT, puis permettez à l'utilisateur de poser une question et retrouvez les 3 phrases les plus pertinentes par similarité cosinus.

Voir la solution 
from sentence_transformers import SentenceTransformer
import numpy as np

modele = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")

# Base de connaissances
documents = [
    "Python est un langage de programmation interprété.",
    "Le deep learning utilise des réseaux de neurones profonds.",
    "La photosynthèse convertit la lumière en énergie chimique.",
    "Les algorithmes de tri organisent les données.",
    "BERT est un modèle de langage bidirectionnel.",
    "La Terre orbite autour du Soleil en 365 jours.",
    "Les bases de données SQL utilisent des tables relationnelles.",
    "L'ADN contient l'information génétique des êtres vivants.",
    "Les Transformers utilisent le mécanisme d'attention.",
    "La gravité est une force fondamentale de la physique.",
]

# Encoder les documents
embeddings_docs = modele.encode(documents)

def rechercher(question, top_k=3):
    emb_question = modele.encode([question])
    # Similarité cosinus
    scores = np.dot(embeddings_docs, emb_question.T).flatten()
    scores /= (np.linalg.norm(embeddings_docs, axis=1) *
                np.linalg.norm(emb_question))
    indices = np.argsort(scores)[::-1][:top_k]
    for i in indices:
        print(f"  [{scores[i]:.3f}] {documents[i]}")

rechercher("Comment fonctionne l'intelligence artificielle ?")
