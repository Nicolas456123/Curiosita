Introduction

Quand les machines apprennent à voir

La vision par ordinateur (Computer Vision) est le domaine de l'IA qui permet aux machines d'interpréter et de comprendre le contenu visuel du monde : images, vidéos, flux de caméras en temps réel. C'est l'un des domaines où le deep learning a eu l'impact le plus spectaculaire, transformant radicalement les performances en quelques années.

Les applications sont innombrables : voitures autonomes qui détectent les piétons et les panneaux, imagerie médicale qui identifie des tumeurs sur des radiographies, contrôle qualité automatisé dans les usines, filtres de réalité augmentée sur les smartphones, ou encore vidéosurveillance intelligente. La vision par ordinateur est au coeur de la transformation numérique de nombreuses industries.

Le tournant ImageNet : en 2012, le réseau AlexNet a réduit le taux d'erreur du concours ImageNet de 26% à 16%, une amélioration sans précédent. Depuis, les CNN puis les Vision Transformers ont ramené ce taux sous les 2%, surpassant les performances humaines (~5%).

Les techniques fondamentales de la vision par ordinateur

Classification d'images

La classification d'images est la tâche la plus fondamentale : attribuer une catégorie à une image entière (chat, chien, voiture, avion...). Le modèle reçoit une image en entrée (matrice de pixels) et produit un vecteur de probabilités sur les classes possibles. Les CNN (réseaux convolutifs) sont l'architecture dominante pour cette tâche.

Les architectures classiques incluent VGG (couches 3x3 empilées), ResNet (connexions résiduelles), Inception (filtres de tailles multiples en parallèle), et EfficientNet (scaling optimisé). Plus récemment, les Vision Transformers (ViT) découpent l'image en patches et les traitent comme une séquence de tokens, atteignant des performances supérieures avec suffisamment de données.

Python — PyTorch
import torch
from torchvision import models, transforms
from PIL import Image

# Charger un modèle pré-entraîné
modele = models.resnet50(pretrained=True)
modele.eval()

# Prétraitement standard pour ImageNet
preprocess = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406],
                         [0.229, 0.224, 0.225])
])

image = Image.open("photo_chat.jpg")
tensor = preprocess(image).unsqueeze(0)

with torch.no_grad():
    probs = torch.softmax(modele(tensor), dim=1)
    top5 = torch.topk(probs, 5)
    print("Top-5 prédictions :", top5)

Détection d'objets

La détection d'objets va plus loin que la classification : elle identifie chaque objet dans une image et localise sa position avec une boîte englobante (bounding box). Chaque détection comprend la classe de l'objet, ses coordonnées (x, y, largeur, hauteur), et un score de confiance.

Les architectures de détection se divisent en deux familles. Les détecteurs two-stage comme Faster R-CNN proposent d'abord des régions d'intérêt, puis les classifient. Les détecteurs one-stage comme YOLO (You Only Look Once) et SSD prédisent directement les boîtes et les classes en un seul passage. YOLO est particulièrement populaire pour la détection en temps réel grâce à sa vitesse (30-60 FPS).

Python — YOLOv8
from ultralytics import YOLO

# Charger le modèle YOLOv8 pré-entraîné
modele = YOLO("yolov8n.pt")  # version nano (rapide)

# Détection sur une image
resultats = modele("rue_ville.jpg")

# Afficher les détections
for r in resultats:
    for box in r.boxes:
        classe = modele.names[int(box.cls)]
        confiance = float(box.conf)
        coords = box.xyxy[0].tolist()  # [x1, y1, x2, y2]
        print(f"{classe}: {confiance:.0%} à {coords}")

# Résultat : personne: 95% à [120, 50, 280, 400]
#            voiture: 88% à [300, 150, 550, 320]

Segmentation d'images

La segmentation sémantique attribue une classe à chaque pixel de l'image. Contrairement à la détection qui donne des boîtes rectangulaires, la segmentation délimite la forme exacte de chaque objet. C'est essentiel pour les voitures autonomes (distinguer route, trottoir, piétons pixel par pixel) et l'imagerie médicale (segmenter tumeurs, organes).

L'architecture U-Net est la référence en segmentation. Elle a une structure encoder-decoder avec des skip connections qui transmettent les détails fins de l'encodeur au décodeur. Le Feature Pyramid Network (FPN) fonctionne sur un principe similaire en fusionnant des features à différentes échelles. La segmentation d'instances (Mask R-CNN) distingue en plus les différentes instances d'une même classe.

Types de segmentation :
Sémantique : une classe par pixel (route, ciel, arbre) — ne distingue pas les instances
D'instances : distingue chaque objet individuellement (personne 1, personne 2)
Panoptique : combine les deux — segmentation complète de la scène

Reconnaissance faciale

La reconnaissance faciale est un problème en plusieurs étapes : (1) détection de visages dans l'image, (2) alignement pour normaliser la position et la taille, (3) extraction d'un embedding facial (vecteur de 128-512 dimensions représentant l'identité), (4) comparaison avec une base de données pour identifier la personne.

Les architectures modernes comme FaceNet utilisent la triplet loss : entraîner le réseau pour que les embeddings de la même personne soient proches, et ceux de personnes différentes soient éloignés. ArcFace améliore cela avec une fonction de coût angulaire qui augmente la marge entre les classes dans l'espace des embeddings. Ces systèmes atteignent des taux de reconnaissance supérieurs à 99% sur les benchmarks.

Python — face_recognition
import face_recognition
import numpy as np

# Charger et encoder un visage de référence
img_ref = face_recognition.load_image_file("alice.jpg")
encoding_ref = face_recognition.face_encodings(img_ref)[0]

# Comparer avec un nouveau visage
img_test = face_recognition.load_image_file("inconnu.jpg")
encodings_test = face_recognition.face_encodings(img_test)

for enc in encodings_test:
    distance = np.linalg.norm(encoding_ref - enc)
    match = distance < 0.6  # Seuil de similarité
    print(f"Distance : {distance:.3f} → {'Match !' if match else 'Différent'}")

# Distance : 0.35 → Match ! (même personne)
# L'embedding est un vecteur de 128 dimensions

Augmentation de données

L'augmentation de données (data augmentation) est une technique essentielle pour améliorer la généralisation des modèles de vision. Elle consiste à appliquer des transformations aléatoires aux images d'entraînement pour créer artificiellement de la diversité : rotations, retournements horizontaux, recadrages aléatoires, variations de luminosité et de contraste, ajout de bruit.

Les techniques avancées incluent CutMix (combiner des morceaux de deux images), MixUp (mélanger deux images et leurs labels), et RandAugment (appliquer automatiquement une combinaison optimale de transformations). L'augmentation est appliquée à la volée pendant l'entraînement, sans stocker les images transformées.

Modèles génératifs pour les images

Les modèles génératifs créent de nouvelles images à partir de bruit aléatoire ou de descriptions textuelles. Les GAN (Generative Adversarial Networks) opposent un générateur (qui crée des images) à un discriminateur (qui distingue le vrai du faux) dans un jeu adversarial. StyleGAN peut générer des visages photoréalistes de personnes qui n'existent pas.

Les modèles de diffusion (Stable Diffusion, DALL-E, Midjourney) ont largement supplanté les GAN pour la génération d'images. Ils fonctionnent en deux phases : bruiter progressivement une image pendant l'entraînement, puis apprendre à la débruiter. À l'inférence, on part de bruit pur et on débruite étape par étape pour obtenir une image cohérente, éventuellement guidée par un prompt textuel.

Vision par ordinateur en temps réel

La vision en temps réel impose des contraintes de latence strictes : la voiture autonome doit détecter un piéton en moins de 50 ms. Cela nécessite des architectures optimisées comme MobileNet (convolutions dépthwise separable), EfficientNet-Lite, ou des versions compactes de YOLO. La quantification réduit la précision des poids de 32 bits à 8 bits, accélérant l'inférence de 2 à 4 fois.

Le pruning (élagage) supprime les connexions de faible importance, réduisant la taille du modèle. La distillation de connaissances entraîne un petit modèle « étudiant » à imiter les sorties d'un grand modèle « professeur ». Ces techniques permettent de déployer des modèles performants sur des appareils mobiles, des caméras embarquées ou des drones.

from collections import Counter

model = YOLO("yolov8n.pt")

Exercice 3 — Segmentation avec U-Net simplifié

Implémentez une version simplifiée de U-Net avec PyTorch pour la segmentation binaire (objet/fond). Utilisez un encodeur à 3 niveaux et un décodeur symétrique avec skip connections. Entraînez sur un petit dataset de masques.

Voir la solution 
import torch
import torch.nn as nn

class UNetSimple(nn.Module):
    def __init__(self):
        super().__init__()
        # Encodeur
        self.enc1 = self._block(3, 64)
        self.enc2 = self._block(64, 128)
        self.enc3 = self._block(128, 256)
        self.pool = nn.MaxPool2d(2)

        # Bottleneck
        self.bottleneck = self._block(256, 512)

        # Décodeur
        self.up3 = nn.ConvTranspose2d(512, 256, 2, stride=2)
        self.dec3 = self._block(512, 256)  # 256+256 skip
        self.up2 = nn.ConvTranspose2d(256, 128, 2, stride=2)
        self.dec2 = self._block(256, 128)
        self.up1 = nn.ConvTranspose2d(128, 64, 2, stride=2)
        self.dec1 = self._block(128, 64)

        self.final = nn.Conv2d(64, 1, 1)  # Segmentation binaire

    def _block(self, in_c, out_c):
        return nn.Sequential(
            nn.Conv2d(in_c, out_c, 3, padding=1), nn.ReLU(),
            nn.Conv2d(out_c, out_c, 3, padding=1), nn.ReLU())

    def forward(self, x):
        e1 = self.enc1(x)
        e2 = self.enc2(self.pool(e1))
        e3 = self.enc3(self.pool(e2))
        b = self.bottleneck(self.pool(e3))
        d3 = self.dec3(torch.cat([self.up3(b), e3], dim=1))
        d2 = self.dec2(torch.cat([self.up2(d3), e2], dim=1))
        d1 = self.dec1(torch.cat([self.up1(d2), e1], dim=1))
        return torch.sigmoid(self.final(d1))

model = UNetSimple()

Exercice 5 — Grad-CAM : visualiser l'attention du modèle

Implémentez Grad-CAM pour visualiser quelles régions de l'image influencent le plus la décision d'un CNN. Utilisez un ResNet pré-entraîné, classifiez une image, puis générez la carte de chaleur (heatmap) montrant les zones les plus importantes pour la prédiction.

Voir la solution 
import torch
import torch.nn.functional as F
from torchvision import models, transforms
from PIL import Image
import matplotlib.pyplot as plt
import numpy as np

model = models.resnet50(pretrained=True)
model.eval()

# Hook pour capturer les feature maps et les gradients
features = {}
def hook_features(module, input, output):
    features['maps'] = output
def hook_gradients(module, grad_in, grad_out):
    features['gradients'] = grad_out[0]

model.layer4.register_forward_hook(hook_features)
model.layer4.register_full_backward_hook(hook_gradients)

# Charger et prétraiter l'image
preprocess = transforms.Compose([
    transforms.Resize(256), transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225])])

image = Image.open("chat.jpg")
tensor = preprocess(image).unsqueeze(0).requires_grad_(True)

# Forward + backward
output = model(tensor)
classe_pred = output.argmax(1).item()
output[0, classe_pred].backward()

# Calculer Grad-CAM
gradients = features['gradients'].mean(dim=[2, 3], keepdim=True)
cam = F.relu((features['maps'] * gradients).sum(dim=1, keepdim=True))
cam = F.interpolate(cam, size=(224, 224), mode='bilinear')
cam = cam.squeeze().detach().numpy()
cam = (cam - cam.min()) / (cam.max() - cam.min())

# Afficher
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(10, 5))
ax1.imshow(image.resize((224, 224))); ax1.set_title("Image originale")
ax2.imshow(image.resize((224, 224))); ax2.imshow(cam, alpha=0.5, cmap='jet')
ax2.set_title(f"Grad-CAM (classe: {classe_pred})")
plt.show()
