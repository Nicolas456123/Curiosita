Introduction

Qu'est-ce que l'apprentissage automatique ?

Le Machine Learning (apprentissage automatique) est une branche de l'intelligence artificielle qui permet aux machines d'apprendre à partir de données, sans être explicitement programmées pour chaque tâche. Au lieu d'écrire des règles à la main, on fournit des exemples au système, et l'algorithme découvre lui-même les patterns et relations qui permettent de faire des prédictions ou des décisions.

L'essor du Machine Learning est lié à trois facteurs : l'explosion de la quantité de données disponibles, l'augmentation de la puissance de calcul, et le développement d'algorithmes de plus en plus sophistiqués. Aujourd'hui, le ML est utilisé dans la reconnaissance vocale, les moteurs de recommandation, la détection de fraude, la médecine personnalisée et bien d'autres domaines.

Paradigme clé : en programmation traditionnelle, on fournit des règles et des données pour obtenir des résultats. En Machine Learning, on fournit des données et des résultats attendus, et l'algorithme découvre les règles.

On distingue trois grandes familles d'apprentissage : supervisé (on donne les réponses au modèle), non supervisé (le modèle découvre des structures cachées), et par renforcement (le modèle apprend par essai-erreur avec des récompenses).

Le cours

Les fondements du Machine Learning

Apprentissage supervisé

L'apprentissage supervisé est le paradigme le plus courant en ML. On dispose d'un jeu de données étiqueté : chaque exemple possède des features (caractéristiques) et un label (étiquette, la réponse attendue). Le modèle apprend une fonction qui associe les features au label, puis généralise à de nouveaux exemples jamais vus.

On distingue deux types de problèmes supervisés : la classification (prédire une catégorie : spam/non-spam, chat/chien) et la régression (prédire une valeur continue : prix d'un appartement, température demain).

Python
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

# Séparer données d'entraînement et de test
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# Entraîner un modèle de régression linéaire
modele = LinearRegression()
modele.fit(X_train, y_train)

# Évaluer sur les données de test
predictions = modele.predict(X_test)
erreur = mean_squared_error(y_test, predictions)
print(f"Erreur quadratique moyenne : {erreur:.2f}")

Apprentissage non supervisé

Dans l'apprentissage non supervisé, les données ne possèdent pas d'étiquettes. L'algorithme doit découvrir des structures cachées par lui-même. Les applications principales sont le clustering (regrouper des données similaires), la réduction de dimensionnalité (simplifier des données complexes) et la détection d'anomalies.

L'algorithme K-Means est l'un des plus populaires pour le clustering. Il partitionne les données en K groupes en minimisant la distance intra-cluster. L'ACP (Analyse en Composantes Principales) est la méthode de référence pour la réduction de dimensionnalité : elle projette les données sur les axes de plus grande variance.

Python
from sklearn.cluster import KMeans
from sklearn.decomposition import PCA

# Clustering K-Means avec 3 groupes
kmeans = KMeans(n_clusters=3, random_state=42)
clusters = kmeans.fit_predict(X)

# Réduction de dimensionnalité avec ACP
pca = PCA(n_components=2)
X_reduit = pca.fit_transform(X)
print(f"Variance expliquée : {pca.explained_variance_ratio_.sum():.1%}")

Régression linéaire et logistique

La régression linéaire modélise la relation entre des variables par une droite (ou un hyperplan). Elle minimise la somme des carrés des erreurs entre les prédictions et les valeurs réelles. C'est le modèle le plus simple et le plus interprétable.

La régression logistique, malgré son nom, est un algorithme de classification. Elle utilise la fonction sigmoïde pour transformer une combinaison linéaire des features en une probabilité entre 0 et 1. Le seuil de décision est généralement fixé à 0.5.

Fonction sigmoïde : σ(z) = 1 / (1 + e^(-z)). Elle transforme toute valeur réelle en une probabilité entre 0 et 1, ce qui en fait la base de la régression logistique.

Arbres de décision et forêts aléatoires

Un arbre de décision partitionne l'espace des features par une série de questions binaires (ex : « âge > 30 ? »). Chaque noeud interne pose une question, et chaque feuille contient une prédiction. L'arbre est construit en choisissant à chaque étape la question qui maximise le gain d'information (ou minimise l'impureté de Gini).

Les forêts aléatoires (Random Forests) combinent des centaines d'arbres de décision entraînés sur des sous-échantillons aléatoires des données. Chaque arbre vote, et la prédiction finale est le vote majoritaire. Cette approche réduit considérablement le surapprentissage.

Python
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score

# Forêt aléatoire avec 100 arbres
foret = RandomForestClassifier(n_estimators=100, random_state=42)
foret.fit(X_train, y_train)

# Prédiction et évaluation
y_pred = foret.predict(X_test)
print(f"Précision : {accuracy_score(y_test, y_pred):.1%}")

# Importance des features
for nom, imp in zip(noms_features, foret.feature_importances_):
    print(f"  {nom}: {imp:.3f}")

Validation croisée et surapprentissage

Le surapprentissage (overfitting) se produit quand un modèle apprend par coeur les données d'entraînement sans généraliser. Il obtient d'excellents scores sur les données d'entraînement mais échoue sur de nouvelles données. À l'inverse, le sous-apprentissage (underfitting) signifie que le modèle est trop simple pour capturer les patterns.

La validation croisée (cross-validation) est la technique standard pour évaluer un modèle de façon fiable. En K-Fold CV, on divise les données en K parties. Tour à tour, chaque partie sert de test et les K-1 autres d'entraînement. On obtient K scores, dont la moyenne donne une estimation robuste de la performance.

Python
from sklearn.model_selection import cross_val_score

# Validation croisée à 5 plis
scores = cross_val_score(modele, X, y, cv=5, scoring='accuracy')
print(f"Précision moyenne : {scores.mean():.1%} ± {scores.std():.1%}")

Métriques d'évaluation

Choisir la bonne métrique est crucial. Pour la classification, l'exactitude (accuracy) mesure le pourcentage de prédictions correctes, mais peut être trompeuse avec des classes déséquilibrées. La précision mesure la proportion de vrais positifs parmi les prédictions positives, le rappel mesure la proportion de vrais positifs détectés, et le F1-score est leur moyenne harmonique.

Pour la régression, on utilise l'erreur quadratique moyenne (MSE), l'erreur absolue moyenne (MAE), et le coefficient de détermination R² qui indique la proportion de variance expliquée par le modèle (1 = parfait, 0 = aussi mauvais que la moyenne).

Matrice de confusion : tableau qui croise les prédictions et les vrais labels. Elle permet de visualiser les vrais positifs, faux positifs, vrais négatifs et faux négatifs, et de calculer toutes les métriques de classification.

Prétraitement des données

Les données brutes sont rarement directement utilisables. Le prétraitement est une étape essentielle : gestion des valeurs manquantes (imputation par la moyenne, la médiane ou un modèle), encodage des variables catégorielles (one-hot encoding, label encoding), et mise à l'échelle des features (normalisation, standardisation).

La standardisation (z-score) centre les données autour de 0 avec un écart-type de 1. La normalisation (min-max) ramène les valeurs entre 0 et 1. Ces transformations sont importantes pour les algorithmes sensibles à l'échelle des features, comme les SVM ou les k-plus-proches-voisins.

Question 1 / 6

Quelle est la différence principale entre classification et régression ?

La classification prédit une classe discrète (spam/non-spam, chat/chien), tandis que la régression prédit une valeur numérique continue (prix, température).

Question 2 / 6

Qu'est-ce que le surapprentissage (overfitting) ?

Le surapprentissage signifie que le modèle a appris par coeur les données d'entraînement, y compris le bruit, et ne parvient pas à généraliser à de nouvelles données.

Question 3 / 6

Dans la validation croisée K-Fold, que fait-on ?

En K-Fold CV, les données sont divisées en K parties. Chaque partie sert de test une fois, tandis que les K-1 autres servent d'entraînement, ce qui donne K évaluations indépendantes.

Question 4 / 6

Quel algorithme est utilisé pour le clustering ?

K-Means est un algorithme de clustering non supervisé qui partitionne les données en K groupes en minimisant la distance intra-cluster.

Question 5 / 6

Pourquoi les forêts aléatoires sont-elles plus robustes qu'un seul arbre de décision ?

Les forêts aléatoires agrègent les prédictions de centaines d'arbres entraînés sur des sous-échantillons différents. Cette diversité réduit la variance et le surapprentissage.

Question 6 / 6

Quelle métrique est la plus adaptée pour un problème de classification avec des classes très déséquilibrées ?

Avec des classes déséquilibrées, l'accuracy peut être trompeuse (99% si 99% des données sont d'une classe). Le F1-score, moyenne harmonique de la précision et du rappel, est plus informatif.

Exercice 1 — Régression linéaire manuelle

Implémentez une régression linéaire simple (une seule feature) sans utiliser scikit-learn. Calculez les coefficients a et b de la droite y = ax + b en utilisant les formules des moindres carrés. Testez sur les données : X = [1, 2, 3, 4, 5], y = [2.1, 4.0, 5.8, 8.1, 9.9].

Voir la solution 
import numpy as np

X = np.array([1, 2, 3, 4, 5])
y = np.array([2.1, 4.0, 5.8, 8.1, 9.9])

# Formules des moindres carrés
n = len(X)
a = (n * np.sum(X * y) - np.sum(X) * np.sum(y)) / (n * np.sum(X**2) - np.sum(X)**2)
b = np.mean(y) - a * np.mean(X)

print(f"y = {a:.2f}x + {b:.2f}")
# Résultat : y ≈ 1.97x + 0.02
# Prédiction pour x=6 : y ≈ 11.84

Exercice 2 — Classification avec K-NN

Utilisez l'algorithme des K plus proches voisins pour classifier le dataset Iris de scikit-learn. Testez avec différentes valeurs de K (1, 3, 5, 7, 11) et utilisez la validation croisée pour trouver la meilleure valeur.

Voir la solution 
from sklearn.datasets import load_iris
from sklearn.neighbors import KNeighborsClassifier
from sklearn.model_selection import cross_val_score

iris = load_iris()
X, y = iris.data, iris.target

for k in [1, 3, 5, 7, 11]:
    knn = KNeighborsClassifier(n_neighbors=k)
    scores = cross_val_score(knn, X, y, cv=5)

# K=1  → ~95.3%, K=3  → ~96.0%, K=5  → ~96.7%
# K=7  → ~96.7%, K=11 → ~97.3%
# K=5 ou K=7 offrent un bon compromis

Exercice 3 — Matrice de confusion

Entraînez un arbre de décision sur le dataset Iris, puis affichez la matrice de confusion et calculez manuellement la précision, le rappel et le F1-score pour chaque classe.

from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import confusion_matrix, classification_report

X_train, X_test, y_train, y_test = train_test_split(

Exercice 4 — Détection du surapprentissage

Générez un dataset synthétique avec make_classification. Entraînez un arbre de décision en faisant varier max_depth de 1 à 20. Tracez les courbes de score sur l'entraînement et le test. Identifiez le point de surapprentissage.

Voir la solution 
from sklearn.datasets import make_classification
import matplotlib.pyplot as plt

X, y = make_classification(n_samples=500, n_features=20, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3)

train_scores, test_scores = [], []
depths = range(1, 21)

for d in depths:
    arbre = DecisionTreeClassifier(max_depth=d, random_state=42)
    arbre.fit(X_train, y_train)
    train_scores.append(arbre.score(X_train, y_train))
    test_scores.append(arbre.score(X_test, y_test))

plt.plot(depths, train_scores, label="Entraînement")
plt.plot(depths, test_scores, label="Test")
plt.xlabel("Profondeur maximale")
plt.ylabel("Précision")
plt.legend()
plt.title("Détection du surapprentissage")
plt.show()
# Le surapprentissage apparaît quand le score d'entraînement
# continue de monter mais le score de test stagne ou diminue

Exercice 5 — Pipeline complet

Construisez un pipeline complet avec scikit-learn qui enchaîne : imputation des valeurs manquantes, standardisation des features, et un classificateur Random Forest. Utilisez la validation croisée pour évaluer le pipeline.

Voir la solution 
from sklearn.pipeline import Pipeline
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score
import numpy as np

# Simuler des données manquantes
X_manquant = X.copy()
indices = np.random.choice(X.size, size=50, replace=False)
X_manquant.flat[indices] = np.nan

# Pipeline complet
pipeline = Pipeline([
    ('imputation', SimpleImputer(strategy='median')),
    ('standardisation', StandardScaler()),
    ('classificateur', RandomForestClassifier(n_estimators=100, random_state=42))
])

scores = cross_val_score(pipeline, X_manquant, y, cv=5, scoring='accuracy')
print(f"Précision : {scores.mean():.1%} ± {scores.std():.1%}")
# Le pipeline gère automatiquement les valeurs manquantes
# et la mise à l'échelle à chaque pli de la validation croisée
