Introduction

Des neurones biologiques aux neurones artificiels

Les réseaux de neurones artificiels sont inspirés du fonctionnement du cerveau humain. Un neurone biologique reçoit des signaux électriques par ses dendrites, les traite dans son corps cellulaire, et transmet un signal par son axone si le stimulus dépasse un certain seuil. De la même façon, un neurone artificiel reçoit des entrées numériques, les pondère, les somme, et applique une fonction d'activation pour produire une sortie.

Le premier modèle de neurone artificiel, le perceptron, a été proposé par Frank Rosenblatt en 1958. Bien que limité (il ne peut résoudre que des problèmes linéairement séparables), il a posé les bases de tout le domaine. C'est en empilant des couches de neurones et en développant l'algorithme de rétropropagation que les réseaux de neurones sont devenus capables de résoudre des problèmes complexes.

Approximation universelle : un réseau de neurones avec une seule couche cachée contenant suffisamment de neurones peut approximer n'importe quelle fonction continue. C'est le théorème d'approximation universelle de Cybenko (1989).

Le cours

Architecture et fonctionnement des réseaux de neurones

Le perceptron

Le perceptron est le neurone artificiel le plus simple. Il prend un vecteur d'entrées x = (x₁, x₂, ..., xₙ), calcule la somme pondérée z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b (où w sont les poids et b le biais), puis applique une fonction d'activation pour produire la sortie. Dans le perceptron classique, la fonction d'activation est une fonction seuil : si z > 0, la sortie est 1, sinon 0.

L'algorithme d'apprentissage du perceptron est itératif : pour chaque exemple mal classé, on ajuste les poids dans la direction qui corrige l'erreur. Si les données sont linéairement séparables, le perceptron converge en un nombre fini d'itérations.

Python
import numpy as np

class Perceptron:
    def __init__(self, n_features, lr=0.01):
        self.poids = np.zeros(n_features)
        self.biais = 0
        self.lr = lr

    def predire(self, x):
        z = np.dot(self.poids, x) + self.biais
        return 1 if z > 0 else 0

    def entrainer(self, X, y, epochs=100):
        for _ in range(epochs):
            for xi, yi in zip(X, y):
                pred = self.predire(xi)
                erreur = yi - pred
                self.poids += self.lr * erreur * xi
                self.biais += self.lr * erreur

Fonctions d'activation

Les fonctions d'activation introduisent la non-linéarité indispensable pour que le réseau puisse apprendre des relations complexes. Sans elles, un réseau multicouche serait équivalent à une simple transformation linéaire, quelle que soit sa profondeur.

La sigmoïde σ(z) = 1/(1+e⁻ᶻ) produit des valeurs entre 0 et 1, mais souffre du problème de gradient évanescent pour les grandes valeurs de z. La tanh est similaire mais centrée en 0 (valeurs entre -1 et 1). La ReLU (Rectified Linear Unit) f(z) = max(0, z) est devenue la plus populaire grâce à sa simplicité et son efficacité : elle ne sature pas pour les valeurs positives et accélère la convergence.

Comparaison des fonctions d'activation :
Sigmoïde : σ(z) = 1/(1+e⁻ᶻ) — sortie [0,1], gradient évanescent
Tanh : tanh(z) — sortie [-1,1], centrée, gradient évanescent
ReLU : max(0,z) — rapide, pas de saturation, neurones « morts »
Leaky ReLU : max(0.01z, z) — corrige le problème des neurones morts
Softmax : eᶻⁱ/Σeᶻʲ — couche de sortie pour classification multi-classes

Architecture multicouche (MLP)

Un perceptron multicouche (Multi-Layer Perceptron, MLP) est constitué de plusieurs couches de neurones : une couche d'entrée, une ou plusieurs couches cachées, et une couche de sortie. Chaque neurone d'une couche est connecté à tous les neurones de la couche suivante (réseau « fully connected »).

La propagation avant (forward pass) calcule la sortie du réseau en traversant les couches de gauche à droite. Chaque couche effectue une transformation affine (multiplication matricielle + biais) suivie d'une fonction d'activation. La sortie de la dernière couche est la prédiction du réseau.

Python
import numpy as np

def relu(z):
    return np.maximum(0, z)

def softmax(z):
    exp_z = np.exp(z - np.max(z))
    return exp_z / exp_z.sum()

# Propagation avant dans un MLP à 2 couches cachées
def forward(x, W1, b1, W2, b2, W3, b3):
    h1 = relu(W1 @ x + b1)       # Couche cachée 1
    h2 = relu(W2 @ h1 + b2)      # Couche cachée 2
    sortie = softmax(W3 @ h2 + b3) # Couche de sortie
    return sortie

Rétropropagation du gradient

La rétropropagation (backpropagation) est l'algorithme fondamental pour entraîner les réseaux de neurones. Elle calcule le gradient de la fonction de coût par rapport à chaque poids du réseau, en appliquant la règle de la chaîne de façon systématique, couche par couche, de la sortie vers l'entrée.

Le processus se décompose en quatre étapes : (1) propagation avant pour calculer la sortie, (2) calcul de l'erreur entre la sortie et la cible, (3) rétropropagation de l'erreur pour calculer les gradients, (4) mise à jour des poids proportionnellement au gradient (descente de gradient).

Python
# Principe de la rétropropagation (réseau simple)
def relu_deriv(z):
    return (z > 0).astype(float)

# Étape 1 : Forward pass
z1 = W1 @ x + b1; h1 = relu(z1)
z2 = W2 @ h1 + b2; sortie = softmax(z2)

# Étape 2 : Calcul de l'erreur (cross-entropy)
perte = -np.log(sortie[classe_cible])

# Étape 3 : Backward pass (gradients)
d_sortie = sortie.copy()
d_sortie[classe_cible] -= 1            # ∂L/∂z2
dW2 = np.outer(d_sortie, h1)            # ∂L/∂W2
d_h1 = W2.T @ d_sortie * relu_deriv(z1) # ∂L/∂z1
dW1 = np.outer(d_h1, x)                 # ∂L/∂W1

# Étape 4 : Mise à jour des poids
W1 -= lr * dW1
W2 -= lr * dW2

Descente de gradient et optimiseurs

La descente de gradient stochastique (SGD) met à jour les poids après chaque exemple (ou mini-batch). Elle est bruitée mais rapide. Le momentum ajoute une inertie qui accélère la convergence et aide à franchir les minima locaux peu profonds.

L'optimiseur Adam (Adaptive Moment Estimation) combine le momentum avec un taux d'apprentissage adaptatif pour chaque paramètre. Il maintient des moyennes mobiles du gradient et du gradient au carré, ce qui le rend robuste et efficace dans la plupart des situations. C'est l'optimiseur par défaut recommandé pour commencer.

Taux d'apprentissage (learning rate) : c'est l'hyperparamètre le plus important. Trop grand, le modèle diverge. Trop petit, l'entraînement est extrêmement lent. On utilise souvent un scheduler qui réduit le taux d'apprentissage au cours de l'entraînement.

Régularisation

Pour éviter le surapprentissage, plusieurs techniques de régularisation sont employées. La régularisation L2 (weight decay) ajoute une pénalité proportionnelle à la norme des poids dans la fonction de coût, ce qui empêche les poids de devenir trop grands.

Le dropout désactive aléatoirement une fraction des neurones à chaque étape d'entraînement. Cela force le réseau à ne pas dépendre d'un seul chemin et à développer des représentations plus robustes. Un taux de dropout de 20% à 50% est courant dans les couches cachées. La batch normalization normalise les activations de chaque couche, ce qui stabilise et accélère l'entraînement.

Fonctions de coût

La fonction de coût (loss function) mesure l'écart entre les prédictions du réseau et les valeurs attendues. Pour la régression, on utilise l'erreur quadratique moyenne (MSE). Pour la classification binaire, l'entropie croisée binaire (binary cross-entropy). Pour la classification multi-classes, l'entropie croisée catégorielle combinée avec softmax.

Le choix de la fonction de coût influence directement la forme du paysage de perte et donc la facilité d'optimisation. L'entropie croisée est préférée à la MSE pour la classification car elle pénalise plus fortement les prédictions très éloignées de la cible.

Question 1 / 6

Pourquoi les fonctions d'activation sont-elles nécessaires dans un réseau de neurones ?

Sans fonctions d'activation non linéaires, un réseau multicouche serait équivalent à une simple transformation linéaire, incapable de modéliser des relations complexes.

Question 2 / 6

Quelle est la principale limitation du perceptron simple ?

Le perceptron simple ne peut tracer qu'une frontière de décision linéaire. Il échoue par exemple sur le problème XOR, qui nécessite une frontière non linéaire.

Question 3 / 6

Que calcule l'algorithme de rétropropagation ?

La rétropropagation applique la règle de la chaîne pour calculer efficacement le gradient de la fonction de coût par rapport à chaque poids, permettant la descente de gradient.

Question 4 / 6

Quel est l'avantage principal de la fonction ReLU ?

ReLU (max(0,z)) ne sature pas pour z > 0, ce qui évite le problème de gradient évanescent. Son gradient est soit 0 soit 1, ce qui accélère considérablement l'entraînement.

Question 5 / 6

Que fait le dropout dans un réseau de neurones ?

Le dropout désactive aléatoirement une fraction des neurones à chaque étape d'entraînement, ce qui force le réseau à développer des représentations redondantes et réduit le surapprentissage.

Question 6 / 6

Quelle fonction d'activation est utilisée en couche de sortie pour la classification multi-classes ?

Softmax convertit un vecteur de scores en une distribution de probabilités (somme = 1), ce qui est idéal pour la classification multi-classes. Chaque sortie représente la probabilité d'appartenir à une classe.

Exercice 1 — Perceptron pour porte logique

Implémentez un perceptron qui apprend la porte logique AND. Les entrées sont (0,0), (0,1), (1,0), (1,1) et les sorties attendues sont 0, 0, 0, 1. Entraînez-le et vérifiez qu'il converge. Essayez ensuite avec la porte XOR. Que constatez-vous ?

Voir la solution 
import numpy as np

X = np.array([[0,0], [0,1], [1,0], [1,1]])
y_and = np.array([0, 0, 0, 1])
y_xor = np.array([0, 1, 1, 0])

class Perceptron:
    def __init__(self, lr=0.1):
        self.w = np.zeros(2)
        self.b = 0
        self.lr = lr

    def predire(self, x):
        return 1 if np.dot(self.w, x) + self.b > 0 else 0

    def entrainer(self, X, y, epochs=100):
        for _ in range(epochs):
            for xi, yi in zip(X, y):
                err = yi - self.predire(xi)
                self.w += self.lr * err * xi
                self.b += self.lr * err

p = Perceptron()
p.entrainer(X, y_and)
print("AND :", [p.predire(x) for x in X])  # [0, 0, 0, 1] ✓

p2 = Perceptron()
p2.entrainer(X, y_xor, epochs=1000)
print("XOR :", [p2.predire(x) for x in X])  # Ne converge PAS
# Le XOR n'est pas linéairement séparable → il faut un MLP

Exercice 2 — Réseau à une couche cachée

Implémentez un réseau de neurones à une couche cachée (2 entrées, 4 neurones cachés, 1 sortie) avec NumPy. Utilisez la sigmoïde comme activation et entraînez-le sur le problème XOR avec la rétropropagation.

Voir la solution 
import numpy as np

def sigmoid(z): return 1 / (1 + np.exp(-z))
def sigmoid_deriv(z): return sigmoid(z) * (1 - sigmoid(z))

X = np.array([[0,0], [0,1], [1,0], [1,1]])
y = np.array([[0], [1], [1], [0]])

np.random.seed(42)
W1 = np.random.randn(2, 4) * 0.5
b1 = np.zeros((1, 4))
W2 = np.random.randn(4, 1) * 0.5
b2 = np.zeros((1, 1))
lr = 1.0

for epoch in range(10000):
    # Forward
    z1 = X @ W1 + b1; h1 = sigmoid(z1)
    z2 = h1 @ W2 + b2; sortie = sigmoid(z2)

    # Backward
    d2 = (sortie - y) * sigmoid_deriv(z2)
    dW2 = h1.T @ d2
    d1 = (d2 @ W2.T) * sigmoid_deriv(z1)
    dW1 = X.T @ d1

    W2 -= lr * dW2; b2 -= lr * d2.sum(axis=0)
    W1 -= lr * dW1; b1 -= lr * d1.sum(axis=0)

print("Prédictions :", np.round(sortie.flatten(), 2))
# [0.02, 0.98, 0.98, 0.02] → XOR résolu !

Exercice 3 — Comparaison des fonctions d'activation

Utilisez PyTorch ou Keras pour créer un MLP (784→128→64→10) pour classifier les chiffres MNIST. Entraînez trois versions : une avec sigmoïde, une avec tanh, et une avec ReLU. Comparez les courbes de convergence et les précisions finales.

Voir la solution 
import torch
import torch.nn as nn
from torchvision import datasets, transforms
from torch.utils.data import DataLoader

transform = transforms.Compose([transforms.ToTensor(), transforms.Normalize((0.5,), (0.5,))])
train = datasets.MNIST('.', train=True, download=True, transform=transform)
loader = DataLoader(train, batch_size=64, shuffle=True)

def creer_modele(activation):
    return nn.Sequential(
        nn.Flatten(),
        nn.Linear(784, 128), activation,
        nn.Linear(128, 64), activation,
        nn.Linear(64, 10)
    )

for nom, act in [("Sigmoid", nn.Sigmoid()), ("Tanh", nn.Tanh()), ("ReLU", nn.ReLU())]:
    modele = creer_modele(act)
    optim = torch.optim.Adam(modele.parameters(), lr=0.001)
    critere = nn.CrossEntropyLoss()

    for epoch in range(5):
        total, correct = 0, 0
        for images, labels in loader:
            pred = modele(images)
            loss = critere(pred, labels)
            optim.zero_grad(); loss.backward(); optim.step()
            correct += (pred.argmax(1) == labels).sum().item()
            total += len(labels)
    print(f"{nom:8s} → Précision : {correct/total:.1%}")
# ReLU converge plus vite et atteint ~97-98%
