Introduction

L'art de cacher l'information

La cryptographie est la science du secret. Depuis les codes de César dans l'Antiquité jusqu'aux protocoles TLS qui sécurisent le web moderne, les humains ont toujours cherché à protéger leurs communications des regards indiscrets. Aujourd'hui, la cryptographie est omniprésente : chaque transaction bancaire, chaque message chiffré, chaque connexion HTTPS repose sur des algorithmes mathématiques sophistiqués.

La cryptographie moderne ne se limite pas au chiffrement. Elle fournit des outils pour garantir la confidentialité (personne ne peut lire le message), l'intégrité (le message n'a pas été modifié), l'authenticité (l'expéditeur est bien celui qu'il prétend être) et la non-répudiation (l'expéditeur ne peut nier avoir envoyé le message).

Chiffrement vs. encodage : le chiffrement transforme des données en un format illisible à l'aide d'une clé secrète — sans la clé, le déchiffrement est computationnellement impossible. L'encodage (Base64, URL encoding) est une simple transformation de format, sans aucune sécurité.

Ce cours explore les deux grandes familles de chiffrement — symétrique et asymétrique — ainsi que les fonctions de hachage, les signatures numériques et les certificats qui forment l'infrastructure de confiance du web.

Algorithmes et protocoles cryptographiques

Chiffrement symétrique

Dans le chiffrement symétrique, la même clé sert à chiffrer et à déchiffrer le message. C'est rapide et efficace, mais pose le problème fondamental de l'échange de clé : comment transmettre la clé secrète de manière sûre ?

AES (Advanced Encryption Standard) est l'algorithme symétrique de référence depuis 2001. Il opère sur des blocs de 128 bits avec des clés de 128, 192 ou 256 bits. AES-256 est considéré comme résistant aux attaques par force brute même avec des ordinateurs quantiques.

Modes de fonctionnement : AES seul chiffre un bloc à la fois. Les modes définissent comment enchaîner les blocs. ECB (Electronic Codebook) est dangereux car les blocs identiques produisent le même chiffré. CBC, CTR et GCM sont les modes recommandés, GCM étant préféré car il assure aussi l'intégrité.

Python
from cryptography.fernet import Fernet

# Chiffrement symétrique avec Fernet (AES-128-CBC + HMAC)
cle = Fernet.generate_key()          # Génère une clé aléatoire
f = Fernet(cle)

message = b"Données confidentielles"
chiffre = f.encrypt(message)          # Chiffrement
dechiffre = f.decrypt(chiffre)        # Déchiffrement

print(chiffre)     # → b'gAAAAA...' (base64, illisible)
print(dechiffre)   # → b'Données confidentielles'

# PROBLÈME : comment transmettre `cle` de manière sûre ?
# → C'est là qu'intervient le chiffrement asymétrique

Chiffrement asymétrique

Le chiffrement asymétrique utilise une paire de clés : une clé publique (partageable) et une clé privée (secrète). Ce qui est chiffré avec la clé publique ne peut être déchiffré qu'avec la clé privée correspondante, et inversement.

RSA (Rivest-Shamir-Adleman) est l'algorithme asymétrique le plus connu. Sa sécurité repose sur la difficulté de factoriser le produit de deux très grands nombres premiers. Avec une clé de 2048 bits, la factorisation prendrait des milliards d'années avec la technologie actuelle.

Courbes elliptiques (ECC) : alternative moderne à RSA, offrant le même niveau de sécurité avec des clés beaucoup plus courtes. Une clé ECC de 256 bits équivaut à une clé RSA de 3072 bits, ce qui la rend plus rapide et économe en ressources.

Python
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives import hashes

# Génération de la paire de clés RSA
cle_privee = rsa.generate_private_key(
    public_exponent=65537,
    key_size=2048
)
cle_publique = cle_privee.public_key()

# Alice chiffre avec la clé publique de Bob
message = b"Message secret pour Bob"
chiffre = cle_publique.encrypt(
    message,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None
    )
)

# Bob déchiffre avec sa clé privée
dechiffre = cle_privee.decrypt(
    chiffre,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None
    )
)
print(dechiffre)  # → b'Message secret pour Bob'

Fonctions de hachage

Une fonction de hachage transforme une entrée de taille quelconque en une empreinte de taille fixe. C'est une fonction à sens unique : il est computationnellement impossible de retrouver l'entrée à partir du hash.

Propriétés essentielles : déterministe (même entrée = même hash), résistance à la préimage (impossible de retrouver l'entrée), résistance aux collisions (impossible de trouver deux entrées avec le même hash), effet avalanche (un changement d'un seul bit modifie radicalement le hash).

Python
import hashlib

# SHA-256 : standard actuel, produit un hash de 256 bits (64 hex)
msg1 = "Bonjour le monde"
msg2 = "Bonjour le Monde"  # Un seul caractère changé

h1 = hashlib.sha256(msg1.encode()).hexdigest()
h2 = hashlib.sha256(msg2.encode()).hexdigest()

print(h1)  # → a7f5f35...  (64 caractères hex)
print(h2)  # → 3e2d8b1...  (complètement différent !)

# Algorithmes à éviter : MD5 et SHA-1 (vulnérables aux collisions)
# Recommandés : SHA-256, SHA-3, BLAKE2

# Pour les mots de passe : utiliser bcrypt, scrypt ou Argon2
import bcrypt
mdp = b"MonMotDePasse123"
sel = bcrypt.gensalt(rounds=12)
hache = bcrypt.hashpw(mdp, sel)
# Vérification : bcrypt.checkpw(mdp, hache) → True

Signatures numériques

La signature numérique est l'équivalent électronique de la signature manuscrite. Elle utilise le chiffrement asymétrique à l'envers : l'expéditeur signe avec sa clé privée, et n'importe qui peut vérifier avec sa clé publique.

Le processus : (1) hacher le message avec SHA-256, (2) chiffrer le hash avec la clé privée de l'expéditeur, (3) joindre la signature au message. Le destinataire vérifie en déchiffrant la signature avec la clé publique et en comparant avec le hash recalculé.

Processus de signature
SIGNATURE :
  Message → SHA-256 → Hash → Chiffrer(clé_privée) → Signature

VÉRIFICATION :
  Signature → Déchiffrer(clé_publique) → Hash_reçu
  Message   → SHA-256                   → Hash_calculé
  Hash_reçu == Hash_calculé ? → Signature valide

Garanties fournies :
  ✓ Authenticité   → Seul le détenteur de la clé privée a pu signer
  ✓ Intégrité      → Toute modification invalide la signature
  ✓ Non-répudiation → L'auteur ne peut nier avoir signé

Certificats et PKI

Un certificat numérique (X.509) lie une clé publique à une identité. Il est émis par une autorité de certification (CA) qui garantit l'authenticité du lien. C'est le fondement de HTTPS : quand vous visitez un site sécurisé, votre navigateur vérifie le certificat du serveur.

La PKI (Public Key Infrastructure) est l'ensemble des politiques, procédures et technologies qui gèrent les certificats : émission, renouvellement, révocation. La chaîne de confiance part des CA racines (préinstallées dans les navigateurs) vers les CA intermédiaires puis les certificats finaux.

Let's Encrypt a démocratisé HTTPS en proposant des certificats gratuits et automatisables. Depuis son lancement, le pourcentage de pages chargées en HTTPS est passé d'environ 40% à plus de 90% du trafic web mondial.

Échange de clés et protocole TLS

Le protocole TLS (Transport Layer Security) combine chiffrement symétrique et asymétrique pour sécuriser les communications web. Le handshake TLS résout le problème de l'échange de clé :

(1) Le client et le serveur négocient les algorithmes (cipher suite). (2) Le serveur présente son certificat. (3) Un échange de clé Diffie-Hellman (ou ECDHE) établit un secret partagé. (4) Ce secret génère les clés symétriques pour le chiffrement AES de la session.

Handshake TLS 1.3 simplifié
Client                                  Serveur
  │                                        │
  │──── ClientHello ──────────────────────→│
  │     (versions, cipher suites,          │
  │      clé ECDHE client)                 │
  │                                        │
  │←─── ServerHello ──────────────────────│
  │     (cipher suite choisie,             │
  │      clé ECDHE serveur,                │
  │      certificat, signature)            │
  │                                        │
  │  Secret partagé calculé des deux côtés  │
  │  via Diffie-Hellman sur courbe elliptique│
  │                                        │
  │←──→ Communication chiffrée AES-GCM ←──→│

TLS 1.3 : un seul aller-retour (1-RTT)
vs TLS 1.2 : deux allers-retours (2-RTT)

Menaces sur la cryptographie

Malgré la robustesse des algorithmes modernes, plusieurs menaces existent :

Informatique quantique : l'algorithme de Shor pourrait casser RSA et ECC en temps polynomial sur un ordinateur quantique suffisamment puissant. La cryptographie post-quantique développe des algorithmes résistants, comme CRYSTALS-Kyber (standardisé par le NIST en 2024).

Erreurs d'implémentation : un algorithme sûr peut être rendu vulnérable par une implémentation défaillante. Les attaques par canal auxiliaire (timing, consommation électrique) exploitent les fuites d'information physiques plutôt que les failles mathématiques.

Mauvaise utilisation : utiliser ECB au lieu de GCM, des clés trop courtes, un générateur de nombres aléatoires prévisible, ou stocker des clés en clair sont des erreurs fréquentes qui annulent la protection cryptographique.

Exercice 1 — Chiffrement de César

Implémentez en Python le chiffrement de César : chaque lettre est décalée de N positions dans l'alphabet. Écrivez les fonctions chiffrer(texte, decalage) et dechiffrer(texte, decalage). Puis écrivez une fonction casser(texte) qui teste les 26 décalages possibles.

Voir la solution 
def chiffrer(texte, decalage):
    resultat = ""
    for c in texte:
        if c.isalpha():
            base = ord('A') if c.isupper() else ord('a')
            resultat += chr((ord(c) - base + decalage) % 26 + base)
        else:
            resultat += c
    return resultat

def dechiffrer(texte, decalage):
    return chiffrer(texte, -decalage)

def casser(texte):
    for d in range(26):
        print(f"Décalage {d:2d} : {dechiffrer(texte, d)}")

# Test
chiffre = chiffrer("Bonjour le monde", 3)
print(chiffre)              # → "Erqmrxu oh prqgh"
print(dechiffrer(chiffre, 3))  # → "Bonjour le monde"

# César est trivial à casser : 26 possibilités seulement !

Exercice 2 — Hachage et intégrité

Écrivez un programme Python qui : (a) calcule le hash SHA-256 d'un fichier, (b) modifie un seul octet du fichier, (c) recalcule le hash et compare. Démontrez l'effet avalanche en affichant le pourcentage de bits différents entre les deux hash.

Voir la solution 
import hashlib

def hash_fichier(chemin):
    h = hashlib.sha256()
    with open(chemin, 'rb') as f:
        for bloc in iter(lambda: f.read(4096), b''):
            h.update(bloc)
    return h.hexdigest()

def bits_differents(h1, h2):
    """Compte les bits différents entre deux hash hex."""
    n1 = int(h1, 16)
    n2 = int(h2, 16)
    xor = n1 ^ n2
    return bin(xor).count('1')

# Créer un fichier de test
with open("test.txt", "w") as f:
    f.write("Contenu original du fichier")

hash1 = hash_fichier("test.txt")

# Modifier un seul caractère
with open("test.txt", "w") as f:
    f.write("Contenu Original du fichier")  # 'o' → 'O'

hash2 = hash_fichier("test.txt")

diff = bits_differents(hash1, hash2)
print(f"Hash 1 : {hash1}")
print(f"Hash 2 : {hash2}")
print(f"Bits différents : {diff}/256 ({diff/256*100:.1f}%)")
# → Environ 50% des bits changent (effet avalanche)

Exercice 3 — Chiffrement symétrique AES

En utilisant la bibliothèque cryptography de Python, chiffrez un message avec AES-GCM (qui assure confidentialité et intégrité). Montrez qu'une modification du texte chiffré est détectée lors du déchiffrement.

Voir la solution 
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

# Générer une clé AES-256 (32 octets)
cle = AESGCM.generate_key(bit_length=256)
aesgcm = AESGCM(cle)

# Chiffrement
nonce = os.urandom(12)  # 96 bits, unique pour chaque message
message = b"Information ultra confidentielle"
chiffre = aesgcm.encrypt(nonce, message, None)

# Déchiffrement normal
dechiffre = aesgcm.decrypt(nonce, chiffre, None)
print(dechiffre)  # → b'Information ultra confidentielle'

# Tentative de modification du texte chiffré
chiffre_modifie = bytearray(chiffre)
chiffre_modifie[0] ^= 0xFF  # Inverser le premier octet
try:
    aesgcm.decrypt(nonce, bytes(chiffre_modifie), None)
except Exception as e:
    print(f"Altération détectée : {e}")
