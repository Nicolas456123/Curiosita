01

Introduction

Pourquoi utiliser un framework ?

Les frameworks front-end structurent le développement d'interfaces utilisateur complexes. Ils introduisent le concept de composants — des blocs d'interface réutilisables qui encapsulent leur propre logique, style et état. Au lieu de manipuler le DOM manuellement, le développeur décrit ce que l'interface doit afficher, et le framework se charge des mises à jour.

Les trois grands : React (Meta, 2013) est une bibliothèque centrée sur le rendu ; Vue.js (Evan You, 2014) offre un framework progressif ; Angular (Google, 2016) est un framework complet avec TypeScript. Chacun a son écosystème, mais les concepts fondamentaux sont partagés.

Ces outils permettent de construire des Single Page Applications (SPA) : des applications web qui ne rechargent pas la page entière à chaque navigation, offrant une expérience fluide comparable à une application native.

02

Le cours

Concepts et frameworks majeurs

L'architecture par composants

Un composant est une unité autonome d'interface. Il reçoit des données en entrée (props), gère son propre état interne, et produit un rendu. Les composants s'imbriquent pour former l'arbre de l'interface complète.

Avantages des composants :
Réutilisabilité — un bouton, un formulaire, une carte peuvent être utilisés partout
Isolation — chaque composant gère sa propre logique
Composabilité — les petits composants s'assemblent en interfaces complexes
Testabilité — chaque composant peut être testé indépendamment

React : composants et JSX

React utilise le JSX, une extension syntaxique qui mélange JavaScript et HTML. Les composants sont des fonctions qui retournent du JSX. L'état est géré avec les Hooks (useState, useEffect, etc.).

JSX (React)
import { useState } from 'react';

function Compteur() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Compteur : {count}</p>
      <button onClick={() => setCount(count + 1)}>
        +1
      </button>
    </div>
  );
}

// Utilisation : <Compteur />

React utilise un DOM virtuel (Virtual DOM) : il calcule les différences entre l'ancien et le nouvel état de l'interface, puis applique uniquement les modifications nécessaires au vrai DOM, ce qui optimise les performances.

Vue.js : réactivité et templates

Vue.js sépare le template HTML, le script JavaScript et le style CSS dans un fichier .vue (Single File Component). Son système de réactivité détecte automatiquement les changements de données et met à jour l'interface.

Vue.js (Composition API)
<template>
  <div>
    <p>Compteur : {{ count }}</p>
    <button @click="count++">+1</button>
  </div>
</template>

<script setup>
import { ref } from 'vue';
const count = ref(0);
</script>

<style scoped>
button { padding: 0.5rem 1rem; }
</style>

Angular : framework complet

Angular est un framework complet écrit en TypeScript. Il fournit nativement le routage, l'injection de dépendances, les formulaires, le client HTTP et les tests. Les composants utilisent des décorateurs et des templates HTML avec des directives.

TypeScript (Angular)
import { Component } from '@angular/core';

@Component({
  selector: 'app-compteur',
  template: `
    <p>Compteur : {{ count }}</p>
    <button (click)="increment()">+1</button>
  `
})
export class CompteurComponent {
  count = 0;
  increment() { this.count++; }
}

Gestion de l'état

Dans une application complexe, l'état (données) doit être partagé entre de nombreux composants. Des solutions de gestion d'état globale centralisent les données et les mutations pour maintenir la cohérence.

React : Context API (simple), Redux ou Zustand (complexe)
Vue.js : Pinia (recommandé), anciennement Vuex
Angular : Services avec RxJS, NgRx pour les applications complexes

Le principe commun est le flux unidirectionnel : les données descendent des composants parents vers les enfants via les props, et les événements remontent des enfants vers les parents. Cela rend le flux de données prévisible et facilite le débogage.

Routage et Single Page Applications

Le routeur permet de naviguer entre les pages sans rechargement complet. Il associe des URLs à des composants et gère l'historique du navigateur. Chaque framework a son routeur : React Router, Vue Router, Angular Router.

React Router
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Accueil</Link>
        <Link to="/produits">Produits</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Accueil />} />
        <Route path="/produits" element={<Produits />} />
        <Route path="/produits/:id" element={<Detail />} />
      </Routes>
    </BrowserRouter>
  );
}

Comparaison et choix

React — flexible, vaste écosystème, courbe d'apprentissage modérée. Idéal pour des projets de toute taille.
Vue.js — progressif, syntaxe intuitive, documentation excellente. Parfait pour les débutants et les projets moyens.
Angular — complet et structuré, TypeScript natif. Adapté aux grandes applications d'entreprise.

Le meilleur choix dépend du projet, de l'équipe et de l'écosystème existant. Les concepts (composants, état, routage) se transfèrent d'un framework à l'autre.

Question 1 / 6

Qu'est-ce qu'un composant dans un framework front-end ?

Un composant encapsule une partie de l'interface (HTML, CSS, logique JavaScript). Il est réutilisable, testable et composable avec d'autres composants pour former l'application.

Question 2 / 6

Qu'est-ce que le Virtual DOM utilisé par React ?

Le Virtual DOM est une copie légère du DOM en mémoire. React compare l'ancien et le nouveau Virtual DOM (diffing), puis applique uniquement les changements nécessaires au vrai DOM (reconciliation).

Question 3 / 6

Quel hook React permet de gérer l'état local d'un composant ?

useState retourne une paire [valeur, setter]. Quand le setter est appelé, React re-rend le composant avec la nouvelle valeur. useEffect gère les effets de bord, useRef stocke une référence mutable.

Question 4 / 6

Qu'est-ce qu'une SPA (Single Page Application) ?

Une SPA charge une seule page HTML puis met à jour le contenu dynamiquement via JavaScript. La navigation entre les « pages » se fait sans rechargement, offrant une expérience fluide.

Question 5 / 6

Quel framework front-end est écrit nativement en TypeScript ?

Angular est construit en TypeScript et l'utilise par défaut dans tous les projets. React et Vue supportent TypeScript mais ne l'imposent pas.

Question 6 / 6

Que sont les « props » dans un framework front-end ?

Les props (propriétés) sont le mécanisme de passage de données du parent vers l'enfant. Elles sont en lecture seule dans l'enfant, ce qui garantit un flux de données unidirectionnel et prévisible.

Exercice 3 — Liste filtrable (React)

Créez un composant React qui affiche une liste de produits filtrables par catégorie. Utilisez useState pour le filtre actif et filter() pour n'afficher que les produits correspondants.

Voir la solution 
import { useState } from 'react';

const produits = [
  { nom: "Laptop", categorie: "Électronique" },
  { nom: "T-shirt", categorie: "Vêtements" },
  { nom: "Casque", categorie: "Électronique" },
  { nom: "Jean", categorie: "Vêtements" },
  { nom: "Tablette", categorie: "Électronique" },
];

function ListeProduits() {
  const [filtre, setFiltre] = useState("Tous");
  const categories = ["Tous", ...new Set(produits.map(p => p.categorie))];
  const affiches = filtre === "Tous"
    ? produits
    : produits.filter(p => p.categorie === filtre);

  return (
    <div>
      {categories.map(c => (
        <button key={c} onClick={() => setFiltre(c)}
          className={c === filtre ? 'active' : ''}>{c}</button>
      ))}
      <ul>
        {affiches.map((p, i) => <li key={i}>{p.nom}</li>)}
      </ul>
    </div>
  );
}

Exercice 4 — Formulaire contrôlé (React)

Créez un formulaire d'inscription avec les champs nom, email et mot de passe. Utilisez useState pour chaque champ. Affichez un résumé sous le formulaire lorsque l'utilisateur saisit. Ajoutez une validation basique (champs requis, email valide).

Voir la solution 
import { useState } from 'react';

function Inscription() {
  const [form, setForm] = useState({ nom: '', email: '', mdp: '' });
  const [erreurs, setErreurs] = useState({});

  const handleChange = (e) => {
    setForm({ ...form, [e.target.name]: e.target.value });
  };

  const valider = () => {
    const err = {};
    if (!form.nom) err.nom = 'Nom requis';
    if (!form.email.includes('@')) err.email = 'Email invalide';
    if (form.mdp.length < 8) err.mdp = '8 caractères minimum';
    setErreurs(err);
    return Object.keys(err).length === 0;
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    if (valider()) alert('Inscription réussie !');
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="nom" value={form.nom} onChange={handleChange} placeholder="Nom" />
      {erreurs.nom && <span className="error">{erreurs.nom}</span>}
      <input name="email" value={form.email} onChange={handleChange} placeholder="Email" />
      {erreurs.email && <span className="error">{erreurs.email}</span>}
      <input name="mdp" type="password" value={form.mdp} onChange={handleChange} placeholder="Mot de passe" />
      {erreurs.mdp && <span className="error">{erreurs.mdp}</span>}
      <button type="submit">S'inscrire</button>
    </form>
  );
}

Exercice 5 — Configuration du routage

Configurez un routeur React avec trois routes : / (Accueil), /a-propos (À propos), /contact (Contact). Ajoutez une barre de navigation avec des liens et une page 404 pour les routes inconnues.

Voir la solution 
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function Accueil() { return <h1>Accueil</h1>; }
function APropos() { return <h1>À propos</h1>; }
function Contact() { return <h1>Contact</h1>; }
function NotFound() { return <h1>404 — Page non trouvée</h1>; }

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Accueil</Link>
        <Link to="/a-propos">À propos</Link>
        <Link to="/contact">Contact</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Accueil />} />
        <Route path="/a-propos" element={<APropos />} />
        <Route path="/contact" element={<Contact />} />
        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  );
}
