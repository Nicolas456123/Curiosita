Introduction et vue d'ensemble

La programmation orientée objet (POO) est un paradigme qui organise le code autour d'objets — des entités qui combinent données (attributs) et comportements (méthodes). Plutôt que de penser en termes de fonctions et de variables globales, on modélise le problème avec des objets qui interagissent entre eux.

Pourquoi la POO ? Elle permet de créer du code modulaire, réutilisable et maintenable. Un objet « Voiture » contient ses données (couleur, vitesse) et ses actions (accélérer, freiner). Tout le code lié à la voiture est regroupé au même endroit.

Python est un langage multi-paradigme : il supporte aussi bien la programmation procédurale que la POO. En Python, tout est un objet — les entiers, les chaînes, les fonctions, les modules. La POO n'est pas une obligation, mais elle devient essentielle pour les projets de taille moyenne et grande.

Ce chapitre couvre les quatre piliers de la POO : les classes, l'héritage, l'encapsulation et le polymorphisme, ainsi que les méthodes spéciales qui donnent toute leur puissance aux objets Python.
