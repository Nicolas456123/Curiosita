Introduction

Un même message, plusieurs comportements

Le polymorphisme (du grec « plusieurs formes ») est la capacité d'utiliser une même interface pour des objets de types différents. Chaque objet répond au même message (appel de méthode) à sa manière. Ce principe permet d'écrire du code générique et extensible qui n'a pas besoin de connaître le type exact des objets qu'il manipule.

Python excelle dans ce domaine grâce au typage dynamique et au duck typing : ce qui compte n'est pas le type d'un objet, mais les méthodes et attributs qu'il possède. Si un objet « marche comme un canard et cancane comme un canard, alors c'est un canard. »

Exemple quotidien : la fonction len() est polymorphe. Elle fonctionne sur des listes, des chaînes, des dictionnaires, des ensembles — tout objet qui implémente __len__(). Vous n'avez pas besoin de len_liste(), len_str(), etc.

Ce cours explore les différentes formes de polymorphisme en Python : le duck typing, la surcharge d'opérateurs, les interfaces informelles et les classes abstraites (ABC).

Le cours

Les formes du polymorphisme en Python

Polymorphisme par héritage

La forme la plus classique : des classes enfants redéfinissent une méthode de la classe parente. Le code appelant manipule des objets via l'interface commune :

class Forme:
    def aire(self):
        raise NotImplementedError

    def description(self):
        return f"{self.__class__.__name__} — aire = {self.aire():.2f}"

class Cercle(Forme):
    def __init__(self, rayon):
        self.rayon = rayon

    def aire(self):
        return 3.14159 * self.rayon ** 2

class Rectangle(Forme):
    def __init__(self, largeur, hauteur):
        self.largeur = largeur
        self.hauteur = hauteur

    def aire(self):
        return self.largeur * self.hauteur

class Triangle(Forme):
    def __init__(self, base, hauteur):
        self.base = base
        self.hauteur = hauteur

    def aire(self):
        return 0.5 * self.base * self.hauteur

# Code polymorphe — fonctionne avec n'importe quelle Forme
formes = [Cercle(5), Rectangle(4, 6), Triangle(3, 8)]
for f in formes:
    print(f.description())
# Cercle — aire = 78.54
# Rectangle — aire = 24.00
# Triangle — aire = 12.00

La boucle for ne sait pas quel type de forme elle manipule — elle appelle simplement aire() et chaque objet répond à sa manière.

Duck typing

En Python, le polymorphisme ne nécessite pas d'héritage. N'importe quel objet qui possède la bonne méthode peut être utilisé :

class Fichier:
    def __init__(self, nom, taille):
        self.nom = nom
        self.taille = taille

    def afficher(self):
        return f"Fichier: {self.nom} ({self.taille} Ko)"

class Dossier:
    def __init__(self, nom, nb_elements):
        self.nom = nom
        self.nb_elements = nb_elements

    def afficher(self):
        return f"Dossier: {self.nom} ({self.nb_elements} éléments)"

class LienSymbolique:
    def __init__(self, nom, cible):
        self.nom = nom
        self.cible = cible

    def afficher(self):
        return f"Lien: {self.nom} → {self.cible}"

# Aucun lien d'héritage entre ces classes !
# Mais elles partagent la même interface (méthode afficher)
elements = [
    Fichier("rapport.pdf", 245),
    Dossier("photos", 42),
    LienSymbolique("raccourci", "/home/docs")
]

for e in elements:
    print(e.afficher())  # Fonctionne grâce au duck typing

Duck typing : « If it walks like a duck and quacks like a duck, then it's a duck. » Python ne vérifie pas le type d'un objet avant d'appeler une méthode — il essaie simplement de l'appeler. Si la méthode existe, tout va bien. Sinon, une AttributeError est levée.

Surcharge d'opérateurs

Python permet de redéfinir le comportement des opérateurs (+, -, *, ==, etc.) pour vos propres classes via les méthodes spéciales :

class Vecteur:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, autre):
        """v1 + v2"""
        return Vecteur(self.x + autre.x, self.y + autre.y)

    def __sub__(self, autre):
        """v1 - v2"""
        return Vecteur(self.x - autre.x, self.y - autre.y)

    def __mul__(self, scalaire):
        """v * 3"""
        return Vecteur(self.x * scalaire, self.y * scalaire)

    def __eq__(self, autre):
        """v1 == v2"""
        return self.x == autre.x and self.y == autre.y

    def __abs__(self):
        """abs(v) — norme du vecteur"""
        return (self.x ** 2 + self.y ** 2) ** 0.5

    def __repr__(self):
        return f"Vecteur({self.x}, {self.y})"

v1 = Vecteur(3, 4)
v2 = Vecteur(1, 2)

print(v1 + v2)      # Vecteur(4, 6)
print(v1 - v2)      # Vecteur(2, 2)
print(v1 * 3)       # Vecteur(9, 12)
print(v1 == v2)     # False
print(abs(v1))      # 5.0

Les opérateurs les plus courants et leurs méthodes spéciales :

  • + → __add__ | - → __sub__ | * → __mul__
  • / → __truediv__ | // → __floordiv__ | % → __mod__
  • == → __eq__ | < → __lt__ | > → __gt__
  • [] → __getitem__ | in → __contains__ | len() → __len__

Protocoles Python (interfaces informelles)

Python définit des protocoles — des ensembles de méthodes spéciales qui permettent à un objet de se comporter comme un type standard :

class Playlist:
    """Se comporte comme une séquence (protocole itérable + indexable)."""

    def __init__(self, nom):
        self.nom = nom
        self._morceaux = []

    def ajouter(self, morceau):
        self._morceaux.append(morceau)

    def __len__(self):
        return len(self._morceaux)

    def __getitem__(self, index):
        return self._morceaux[index]

    def __contains__(self, morceau):
        return morceau in self._morceaux

    def __iter__(self):
        return iter(self._morceaux)

playlist = Playlist("Rock classique")
playlist.ajouter("Bohemian Rhapsody")
playlist.ajouter("Stairway to Heaven")
playlist.ajouter("Hotel California")

print(len(playlist))              # 3
print(playlist[0])                # Bohemian Rhapsody
print("Hotel California" in playlist)  # True

for morceau in playlist:          # itérable !
    print(f"  ♪ {morceau}")

Classes abstraites (ABC)

Le module abc (Abstract Base Classes) permet de définir des interfaces formelles que les sous-classes doivent obligatoirement implémenter :

from abc import ABC, abstractmethod

class FormeFermee(ABC):
    """Classe abstraite — ne peut pas être instanciée directement."""

    @abstractmethod
    def aire(self):
        """Doit être implémentée par les sous-classes."""
        pass

    @abstractmethod
    def perimetre(self):
        """Doit être implémentée par les sous-classes."""
        pass

    def rapport(self):
        """Méthode concrète — héritée par toutes les sous-classes."""
        return f"Aire/Périmètre = {self.aire() / self.perimetre():.3f}"

# f = FormeFermee()  # TypeError: Can't instantiate abstract class

class Carre(FormeFermee):
    def __init__(self, cote):
        self.cote = cote

    def aire(self):
        return self.cote ** 2

    def perimetre(self):
        return 4 * self.cote

class CercleF(FormeFermee):
    def __init__(self, rayon):
        self.rayon = rayon

    def aire(self):
        return 3.14159 * self.rayon ** 2

    def perimetre(self):
        return 2 * 3.14159 * self.rayon

# Si on oublie d'implémenter une méthode abstraite :
# class FormeIncomplete(FormeFermee):
#     def aire(self):
#         return 0
# # TypeError: Can't instantiate abstract class FormeIncomplete
# # with abstract method perimetre

c = Carre(5)
print(c.aire())       # 25
print(c.rapport())    # Aire/Périmètre = 1.250

ABC vs duck typing : les classes abstraites ajoutent un contrat explicite. Si une sous-classe oublie d'implémenter une méthode abstraite, Python lève une erreur dès l'instanciation, pas lors de l'appel. C'est plus sûr pour les grands projets et les bibliothèques.

Protocoles avec @abstractmethod

On peut combiner ABC avec des propriétés abstraites pour créer des interfaces riches :

from abc import ABC, abstractmethod

class Exportable(ABC):
    @abstractmethod
    def to_dict(self):
        pass

    @abstractmethod
    def to_json(self):
        pass

class Affichable(ABC):
    @abstractmethod
    def afficher(self):
        pass

class Produit(Exportable, Affichable):
    def __init__(self, nom, prix):
        self.nom = nom
        self.prix = prix

    def to_dict(self):
        return {"nom": self.nom, "prix": self.prix}

    def to_json(self):
        import json
        return json.dumps(self.to_dict(), ensure_ascii=False)

    def afficher(self):
        return f"{self.nom} — {self.prix:.2f}€"

p = Produit("Clavier mécanique", 89.99)
print(p.afficher())    # Clavier mécanique — 89.99€
print(p.to_json())     # {"nom": "Clavier mécanique", "prix": 89.99}

Fonctions polymorphes avec dispatch

Le module functools propose singledispatch pour créer des fonctions qui se comportent différemment selon le type de leur argument :

from functools import singledispatch

@singledispatch
def formater(valeur):
    """Comportement par défaut."""
    return str(valeur)

@formater.register(int)
def _(valeur):
    return f"{valeur:,}".replace(",", " ")

@formater.register(float)
def _(valeur):
    return f"{valeur:.2f}"

@formater.register(list)
def _(valeur):
    return f"[{len(valeur)} éléments]"

print(formater(1234567))     # 1 234 567
print(formater(3.14159))     # 3.14
print(formater([1, 2, 3]))   # [3 éléments]
print(formater("texte"))     # texte (comportement par défaut)

Quiz

🐍 Polymorphisme — Quiz Score : 0 / 6

Question 1 / 6

Qu'est-ce que le « duck typing » en Python ?

Le duck typing signifie : « si ça marche comme un canard et cancane comme un canard, c'est un canard. » Python vérifie les capacités d'un objet (ses méthodes) plutôt que son type déclaré.

Question 2 / 6

Quelle méthode spéciale permet de surcharger l'opérateur + ?

__add__(self, other) est appelée quand on utilise l'opérateur +. Par exemple, a + b appelle a.__add__(b). C'est ce qui permet à + de fonctionner avec des entiers, des chaînes et des listes.

Question 3 / 6

Que se passe-t-il si on essaie d'instancier une classe abstraite (ABC) ?

Python lève un TypeError si on essaie d'instancier une classe abstraite ou une sous-classe qui n'a pas implémenté toutes les méthodes abstraites. C'est un contrat strict.

Question 4 / 6

Quel module Python fournit les classes abstraites (ABC) ?

Le module abc (Abstract Base Classes) fournit ABC et le décorateur @abstractmethod. On utilise from abc import ABC, abstractmethod pour définir des classes abstraites.

Question 5 / 6

Quelle méthode spéciale faut-il implémenter pour que len(obj) fonctionne ?

__len__ doit retourner un entier positif. Quand on appelle len(obj), Python appelle en réalité obj.__len__(). C'est un exemple de protocole Python.

Question 6 / 6

Quelle est la différence principale entre le duck typing et les ABC ?

Avec le duck typing, l'erreur survient quand on appelle une méthode manquante (AttributeError). Avec les ABC, l'erreur survient dès l'instanciation si une méthode abstraite n'est pas implémentée (TypeError). Les ABC offrent une détection plus précoce des erreurs.

Exercice 1 — Classe Monnaie avec surcharge d'opérateurs

Créez une classe Monnaie avec montant et devise. Surchargez + (addition de deux montants de même devise), * (multiplication par un scalaire), == et <. Levez une erreur si on additionne des devises différentes.

Voir la solution 
class Monnaie:
    def __init__(self, montant, devise="EUR"):
        self.montant = montant
        self.devise = devise

    def __add__(self, autre):
        if self.devise != autre.devise:
            raise ValueError(f"Devises différentes : {self.devise} et {autre.devise}")
        return Monnaie(self.montant + autre.montant, self.devise)

    def __mul__(self, scalaire):
        return Monnaie(self.montant * scalaire, self.devise)

    def __eq__(self, autre):
        return self.montant == autre.montant and self.devise == autre.devise

    def __lt__(self, autre):
        if self.devise != autre.devise:
            raise ValueError("Comparaison impossible entre devises différentes")
        return self.montant < autre.montant

    def __repr__(self):
        return f"{self.montant:.2f} {self.devise}"

# Test
a = Monnaie(100, "EUR")
b = Monnaie(50, "EUR")
print(a + b)       # 150.00 EUR
print(a * 3)       # 300.00 EUR
print(a == b)      # False
print(b < a)       # True

Exercice 2 — Classe abstraite Animal

Créez une classe abstraite Animal avec les méthodes abstraites parler() et se_deplacer(), et une propriété abstraite regime. Créez trois implémentations : Aigle, Dauphin et Serpent.

Voir la solution 
from abc import ABC, abstractmethod

class Animal(ABC):
    def __init__(self, nom):
        self.nom = nom

    @abstractmethod
    def parler(self):
        pass

    @abstractmethod
    def se_deplacer(self):
        pass

    @property
    @abstractmethod
    def regime(self):
        pass

    def fiche(self):
        return f"{self.nom} | {self.regime} | {self.se_deplacer()}"

class Aigle(Animal):
    def parler(self):
        return "Cri perçant !"

    def se_deplacer(self):
        return "Vole dans les airs"

    @property
    def regime(self):
        return "Carnivore"

class Dauphin(Animal):
    def parler(self):
        return "Clics et sifflements"

    def se_deplacer(self):
        return "Nage dans l'océan"

    @property
    def regime(self):
        return "Piscivore"

class Serpent(Animal):
    def parler(self):
        return "Sssssss"

    def se_deplacer(self):
        return "Rampe au sol"

    @property
    def regime(self):
        return "Carnivore"

# Test
animaux = [Aigle("Horus"), Dauphin("Flipper"), Serpent("Kaa")]

Exercice 3 — Collection personnalisée (protocoles)

Créez une classe FileAttente (FIFO) qui implémente les protocoles __len__, __contains__, __iter__ et __bool__. Ajoutez les méthodes enfiler() et defiler().

Voir la solution 
class FileAttente:
    def __init__(self):
        self._elements = []

    def enfiler(self, element):
        self._elements.append(element)

    def defiler(self):
        if not self._elements:
            raise IndexError("La file est vide")
        return self._elements.pop(0)

    def __len__(self):
        return len(self._elements)

    def __contains__(self, element):
        return element in self._elements

    def __iter__(self):
        return iter(self._elements)

    def __bool__(self):
        return len(self._elements) > 0

    def __repr__(self):
        return f"FileAttente({self._elements})"

# Test
f = FileAttente()
f.enfiler("Alice")
f.enfiler("Bob")
f.enfiler("Charlie")

print(len(f))           # 3
print("Bob" in f)       # True
print(bool(f))          # True

for personne in f:
    print(f"  → {personne}")

print(f.defiler())      # Alice (premier arrivé, premier sorti)
print(f)                # FileAttente(['Bob', 'Charlie'])

Exercice 4 — Matrice 2x2 avec surcharge

Créez une classe Matrice2x2 qui représente une matrice 2x2. Surchargez +, * (multiplication matricielle), == et __repr__. Ajoutez une méthode determinant().

Voir la solution 
class Matrice2x2:
    def __init__(self, a, b, c, d):
        # | a  b |
        # | c  d |
        self.a, self.b = a, b
        self.c, self.d = c, d

    def __add__(self, autre):
        return Matrice2x2(
            self.a + autre.a, self.b + autre.b,
            self.c + autre.c, self.d + autre.d
        )

    def __mul__(self, autre):
        if isinstance(autre, Matrice2x2):
            return Matrice2x2(
                self.a*autre.a + self.b*autre.c,
                self.a*autre.b + self.b*autre.d,
                self.c*autre.a + self.d*autre.c,
                self.c*autre.b + self.d*autre.d
            )
        # Multiplication par un scalaire
        return Matrice2x2(self.a*autre, self.b*autre,
                          self.c*autre, self.d*autre)

    def __eq__(self, autre):
        return (self.a == autre.a and self.b == autre.b
                and self.c == autre.c and self.d == autre.d)

    def determinant(self):
        return self.a * self.d - self.b * self.c

    def __repr__(self):
        return f"|{self.a} {self.b}|\n|{self.c} {self.d}|"

# Test
m1 = Matrice2x2(1, 2, 3, 4)
m2 = Matrice2x2(5, 6, 7, 8)
print(m1 + m2)           # |6 8|\n|10 12|
print(m1 * m2)           # |19 22|\n|43 50|
print(m1.determinant())  # -2

Exercice 5 — Système de paiement polymorphe

Créez une classe abstraite MethodePaiement avec payer(montant) et solde_suffisant(montant). Implémentez CarteBancaire, Especes et PayPal. Créez une fonction effectuer_achat(methode, montant) qui fonctionne avec toutes.

Voir la solution 
from abc import ABC, abstractmethod

class MethodePaiement(ABC):
    @abstractmethod
    def payer(self, montant):
        pass

    @abstractmethod
    def solde_suffisant(self, montant):
        pass

class CarteBancaire(MethodePaiement):
    def __init__(self, plafond):
        self.plafond = plafond
        self.depenses = 0

    def solde_suffisant(self, montant):
        return self.depenses + montant <= self.plafond

    def payer(self, montant):
        if not self.solde_suffisant(montant):
            return f"CB refusée : plafond dépassé"
        self.depenses += montant
        return f"CB : {montant:.2f}€ débité (reste {self.plafond - self.depenses:.2f}€)"

class Especes(MethodePaiement):
    def __init__(self, montant_disponible):
        self.montant_disponible = montant_disponible

    def solde_suffisant(self, montant):
        return self.montant_disponible >= montant

    def payer(self, montant):
        if not self.solde_suffisant(montant):
            return "Espèces insuffisantes"
        self.montant_disponible -= montant
        rendu = 0
        return f"Espèces : {montant:.2f}€ payé (reste {self.montant_disponible:.2f}€)"

class PayPal(MethodePaiement):
    def __init__(self, email, solde):
        self.email = email
        self.solde = solde

    def solde_suffisant(self, montant):
        return self.solde >= montant

    def payer(self, montant):
        if not self.solde_suffisant(montant):
            return f"PayPal : solde insuffisant"
        self.solde -= montant
        return f"PayPal ({self.email}) : {montant:.2f}€ débité"

def effectuer_achat(methode, montant):
    """Fonction polymorphe — accepte toute MethodePaiement."""
    print(methode.payer(montant))

# Test
cb = CarteBancaire(500)
cash = Especes(100)
pp = PayPal("alice@mail.com", 250)

effectuer_achat(cb, 150)    # CB : 150.00€ débité
effectuer_achat(cash, 30)   # Espèces : 30.00€ payé
effectuer_achat(pp, 99.99)  # PayPal (alice@mail.com) : 99.99€ débité
