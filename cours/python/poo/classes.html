Introduction

Penser en objets

La programmation orientée objet (POO) est un paradigme qui organise le code autour d'objets plutôt que de fonctions isolées. Un objet regroupe des données (attributs) et des comportements (méthodes) dans une seule entité cohérente. En Python, tout est un objet : les entiers, les chaînes de caractères, les listes, les fonctions elles-mêmes.

Une classe est un plan de construction (un « moule ») qui décrit la structure et le comportement d'un type d'objet. Créer un objet à partir d'une classe s'appelle l'instanciation. On peut créer autant d'instances que nécessaire à partir d'une même classe, chacune ayant ses propres données.

Analogie : une classe Voiture décrit ce qu'est une voiture (marque, couleur, vitesse, démarrer, freiner). Chaque voiture réelle — votre Peugeot bleue, la Tesla rouge du voisin — est une instance de cette classe, avec ses propres valeurs d'attributs.

Ce cours couvre les fondamentaux de la création de classes en Python : la syntaxe, le constructeur __init__, les attributs d'instance et de classe, les méthodes, et le rôle essentiel du paramètre self.

Créer et utiliser des classes Python

Définir une classe

En Python, on définit une classe avec le mot-clé class. Par convention, le nom d'une classe utilise le PascalCase (première lettre de chaque mot en majuscule). Le corps de la classe contient des méthodes (fonctions définies à l'intérieur de la classe).

class Animal:
    """Représente un animal avec un nom et un son."""

    def parler(self):
        return "..."

# Instanciation
mon_animal = Animal()
print(type(mon_animal))  # <class '__main__.Animal'>
print(isinstance(mon_animal, Animal))  # True

L'appel Animal() crée une nouvelle instance de la classe. La variable mon_animal contient une référence vers cet objet en mémoire.

Le constructeur __init__

La méthode spéciale __init__ est le constructeur de la classe. Elle est appelée automatiquement à chaque création d'instance et permet d'initialiser les attributs de l'objet.

class Chien:
    def __init__(self, nom, race, age):
        self.nom = nom      # attribut d'instance
        self.race = race    # attribut d'instance
        self.age = age      # attribut d'instance

    def aboyer(self):
        return f"{self.nom} fait : Wouf !"

# Création d'instances
rex = Chien("Rex", "Berger allemand", 5)
luna = Chien("Luna", "Labrador", 3)

print(rex.nom)       # Rex
print(luna.aboyer())  # Luna fait : Wouf !

Chaque instance possède ses propres copies des attributs définis dans __init__. Modifier rex.age n'affecte pas luna.age.

Le paramètre self

self est une référence vers l'instance courante. Il est toujours le premier paramètre de chaque méthode d'instance, mais on ne le passe pas explicitement lors de l'appel — Python le fournit automatiquement.

class Compteur:
    def __init__(self):
        self.valeur = 0

    def incrementer(self):
        self.valeur += 1

    def afficher(self):
        print(f"Valeur : {self.valeur}")

c = Compteur()
c.incrementer()    # Python appelle en réalité Compteur.incrementer(c)
c.incrementer()
c.afficher()       # Valeur : 2

Important : self n'est pas un mot réservé en Python — c'est une convention universelle. Techniquement, vous pourriez utiliser un autre nom, mais ne le faites jamais : tout le monde s'attend à lire self.

Attributs d'instance vs attributs de classe

Il existe deux types d'attributs :

  • Attributs d'instance — définis avec self.attribut dans __init__ (ou une autre méthode), propres à chaque instance
  • Attributs de classe — définis directement dans le corps de la classe, partagés par toutes les instances
class Etudiant:
    # Attribut de classe — partagé par toutes les instances
    universite = "Sorbonne"
    nombre_etudiants = 0

    def __init__(self, nom, filiere):
        # Attributs d'instance — propres à chaque étudiant
        self.nom = nom
        self.filiere = filiere
        Etudiant.nombre_etudiants += 1  # on modifie l'attribut de classe

e1 = Etudiant("Alice", "Informatique")
e2 = Etudiant("Bob", "Mathématiques")

print(e1.universite)             # Sorbonne (accès via l'instance)
print(Etudiant.universite)       # Sorbonne (accès via la classe)
print(Etudiant.nombre_etudiants) # 2

Attention : si vous écrivez e1.universite = "MIT", cela crée un attribut d'instance local à e1 qui masque l'attribut de classe. Les autres instances ne sont pas affectées.

Méthodes d'instance

Les méthodes d'instance sont des fonctions définies dans la classe qui prennent self comme premier paramètre. Elles peuvent accéder et modifier les attributs de l'instance.

class Rectangle:
    def __init__(self, largeur, hauteur):
        self.largeur = largeur
        self.hauteur = hauteur

    def aire(self):
        return self.largeur * self.hauteur

    def perimetre(self):
        return 2 * (self.largeur + self.hauteur)

    def est_carre(self):
        return self.largeur == self.hauteur

    def redimensionner(self, facteur):
        self.largeur *= facteur
        self.hauteur *= facteur

r = Rectangle(4, 6)
print(r.aire())         # 24
print(r.perimetre())    # 20
print(r.est_carre())    # False
r.redimensionner(2)
print(r.aire())         # 96

Méthodes de classe et méthodes statiques

Python offre deux autres types de méthodes via des décorateurs :

class Date:
    def __init__(self, jour, mois, annee):
        self.jour = jour
        self.mois = mois
        self.annee = annee

    @classmethod
    def depuis_chaine(cls, chaine_date):
        """Constructeur alternatif à partir d'une chaîne 'JJ-MM-AAAA'."""
        jour, mois, annee = map(int, chaine_date.split("-"))
        return cls(jour, mois, annee)  # cls = la classe Date

    @staticmethod
    def est_bissextile(annee):
        """Ne dépend ni de l'instance ni de la classe."""
        return annee % 4 == 0 and (annee % 100 != 0 or annee % 400 == 0)

    def afficher(self):
        return f"{self.jour:02d}/{self.mois:02d}/{self.annee}"

# Utilisation
d1 = Date(15, 3, 2024)
d2 = Date.depuis_chaine("25-12-2023")  # constructeur alternatif

print(d1.afficher())               # 15/03/2024
print(d2.afficher())               # 25/12/2023
print(Date.est_bissextile(2024))   # True
  • @classmethod — reçoit la classe (cls) en premier paramètre. Utile pour créer des constructeurs alternatifs
  • @staticmethod — ne reçoit ni self ni cls. C'est une fonction utilitaire logiquement liée à la classe

Un exemple complet

Voici une classe plus élaborée illustrant tous les concepts vus ensemble :

class CompteBancaire:
    """Modélise un compte bancaire simplifié."""

    taux_interet = 0.02  # attribut de classe : 2%
    _nombre_comptes = 0

    def __init__(self, titulaire, solde_initial=0):
        self.titulaire = titulaire
        self.solde = solde_initial
        self.historique = []
        CompteBancaire._nombre_comptes += 1

    def deposer(self, montant):
        if montant <= 0:
            raise ValueError("Le montant doit être positif")
        self.solde += montant
        self.historique.append(f"+{montant}€")

    def retirer(self, montant):
        if montant > self.solde:
            raise ValueError("Solde insuffisant")
        self.solde -= montant
        self.historique.append(f"-{montant}€")

    def appliquer_interets(self):
        interets = self.solde * self.taux_interet
        self.solde += interets
        self.historique.append(f"Intérêts: +{interets:.2f}€")

    @classmethod
    def get_nombre_comptes(cls):
        return cls._nombre_comptes

    def __str__(self):
        return f"Compte de {self.titulaire} : {self.solde:.2f}€"

# Utilisation
compte = CompteBancaire("Alice", 1000)
compte.deposer(500)
compte.retirer(200)
compte.appliquer_interets()
print(compte)              # Compte de Alice : 1326.00€
print(compte.historique)   # ['+500€', '-200€', 'Intérêts: +26.00€']

Question 1 / 6

Quelle méthode spéciale est appelée automatiquement lors de la création d'une instance ?

__init__ est le constructeur en Python. Il est appelé automatiquement après la création de l'objet (par __new__) pour initialiser ses attributs.

Question 2 / 6

Que représente le paramètre self dans une méthode d'instance ?

self est une référence vers l'instance courante. Il permet d'accéder aux attributs et méthodes de l'objet. Ce n'est pas un mot réservé, mais une convention universelle.

Question 3 / 6

Quelle est la différence entre un attribut de classe et un attribut d'instance ?

Les attributs de classe sont définis dans le corps de la classe et partagés par toutes les instances. Les attributs d'instance sont définis via self dans __init__ et appartiennent à chaque objet individuellement.

Question 4 / 6

Quel décorateur permet de créer un constructeur alternatif qui reçoit la classe en premier paramètre ?

@classmethod reçoit la classe (cls) en premier paramètre et peut l'utiliser pour créer de nouvelles instances. C'est le pattern standard pour les constructeurs alternatifs en Python.

Question 5 / 6

Que retourne type(mon_objet) si mon_objet = MaClasse() ?

type() retourne la classe de l'objet. Pour une instance de MaClasse définie dans le module principal, cela donne <class '__main__.MaClasse'>.

Question 6 / 6

Quelle convention de nommage utilise-t-on pour les noms de classes en Python ?

La PEP 8 recommande le PascalCase pour les noms de classes : chaque mot commence par une majuscule, sans underscore. Les variables et fonctions utilisent le snake_case.

Exercice 1 — Classe Personne

Créez une classe Personne avec les attributs nom, prenom et age. Ajoutez une méthode se_presenter() qui retourne une chaîne du type "Je m'appelle Alice Dupont et j'ai 25 ans." et une méthode est_majeur() qui retourne True si la personne a 18 ans ou plus.

Voir la solution 
class Personne:
    def __init__(self, nom, prenom, age):
        self.nom = nom
        self.prenom = prenom
        self.age = age

    def se_presenter(self):
        return f"Je m'appelle {self.prenom} {self.nom} et j'ai {self.age} ans."

    def est_majeur(self):
        return self.age >= 18

# Test
p = Personne("Dupont", "Alice", 25)
print(p.se_presenter())  # Je m'appelle Alice Dupont et j'ai 25 ans.
print(p.est_majeur())    # True

Exercice 2 — Classe Cercle

Créez une classe Cercle qui prend un rayon en paramètre. Ajoutez les méthodes aire() et perimetre(). Utilisez math.pi. Ajoutez un attribut de classe nombre_cercles qui compte le nombre d'instances créées.

Voir la solution 
import math

class Cercle:
    nombre_cercles = 0

    def __init__(self, rayon):
        self.rayon = rayon
        Cercle.nombre_cercles += 1

    def aire(self):
        return math.pi * self.rayon ** 2

    def perimetre(self):
        return 2 * math.pi * self.rayon

# Test
c1 = Cercle(5)
c2 = Cercle(3)
print(f"Aire : {c1.aire():.2f}")         # Aire : 78.54
print(f"Périmètre : {c1.perimetre():.2f}")  # Périmètre : 31.42
print(Cercle.nombre_cercles)              # 2

Exercice 3 — Classe Livre avec classmethod

Créez une classe Livre avec les attributs titre, auteur et pages. Ajoutez une méthode de classe depuis_dict(cls, d) qui crée un livre à partir d'un dictionnaire {"titre": "...", "auteur": "...", "pages": 300}. Ajoutez une méthode est_long() qui retourne True si le livre dépasse 500 pages.

Voir la solution 
class Livre:
    def __init__(self, titre, auteur, pages):
        self.titre = titre
        self.auteur = auteur
        self.pages = pages

    @classmethod
    def depuis_dict(cls, d):
        return cls(d["titre"], d["auteur"], d["pages"])

    def est_long(self):
        return self.pages > 500

# Test
data = {"titre": "Les Misérables", "auteur": "Victor Hugo", "pages": 1900}
livre = Livre.depuis_dict(data)
print(f"{livre.titre} par {livre.auteur}")  # Les Misérables par Victor Hugo
print(livre.est_long())                     # True

Exercice 4 — Classe Inventaire

Créez une classe Inventaire qui gère une liste de produits. Chaque produit est un dictionnaire {"nom": "...", "prix": ..., "quantite": ...}. Implémentez les méthodes ajouter(produit), supprimer(nom), valeur_totale() (somme prix * quantité) et rechercher(nom).

Voir la solution 
class Inventaire:
    def __init__(self):
        self.produits = []

    def ajouter(self, produit):
        self.produits.append(produit)

    def supprimer(self, nom):
        self.produits = [p for p in self.produits if p["nom"] != nom]

    def valeur_totale(self):
        return sum(p["prix"] * p["quantite"] for p in self.produits)

    def rechercher(self, nom):
        for p in self.produits:
            if p["nom"].lower() == nom.lower():
                return p
        return None

# Test
inv = Inventaire()
inv.ajouter({"nom": "Clavier", "prix": 49.99, "quantite": 10})
inv.ajouter({"nom": "Souris", "prix": 29.99, "quantite": 25})
print(inv.valeur_totale())       # 1249.65
print(inv.rechercher("souris"))  # {'nom': 'Souris', 'prix': 29.99, 'quantite': 25}
inv.supprimer("Clavier")
print(inv.valeur_totale())       # 749.75

Exercice 5 — Classe Chronometre

Créez une classe Chronometre qui possède les méthodes demarrer(), arreter() et duree(). Utilisez le module time. La méthode duree() retourne le temps écoulé en secondes. Ajoutez une méthode statique formater(secondes) qui convertit un nombre de secondes en chaîne "Xm Ys".

Voir la solution 
import time

class Chronometre:
    def __init__(self):
        self._debut = None
        self._fin = None

    def demarrer(self):
        self._debut = time.time()
        self._fin = None

    def arreter(self):
        if self._debut is None:
            raise RuntimeError("Le chronomètre n'a pas été démarré")
        self._fin = time.time()

    def duree(self):
        if self._debut is None:
            return 0.0
        fin = self._fin if self._fin else time.time()
        return fin - self._debut

    @staticmethod
    def formater(secondes):
        minutes = int(secondes) // 60
        secs = int(secondes) % 60
        return f"{minutes}m {secs}s"

# Test
chrono = Chronometre()
chrono.demarrer()
time.sleep(2)
chrono.arreter()
d = chrono.duree()
print(f"Durée : {d:.2f}s")                # Durée : 2.00s
print(Chronometre.formater(125))           # 2m 5s
