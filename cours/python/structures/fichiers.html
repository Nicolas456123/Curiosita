Introduction

Persister les données sur le disque

Les variables existent seulement en mémoire — elles disparaissent quand le programme s'arrête. Pour sauvegarder des données, il faut les écrire dans des fichiers. Python rend cette opération simple et sûre grâce au mot-clé with.

Bonne pratique : toujours utiliser with open(...) as f: qui ferme automatiquement le fichier, même en cas d'erreur. Ne jamais utiliser open() sans with dans du code de production.

02

Le cours

open(), with, lecture et écriture

Lire un fichier

Python
# Lire tout le contenu d'un coup
with open("data.txt", "r", encoding="utf-8") as f:
    contenu = f.read()
    print(contenu)

# Lire ligne par ligne (efficace pour gros fichiers)
with open("data.txt", "r") as f:
    for ligne in f:
        print(ligne.strip())  # strip() enlève le \n

# Lire toutes les lignes dans une liste
with open("data.txt", "r") as f:
    lignes = f.readlines()
    print(len(lignes), "lignes lues")

Écrire dans un fichier

Python
# Écrire (écrase le fichier existant)
with open("resultat.txt", "w", encoding="utf-8") as f:
    f.write("Première ligne\n")
    f.write("Deuxième ligne\n")

# Ajouter à la fin (append)
with open("journal.txt", "a") as f:
    f.write("Nouvelle entrée\n")

# Écrire plusieurs lignes
noms = ["Alice", "Bob", "Clara"]
with open("noms.txt", "w") as f:
    for nom in noms:
        f.write(nom + "\n")

Travailler avec CSV

Python
import csv

# Lire un CSV
with open("notes.csv", "r") as f:
    lecteur = csv.reader(f)
    for ligne in lecteur:
        print(ligne)  # ['Alice', '15', '18']

# Écrire un CSV
donnees = [["Nom", "Note"], ["Alice", 15], ["Bob", 12]]
with open("export.csv", "w", newline="") as f:
    ecrivain = csv.writer(f)
    ecrivain.writerows(donnees)

Les modes d'ouverture

La fonction open() accepte différents modes qui déterminent ce que vous pouvez faire avec le fichier.

Python
# Modes d'ouverture principaux
# "r"  → lecture seule (défaut) — erreur si fichier absent
# "w"  → écriture — ÉCRASE le contenu existant
# "a"  → ajout — écrit à la fin du fichier
# "x"  → création exclusive — erreur si fichier existe
# "r+" → lecture ET écriture
# "b"  → mode binaire (ajouté aux autres : "rb", "wb")

# Vérifier si un fichier existe avant de l'ouvrir
import os

if os.path.exists("data.txt"):
    with open("data.txt", "r") as f:
        contenu = f.read()
else:
    print("Le fichier n'existe pas")

# Mode "x" — création sécurisée
try:
    with open("nouveau.txt", "x") as f:
        f.write("Contenu initial")
except FileExistsError:
    print("Le fichier existe déjà")

Travailler avec JSON

Le format JSON est très utilisé pour stocker des données structurées. Python le gère nativement avec le module json.

Python
import json

# Écrire des données Python en JSON
donnees = {
    "nom": "Alice",
    "age": 25,
    "langages": ["Python", "JavaScript"],
    "actif": True
}

with open("profil.json", "w", encoding="utf-8") as f:
    json.dump(donnees, f, indent=2, ensure_ascii=False)

# Lire un fichier JSON
with open("profil.json", "r", encoding="utf-8") as f:
    profil = json.load(f)

print(profil["nom"])      # Alice
print(profil["langages"]) # ['Python', 'JavaScript']

indent et ensure_ascii : indent=2 rend le JSON lisible avec une indentation de 2 espaces. ensure_ascii=False permet les caractères spéciaux comme les accents.

Gestion des chemins avec pathlib

Le module pathlib (Python 3.4+) offre une manière moderne et élégante de manipuler les chemins de fichiers.

Python
from pathlib import Path

# Créer un chemin
dossier = Path("donnees")
fichier = dossier / "resultats.txt"  # donnees/resultats.txt

# Vérifications
print(fichier.exists())    # True/False
print(fichier.is_file())   # True si c'est un fichier
print(fichier.suffix)      # .txt
print(fichier.stem)        # resultats (sans extension)

# Lire/écrire en une ligne
fichier.write_text("Bonjour !", encoding="utf-8")
contenu = fichier.read_text(encoding="utf-8")

# Lister les fichiers d'un dossier
for f in Path(".").glob("*.py"):
    print(f.name)

Pièges courants

with open("fichier.txt", "r", encoding="utf-8") as f:

# Si vous voulez ajouter, utilisez "a" !

with open("data.csv", "w", newline="") as f:
    ecrivain = csv.writer(f)

# PIÈGE 4 : Oublier de fermer le fichier

# Utilisez TOUJOURS with pour la fermeture automatique

Bonnes pratiques

  • Toujours with : utilisez with open(...) as f: pour garantir la fermeture du fichier.
  • Toujours encoding : spécifiez encoding="utf-8" pour éviter les problèmes d'encodage.
  • pathlib : préférez pathlib.Path à os.path pour la manipulation de chemins.
  • JSON pour les données : utilisez JSON plutôt que des fichiers texte bruts pour les données structurées.
  • Gestion d'erreurs : encadrez les opérations fichier avec try/except pour gérer les erreurs (FileNotFoundError, PermissionError).

Quiz

Testez votre compréhension

📁 Fichiers — QuizScore : 0 / 6

Question 1 / 6

Quel mode d'ouverture ajoute du texte à la fin d'un fichier existant ?

Le mode "a" (append) ajoute à la fin. "w" écrase le fichier. "r" lit seulement. "x" crée un nouveau fichier (erreur s'il existe).

Question 2 / 6

Pourquoi utiliser with open(...) plutôt que open() seul ?

with est un context manager qui garantit la fermeture du fichier, même si une exception survient. C'est la bonne pratique universelle.

Question 3 / 6

Quelle méthode lit tout le contenu d'un fichier en une seule chaîne ?

.read() lit tout en une chaîne. .readline() lit une seule ligne. .readlines() lit toutes les lignes dans une liste.

Question 4 / 6

Quelle fonction du module json écrit des données Python dans un fichier ?

json.dump(data, file) écrit dans un fichier. json.dumps(data) retourne une chaîne JSON (le « s » signifie « string »). Pour lire : json.load(file) et json.loads(string).

Question 5 / 6

Que fait le mode "x" dans open("fichier.txt", "x") ?

Le mode "x" (exclusive creation) crée un nouveau fichier. Si le fichier existe déjà, il lève une FileExistsError. C'est utile pour éviter d'écraser des données.

Question 6 / 6

Pourquoi utiliser newline="" avec csv.writer sur Windows ?

Sur Windows, sans newline="", le module csv ajoute un retour chariot supplémentaire, ce qui crée des lignes vides entre chaque ligne de données.

Exercice 1 --- Journal de bord

Créez un programme qui demande à l'utilisateur de saisir une entrée de journal, puis l'ajoute avec la date et l'heure actuelles dans un fichier journal.txt. Le programme doit afficher les entrées existantes au démarrage.

Voir la solution 
from datetime import datetime
from pathlib import Path

fichier = Path("journal.txt")

# Afficher les entrées existantes
if fichier.exists():
    print("=== Entrées précédentes ===")
    print(fichier.read_text(encoding="utf-8"))
    print("=" * 30)
else:
    print("Nouveau journal créé.")

# Nouvelle entrée
entree = input("Nouvelle entrée : ").strip()
if entree:
    horodatage = datetime.now().strftime("%Y-%m-%d %H:%M")
    with open(fichier, "a", encoding="utf-8") as f:
        f.write(f"[{horodatage}] {entree}\n")
    print("Entrée ajoutée !")

Exercice 2 --- Convertisseur CSV vers JSON

Écrivez un programme qui lit un fichier CSV avec des colonnes « nom », « age », « ville » et le convertit en fichier JSON. Gérez le cas où le fichier CSV n'existe pas.

Voir la solution 
import csv
import json

try:
    with open("personnes.csv", "r", encoding="utf-8") as f:
        lecteur = csv.DictReader(f)
        personnes = list(lecteur)

    # Convertir l'âge en entier
    for p in personnes:
        p["age"] = int(p["age"])

    with open("personnes.json", "w", encoding="utf-8") as f:
        json.dump(personnes, f, indent=2, ensure_ascii=False)

    print(f"{len(personnes)} personnes converties en JSON.")

except FileNotFoundError:
    print("Erreur : le fichier personnes.csv n'existe pas.")

Exercice 3 --- Compteur de mots dans un fichier

Créez un programme qui lit un fichier texte et affiche : le nombre de lignes, le nombre de mots, le nombre de caractères, et les 5 mots les plus fréquents.

Voir la solution 
from collections import Counter

fichier = "texte.txt"

try:
    with open(fichier, "r", encoding="utf-8") as f:
        contenu = f.read()

    lignes = contenu.split("\n")
    mots = contenu.split()
    caracteres = len(contenu)

    print(f"Lignes     : {len(lignes)}")
    print(f"Mots       : {len(mots)}")
    print(f"Caractères : {caracteres}")

    # Top 5 des mots les plus fréquents
    compteur = Counter(mot.lower().strip(".,!?;:") for mot in mots)
    print("\nTop 5 des mots :")
    for mot, freq in compteur.most_common(5):
        print(f"  {mot}: {freq} fois")

except FileNotFoundError:
    print(f"Fichier '{fichier}' introuvable.")

Exercice 4 --- Configuration JSON

Créez un programme qui gère un fichier de configuration JSON. Au premier lancement, créez un fichier avec des valeurs par défaut. Aux lancements suivants, lisez la config, affichez-la et permettez à l'utilisateur de modifier une valeur.

Voir la solution 
import json
from pathlib import Path

CONFIG_FILE = Path("config.json")

CONFIG_DEFAUT = {
    "theme": "sombre",
    "langue": "fr",
    "taille_police": 14,
    "notifications": True
}

def charger_config():
    if CONFIG_FILE.exists():
        with open(CONFIG_FILE, "r", encoding="utf-8") as f:
            return json.load(f)
    return CONFIG_DEFAUT.copy()

def sauvegarder_config(config):
    with open(CONFIG_FILE, "w", encoding="utf-8") as f:
        json.dump(config, f, indent=2, ensure_ascii=False)

# Programme principal
config = charger_config()
sauvegarder_config(config)

print("Configuration actuelle :")
for cle, valeur in config.items():
    print(f"  {cle}: {valeur}")

cle = input("\nClé à modifier (ou Entrée pour quitter) : ").strip()
if cle and cle in config:
    nouvelle_valeur = input(f"Nouvelle valeur pour '{cle}' : ").strip()
    # Conversion de type simple
    if nouvelle_valeur.lower() in ("true", "false"):
        config[cle] = nouvelle_valeur.lower() == "true"
    elif nouvelle_valeur.isdigit():
        config[cle] = int(nouvelle_valeur)
    else:
        config[cle] = nouvelle_valeur
    sauvegarder_config(config)
    print("Configuration mise à jour !")
