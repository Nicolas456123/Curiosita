Introduction

La révolution de l'apprentissage profond

Le Deep Learning (apprentissage profond) désigne l'utilisation de réseaux de neurones comportant de nombreuses couches (dizaines, voire centaines). Cette profondeur permet au modèle d'apprendre des représentations hiérarchiques des données : les premières couches détectent des motifs simples (bords, textures), et les couches profondes combinent ces motifs en concepts de plus en plus abstraits (visages, objets, phrases).

L'essor du deep learning date de 2012, quand le réseau AlexNet a remporté le concours ImageNet avec une avance spectaculaire. Depuis, les architectures profondes dominent la vision par ordinateur, le traitement du langage naturel, la reconnaissance vocale, et même la génération de contenu. Trois facteurs expliquent ce succès : les datasets massifs, la puissance des GPU, et les avancées algorithmiques (batch normalization, skip connections, attention).

Profondeur vs largeur : un réseau profond avec des couches étroites est souvent plus expressif qu'un réseau large avec peu de couches. La profondeur permet de composer des transformations, créant des représentations compactes et puissantes.

Les architectures fondamentales du Deep Learning

Réseaux convolutifs (CNN)

Les réseaux de neurones convolutifs (Convolutional Neural Networks) sont spécialement conçus pour traiter des données structurées en grille, comme les images. Au lieu de connecter chaque neurone à tous les pixels, un CNN utilise des filtres (kernels) de petite taille (3x3, 5x5) qui glissent sur l'image pour détecter des motifs locaux.

Une couche de convolution applique plusieurs filtres, chacun produisant une carte de caractéristiques (feature map). Les couches de pooling (max-pooling ou average-pooling) réduisent la taille spatiale tout en conservant les informations importantes. En empilant convolution, activation (ReLU) et pooling, le réseau construit une hiérarchie de features de plus en plus abstraites.

Python — PyTorch
import torch.nn as nn

class CNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(1, 32, kernel_size=3, padding=1),  # 28x28→28x28
            nn.ReLU(),
            nn.MaxPool2d(2),                               # 28x28→14x14
            nn.Conv2d(32, 64, kernel_size=3, padding=1), # 14x14→14x14
            nn.ReLU(),
            nn.MaxPool2d(2),                               # 14x14→7x7
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(64 * 7 * 7, 128),
            nn.ReLU(),
            nn.Linear(128, 10)
        )

    def forward(self, x):
        x = self.features(x)
        return self.classifier(x)

Réseaux récurrents (RNN et LSTM)

Les réseaux récurrents (RNN) traitent des séquences en maintenant un état caché qui se met à jour à chaque pas de temps. À chaque instant t, le réseau reçoit l'entrée xₜ et l'état caché hₜ₋₁, puis calcule un nouvel état hₜ. Cela permet de capturer des dépendances temporelles dans les séries chronologiques, le texte ou l'audio.

Les RNN simples souffrent du problème de gradient évanescent : les informations lointaines sont oubliées. Les LSTM (Long Short-Term Memory) résolvent ce problème grâce à un mécanisme de portes (forget gate, input gate, output gate) qui contrôle le flux d'information. Les GRU (Gated Recurrent Unit) sont une variante simplifiée avec deux portes au lieu de trois.

Python — PyTorch
# LSTM pour la classification de séquences
class LSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, n_classes):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.lstm = nn.LSTM(embed_dim, hidden_dim, batch_first=True)
        self.fc = nn.Linear(hidden_dim, n_classes)

    def forward(self, x):
        embeds = self.embedding(x)          # (batch, seq_len, embed_dim)
        output, (h_n, c_n) = self.lstm(embeds)
        return self.fc(h_n.squeeze(0))    # Utiliser le dernier état caché

Transformers et mécanisme d'attention

L'architecture Transformer, introduite en 2017, a remplacé les RNN pour la plupart des tâches de séquences. Son innovation clé est le mécanisme d'auto-attention (self-attention), qui permet à chaque élément de la séquence de « regarder » tous les autres éléments directement, sans passer par un état caché séquentiel.

L'attention calcule pour chaque position un vecteur Query (Q), Key (K) et Value (V). Le score d'attention entre deux positions est le produit scalaire de leurs vecteurs Q et K, normalisé par la racine de la dimension. Les scores sont transformés en probabilités via softmax, puis utilisés pour pondérer les vecteurs V. Le multi-head attention répète ce processus avec plusieurs « têtes » en parallèle, capturant différents types de relations.

Avantage du Transformer : contrairement aux RNN qui traitent les tokens séquentiellement, le Transformer traite toute la séquence en parallèle. Cela permet un entraînement beaucoup plus rapide sur GPU et capture les dépendances longue distance sans gradient évanescent.

Entraînement sur GPU

Les GPU (Graphics Processing Units) sont essentiels au deep learning car ils excellents dans le calcul matriciel parallèle. Un GPU moderne possède des milliers de coeurs, capables d'effectuer des multiplications matricielles massives simultanément. Les frameworks comme PyTorch et TensorFlow permettent de déplacer les calculs sur GPU avec une seule ligne de code.

L'entraînement distribué sur plusieurs GPU (data parallelism, model parallelism) permet de traiter des modèles encore plus grands. Les TPU (Tensor Processing Units) de Google sont des puces spécialisées encore plus rapides pour les opérations tensorielles. Le choix du matériel et l'optimisation mémoire (mixed precision, gradient checkpointing) sont cruciaux pour entraîner les modèles modernes.

Transfer learning et fine-tuning

Le transfer learning consiste à réutiliser un modèle pré-entraîné sur un grand dataset (comme ImageNet ou un large corpus de texte) et à l'adapter à une tâche spécifique. Les couches profondes du modèle ont déjà appris des représentations génériques (bords, textures, structures syntaxiques) qui sont utiles pour de nombreuses tâches.

Le fine-tuning est la technique la plus courante : on remplace la dernière couche du modèle pré-entraîné par une nouvelle couche adaptée à notre tâche, puis on ré-entraîne tout le réseau (ou seulement les dernières couches) avec un taux d'apprentissage faible. Cela permet d'obtenir d'excellents résultats même avec peu de données, car le modèle part d'une base solide.

Python — Transfer Learning
from torchvision import models

# Charger ResNet18 pré-entraîné sur ImageNet
modele = models.resnet18(pretrained=True)

# Geler les couches pré-entraînées
for param in modele.parameters():
    param.requires_grad = False

# Remplacer la dernière couche pour notre tâche (5 classes)
modele.fc = nn.Linear(modele.fc.in_features, 5)

# Seule la nouvelle couche sera entraînée
optimiseur = torch.optim.Adam(modele.fc.parameters(), lr=0.001)

Architectures modernes

Les architectures de deep learning évoluent rapidement. ResNet (2015) a introduit les connexions résiduelles (skip connections), permettant d'entraîner des réseaux de plus de 100 couches sans dégradation. EfficientNet optimise simultanément la profondeur, la largeur et la résolution du réseau.

Les Vision Transformers (ViT) appliquent l'architecture Transformer aux images en les découpant en patches. Les modèles de diffusion (Stable Diffusion, DALL-E) génèrent des images en partant de bruit aléatoire et en le « dénoisifiant » progressivement. Les LLM (Large Language Models) comme GPT utilisent des Transformers massivement paramétrés (des milliards de paramètres) pour la génération de texte.

Bonnes pratiques d'entraînement

L'entraînement d'un modèle profond demande de la rigueur. Il faut surveiller les courbes de perte (train et validation) pour détecter le surapprentissage. L'early stopping arrête l'entraînement quand la perte de validation cesse de diminuer. L'augmentation de données (rotations, recadrages, retournements) augmente artificiellement la diversité du jeu d'entraînement.

Le learning rate scheduling réduit le taux d'apprentissage au cours de l'entraînement (cosine annealing, step decay, warmup). La taille du batch influence la vitesse et la qualité de convergence : des batchs plus grands accélèrent l'entraînement sur GPU mais peuvent nuire à la généralisation. Un batch de 32 à 256 est un bon point de départ.

Exercice 1 — CNN pour MNIST

Construisez un CNN avec deux couches de convolution (32 puis 64 filtres, kernel 3x3), max-pooling, et deux couches denses pour classifier les chiffres MNIST. Entraînez pendant 5 epochs et mesurez la précision sur le jeu de test.

Voir la solution 
import torch
import torch.nn as nn
from torchvision import datasets, transforms
from torch.utils.data import DataLoader

transform = transforms.Compose([transforms.ToTensor(), transforms.Normalize((0.5,), (0.5,))])
train_data = datasets.MNIST('.', train=True, download=True, transform=transform)
test_data = datasets.MNIST('.', train=False, transform=transform)
train_loader = DataLoader(train_data, batch_size=64, shuffle=True)
test_loader = DataLoader(test_data, batch_size=64)

class CNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv = nn.Sequential(
            nn.Conv2d(1, 32, 3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
            nn.Conv2d(32, 64, 3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
        )
        self.fc = nn.Sequential(
            nn.Flatten(), nn.Linear(64*7*7, 128), nn.ReLU(), nn.Linear(128, 10))

    def forward(self, x): return self.fc(self.conv(x))

model = CNN()
optim = torch.optim.Adam(model.parameters(), lr=0.001)
critere = nn.CrossEntropyLoss()

for epoch in range(5):
    for images, labels in train_loader:
        loss = critere(model(images), labels)
        optim.zero_grad(); loss.backward(); optim.step()

# Test
correct = sum((model(img).argmax(1) == lab).sum().item() for img, lab in test_loader)
print(f"Précision test : {correct/len(test_data):.1%}")  # ~99%

Exercice 4 — Implémentation du self-attention

Implémentez le mécanisme de self-attention scaled dot-product avec NumPy. Prenez Q, K, V comme des matrices aléatoires de taille (4, 8) (4 tokens, dimension 8) et calculez la sortie de l'attention.

Voir la solution 
import numpy as np

def softmax(x, axis=-1):
    exp_x = np.exp(x - np.max(x, axis=axis, keepdims=True))
    return exp_x / exp_x.sum(axis=axis, keepdims=True)

def self_attention(Q, K, V):
    d_k = K.shape[-1]
    # Scores d'attention : QK^T / sqrt(d_k)
    scores = Q @ K.T / np.sqrt(d_k)
    print(f"Scores d'attention (4x4) :\n{np.round(scores, 2)}")

    # Probabilités d'attention
    attn_weights = softmax(scores)
    print(f"\nPoids d'attention :\n{np.round(attn_weights, 2)}")

    # Sortie : pondération des valeurs
    output = attn_weights @ V
    return output, attn_weights

np.random.seed(42)
Q = np.random.randn(4, 8)
K = np.random.randn(4, 8)
V = np.random.randn(4, 8)

output, weights = self_attention(Q, K, V)
print(f"\nSortie shape : {output.shape}")  # (4, 8)
# Chaque token est une combinaison pondérée de toutes les valeurs

Exercice 5 — Comparaison CPU vs GPU

Mesurez le temps d'exécution d'une multiplication matricielle de taille (4096, 4096) sur CPU et sur GPU avec PyTorch. Calculez le facteur d'accélération. Répétez avec différentes tailles pour tracer la courbe speedup en fonction de la taille.

Voir la solution 
import torch, time

def benchmark(size, device, n_iter=10):
    A = torch.randn(size, size, device=device)
    B = torch.randn(size, size, device=device)
    if device.type == 'cuda':
        torch.cuda.synchronize()

    start = time.time()
    for _ in range(n_iter):
        C = A @ B
    if device.type == 'cuda':
        torch.cuda.synchronize()
    return (time.time() - start) / n_iter

sizes = [256, 512, 1024, 2048, 4096]
for s in sizes:
    t_cpu = benchmark(s, torch.device('cpu'))
    if torch.cuda.is_available():
        t_gpu = benchmark(s, torch.device('cuda'))
        speedup = t_cpu / t_gpu
        print(f"Taille {s:5d} → CPU: {t_cpu:.4f}s, GPU: {t_gpu:.4f}s, x{speedup:.0f}")
    else:
        print(f"Taille {s:5d} → CPU: {t_cpu:.4f}s (GPU non disponible)")

# Résultats typiques (RTX 3080) :
# Taille 4096 → CPU: 1.2s, GPU: 0.005s, x240
