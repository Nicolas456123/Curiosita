Introduction

Pourquoi trier est fondamental

Le tri est l'une des opérations les plus fréquentes en informatique. Trier une liste de noms par ordre alphabétique, classer des produits par prix, organiser des données pour permettre une recherche rapide — le tri est partout. C'est aussi l'un des problèmes les mieux étudiés de l'algorithmique, avec des dizaines d'algorithmes différents, chacun avec ses forces et ses faiblesses.

Borne inférieure : il a été prouvé mathématiquement qu'aucun algorithme de tri par comparaison ne peut faire mieux que O(n log n) dans le pire cas. Les tris fusion et tas atteignent cette borne optimale.

Un algorithme de tri est dit stable s'il préserve l'ordre relatif des éléments égaux, et en place s'il n'utilise que O(1) de mémoire supplémentaire. Ces propriétés influencent le choix de l'algorithme selon le contexte.

Le cours

Les grands algorithmes de tri

Tri par insertion

Le tri par insertion construit le résultat trié élément par élément, en insérant chaque nouvel élément à sa place dans la partie déjà triée. Simple et efficace pour les petites listes ou les listes presque triées.

Python
def tri_insertion(liste):
    for i in range(1, len(liste)):
        cle = liste[i]
        j = i - 1
        while j >= 0 and liste[j] > cle:
            liste[j + 1] = liste[j]  # Décaler vers la droite
            j -= 1
        liste[j + 1] = cle  # Insérer à la bonne position
    return liste

# Complexité : O(n²) pire cas, O(n) meilleur cas (liste triée)
# Stable : oui | En place : oui

Tri par sélection

Le tri par sélection trouve le minimum de la partie non triée et le place à sa position définitive. Simple mais toujours en O(n²), même pour une liste déjà triée.

Python
def tri_selection(liste):
    n = len(liste)
    for i in range(n):
        i_min = i
        for j in range(i + 1, n):
            if liste[j] < liste[i_min]:
                i_min = j
        liste[i], liste[i_min] = liste[i_min], liste[i]
    return liste

# Complexité : O(n²) dans tous les cas
# Stable : non | En place : oui

Tri fusion (Merge Sort)

Le tri fusion divise la liste en deux moitiés, trie récursivement chaque moitié, puis fusionne les deux moitiés triées. C'est un algorithme « diviser pour régner » qui garantit O(n log n) dans tous les cas.

Python
def tri_fusion(liste):
    if len(liste) <= 1:
        return liste

    milieu = len(liste) // 2
    gauche = tri_fusion(liste[:milieu])
    droite = tri_fusion(liste[milieu:])
    return fusionner(gauche, droite)

def fusionner(g, d):
    resultat = []
    i = j = 0
    while i < len(g) and j < len(d):
        if g[i] <= d[j]:
            resultat.append(g[i]); i += 1
        else:
            resultat.append(d[j]); j += 1
    resultat.extend(g[i:])
    resultat.extend(d[j:])
    return resultat

# Complexité : O(n log n) dans tous les cas
# Stable : oui | En place : non (O(n) en espace)

Tri rapide (Quick Sort)

Le tri rapide choisit un « pivot », partitionne la liste en éléments plus petits et plus grands que le pivot, puis trie récursivement. En moyenne O(n log n) et souvent le plus rapide en pratique.

Python
def tri_rapide(liste):
    if len(liste) <= 1:
        return liste

    pivot = liste[len(liste) // 2]
    gauche = [x for x in liste if x < pivot]
    milieu = [x for x in liste if x == pivot]
    droite = [x for x in liste if x > pivot]
    return tri_rapide(gauche) + milieu + tri_rapide(droite)

# Complexité : O(n log n) moyen, O(n²) pire cas
# Stable : non | En place : oui (version partition)

Tableau comparatif

Insertion : O(n²) | stable, en place, bon pour petites listes
Sélection : O(n²) | non stable, en place, peu d'échanges
Fusion : O(n log n) | stable, O(n) espace, garanti optimal
Rapide : O(n log n) moyen | non stable, en place, le plus rapide en pratique

Quiz

📊 Algorithmes de tri — Quiz Score : 0 / 5

Question 1 / 5

Quel tri est le plus efficace pour une liste presque triée ?

Le tri par insertion est en O(n) sur une liste presque triée car il ne déplace que quelques éléments. Les autres restent en O(n log n) ou O(n²).

Question 2 / 5

Quelle est la complexité du tri fusion dans le pire cas ?

Le tri fusion garantit O(n log n) dans tous les cas (meilleur, moyen et pire). C'est sa grande force par rapport au tri rapide.

Question 3 / 5

Qu'est-ce qu'un tri « stable » ?

Un tri stable conserve l'ordre relatif des éléments ayant la même clé de tri. Par exemple, si deux étudiants ont la même note, leur ordre initial est préservé.

Question 4 / 5

Le tri rapide a un pire cas en O(n²). Quand cela se produit-il ?

Si le pivot est toujours le plus petit ou le plus grand élément, la partition est maximalement déséquilibrée : un sous-tableau de taille 0 et un de taille n-1, donnant O(n²).

Question 5 / 5

Quel tri utilise la stratégie « diviser pour régner » ?

Le tri fusion et le tri rapide utilisent tous deux la stratégie « diviser pour régner ». Le tri fusion divise en deux moitiés puis fusionne ; le tri rapide partitionne autour d'un pivot.

Exercice 1 — Tri à bulles

Implémentez le tri à bulles : parcourez la liste en échangeant les éléments adjacents mal ordonnés, et recommencez jusqu'à ce qu'aucun échange ne soit nécessaire. Quelle est sa complexité ?

Voir la solution 
def tri_bulles(liste):
    n = len(liste)
    for i in range(n):
        echange = False
        for j in range(n - 1 - i):
            if liste[j] > liste[j + 1]:
                liste[j], liste[j + 1] = liste[j + 1], liste[j]
                echange = True
        if not echange:
            break
    return liste
# O(n²) pire/moyen, O(n) meilleur (déjà trié grâce au flag)

Exercice 2 — Compter les comparaisons

Modifiez le tri par insertion et le tri fusion pour qu'ils comptent et retournent le nombre de comparaisons effectuées. Testez sur une liste de 1000 éléments aléatoires.

Voir la solution 
def tri_insertion_count(liste):
    comps = 0
    for i in range(1, len(liste)):
        cle = liste[i]
        j = i - 1
        while j >= 0:
            comps += 1
            if liste[j] > cle:
                liste[j + 1] = liste[j]
                j -= 1
            else:
                break
        liste[j + 1] = cle
    return liste, comps
# Pour 1000 éléments aléatoires :
# insertion ~ 250 000 comparaisons
# fusion ~ 10 000 comparaisons
