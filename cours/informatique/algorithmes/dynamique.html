Introduction

Diviser, mémoriser, conquérir

La programmation dynamique (PD) est une technique algorithmique qui résout des problèmes complexes en les décomposant en sous-problèmes plus simples, dont les solutions sont stockées pour éviter de les recalculer. Elle s'applique quand un problème possède des sous-problèmes chevauchants et une sous-structure optimale.

Mémoïsation vs Tabulation : la mémoïsation (top-down) utilise la récursion avec un cache. La tabulation (bottom-up) remplit un tableau itérativement. Les deux donnent le même résultat mais diffèrent en style et en gestion de la mémoire.

La programmation dynamique transforme des algorithmes exponentiels en algorithmes polynomiaux. C'est l'une des techniques les plus puissantes et les plus demandées en entretien technique. Ce cours vous apprend à reconnaître les problèmes de PD et à les résoudre méthodiquement.

Principes et applications

Fibonacci : le problème classique

Le calcul de Fibonacci illustre parfaitement la différence entre une approche naïve (exponentielle) et la programmation dynamique (linéaire).

Python
# Approche naïve — O(2ⁿ) : recalcule les mêmes valeurs
def fib_naif(n):
    if n <= 1:
        return n
    return fib_naif(n - 1) + fib_naif(n - 2)

# Mémoïsation (top-down) — O(n)
def fib_memo(n, cache={}):
    if n in cache:
        return cache[n]
    if n <= 1:
        return n
    cache[n] = fib_memo(n - 1) + fib_memo(n - 2)
    return cache[n]

# Tabulation (bottom-up) — O(n)
def fib_tab(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]

Le problème du sac à dos (Knapsack)

Étant donné un sac de capacité W et n objets avec des poids et des valeurs, maximiser la valeur totale sans dépasser la capacité.

Python
def sac_a_dos(capacite, poids, valeurs, n):
    dp = [[0] * (capacite + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for w in range(capacite + 1):
            if poids[i-1] <= w:
                dp[i][w] = max(
                    dp[i-1][w],
                    dp[i-1][w - poids[i-1]] + valeurs[i-1]
                )
            else:
                dp[i][w] = dp[i-1][w]
    return dp[n][capacite]

# Exemple : capacité 7, 4 objets
poids =   [1, 3, 4, 5]
valeurs = [1, 4, 5, 7]
print(sac_a_dos(7, poids, valeurs, 4))  # 9

Plus longue sous-séquence commune (LCS)

Trouver la plus longue sous-séquence commune à deux chaînes. Une sous-séquence conserve l'ordre mais pas la contiguïté.

Python
def lcs(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]

print(lcs("ABCBDAB", "BDCAB"))  # 4 → "BCAB"

Reconnaître un problème de PD

Un problème se prête à la programmation dynamique s'il possède ces deux propriétés :

  • Sous-structure optimale : la solution optimale du problème contient les solutions optimales de ses sous-problèmes.
  • Chevauchement des sous-problèmes : les mêmes sous-problèmes sont résolus plusieurs fois dans l'approche récursive naïve.

Méthodologie : 1) Identifier l'état (quels paramètres définissent un sous-problème). 2) Écrire la relation de récurrence. 3) Définir les cas de base. 4) Choisir mémoïsation ou tabulation. 5) Optimiser l'espace si possible.

Question 1 / 5

Quelle est la complexité de Fibonacci naïf récursif ?

Sans mémoïsation, chaque appel se divise en deux, créant un arbre d'appels de taille exponentielle O(2ⁿ). Avec la PD, on passe à O(n).

Question 2 / 5

Quelle est la différence entre mémoïsation et tabulation ?

La mémoïsation part du problème complet et descend récursivement (top-down) en cachant les résultats. La tabulation remplit un tableau depuis les cas de base (bottom-up).

Question 3 / 5

Quelle propriété est nécessaire pour appliquer la programmation dynamique ?

La PD nécessite que les mêmes sous-problèmes soient résolus plusieurs fois (chevauchement) et que la solution optimale contienne les solutions optimales des sous-problèmes (sous-structure optimale).

Question 4 / 5

Quelle est la complexité du sac à dos 0/1 avec n objets et capacité W ?

Le tableau DP a n lignes et W colonnes, chaque case étant calculée en O(1). La complexité totale est donc O(n × W), ce qu'on appelle pseudo-polynomial.

Question 5 / 5

Quelle technique la PD remplace-t-elle avantageusement ?

La PD élimine les recalculs redondants de la récursion naïve en mémorisant les résultats des sous-problèmes, transformant souvent un algorithme exponentiel en polynomial.

Exercice 1 — Montée d'escalier

Vous montez un escalier de n marches. À chaque pas, vous pouvez monter 1 ou 2 marches. Combien de façons différentes de monter l'escalier ?

Voir la solution 
def monter_escalier(n):
    if n <= 2:
        return n
    dp = [0] * (n + 1)
    dp[1], dp[2] = 1, 2
    for i in range(3, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
# C'est en fait Fibonacci ! monter_escalier(5) = 8

Exercice 2 — Rendu de monnaie

Étant donné un montant et des pièces de valeurs données, trouvez le nombre minimum de pièces pour atteindre le montant.

Voir la solution 
def rendu_monnaie(pieces, montant):
    dp = [float('inf')] * (montant + 1)
    dp[0] = 0
    for m in range(1, montant + 1):
        for p in pieces:
            if p <= m and dp[m - p] + 1 < dp[m]:
                dp[m] = dp[m - p] + 1
    return dp[montant] if dp[montant] != float('inf') else -1

print(rendu_monnaie([1, 5, 10, 25], 36))  # 3 (25+10+1)

Exercice 3 — Plus longue sous-séquence croissante

Trouvez la longueur de la plus longue sous-séquence strictement croissante dans un tableau d'entiers.

Voir la solution 
def lis(nums):
    n = len(nums)
    dp = [1] * n
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)

print(lis([10, 9, 2, 5, 3, 7, 101, 18]))  # 4 → [2, 5, 7, 101]
