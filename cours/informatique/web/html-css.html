Introduction

Les deux piliers du web

HTML (HyperText Markup Language) et CSS (Cascading Style Sheets) forment le socle de toute page web. HTML définit la structure et le contenu (titres, paragraphes, images, liens), tandis que CSS contrôle la présentation visuelle (couleurs, typographie, mise en page). Ensemble, ils permettent de créer des sites accessibles, esthétiques et adaptatifs.

Séparation des préoccupations : HTML s'occupe du « quoi » (le contenu et sa sémantique), CSS du « comment » (l'apparence). Cette séparation facilite la maintenance, l'accessibilité et le travail en équipe.

Le web moderne repose sur HTML5 et CSS3, qui apportent des balises sémantiques (<header>, <article>, <nav>), des modules de mise en page puissants (Flexbox, Grid) et des animations natives, réduisant considérablement le besoin de JavaScript pour la présentation.

Le cours

Structure et mise en forme

Structure d'un document HTML

Un document HTML est un arbre d'éléments imbriqués. Chaque élément est délimité par une balise ouvrante et une balise fermante. Le squelette minimal comprend <!DOCTYPE html>, <html>, <head> (métadonnées) et <body> (contenu visible).

HTML
<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Ma page</title>
</head>
<body>
  <h1>Bonjour le monde</h1>
  <p>Mon premier paragraphe.</p>
</body>
</html>

Balises sémantiques HTML5

HTML5 introduit des balises qui décrivent le rôle du contenu, améliorant l'accessibilité et le référencement. Les principales sont :

<header> — en-tête de page ou de section
<nav> — navigation principale
<main> — contenu principal (unique par page)
<article> — contenu autonome (article, billet)
<section> — regroupement thématique
<aside> — contenu secondaire (barre latérale)
<footer> — pied de page

Utiliser ces balises au lieu de <div> permet aux lecteurs d'écran et aux moteurs de recherche de comprendre la structure de la page.

Sélecteurs et spécificité CSS

CSS cible les éléments HTML via des sélecteurs. La spécificité détermine quelle règle s'applique lorsque plusieurs règles ciblent le même élément. Elle se calcule selon le nombre d'identifiants, de classes et d'éléments dans le sélecteur.

CSS
/* Sélecteur d'élément — spécificité (0,0,1) */
p { color: #333; }

/* Sélecteur de classe — spécificité (0,1,0) */
.intro { font-size: 1.2rem; }

/* Sélecteur d'id — spécificité (1,0,0) */
#titre { font-weight: bold; }

/* Combinaison — spécificité (0,1,1) */
article .intro { line-height: 1.8; }

/* Pseudo-classes et pseudo-éléments */
a:hover { color: #2980b9; }
p::first-line { font-variant: small-caps; }

Le modèle de boîte (Box Model)

Chaque élément HTML est une « boîte » composée de quatre couches : le contenu, le padding (marge intérieure), la bordure et la margin (marge extérieure). Par défaut, width ne concerne que le contenu ; avec box-sizing: border-box, la largeur inclut padding et bordure, ce qui simplifie les calculs.

Astuce universelle : la règle *, *::before, *::after { box-sizing: border-box; } est utilisée dans la quasi-totalité des projets modernes pour éviter les surprises de dimensionnement.

Flexbox : mise en page en une dimension

Flexbox permet d'aligner et de distribuer l'espace entre les éléments d'un conteneur, sur un seul axe (horizontal ou vertical). On active Flexbox avec display: flex sur le conteneur parent.

CSS
.navbar {
  display: flex;
  justify-content: space-between; /* répartition horizontale */
  align-items: center;           /* centrage vertical */
  gap: 1rem;                     /* espace entre éléments */
}

.card-container {
  display: flex;
  flex-wrap: wrap;               /* retour à la ligne */
  gap: 1.5rem;
}

.card {
  flex: 1 1 300px;              /* grandir, rétrécir, base 300px */
}

CSS Grid : mise en page en deux dimensions

CSS Grid permet de créer des grilles complexes sur deux axes simultanément. C'est l'outil idéal pour les mises en page de type magazine, tableaux de bord ou galeries d'images.

CSS
.page-layout {
  display: grid;
  grid-template-columns: 250px 1fr;        /* sidebar + contenu */
  grid-template-rows: auto 1fr auto;       /* header, main, footer */
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
  min-height: 100vh;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

/* Grille responsive avec auto-fit */
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1rem;
}

Responsive Design et Media Queries

Le responsive design adapte l'affichage à la taille de l'écran. Les media queries permettent d'appliquer des règles CSS conditionnelles selon la largeur du viewport, la résolution ou l'orientation.

CSS
/* Mobile-first : styles de base pour mobile */
.container {
  padding: 1rem;
  max-width: 1200px;
  margin: 0 auto;
}

/* Tablette */
@media (min-width: 768px) {
  .container { padding: 2rem; }
  .grid { grid-template-columns: 1fr 1fr; }
}

/* Desktop */
@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

L'approche mobile-first consiste à concevoir d'abord pour les petits écrans, puis à ajouter des enrichissements pour les grands. Les unités relatives (rem, %, vw) et les fonctions CSS (clamp(), min(), max()) contribuent aussi à la fluidité.

Quiz

Testez votre compréhension

📊 HTML & CSS — Quiz Score : 0 / 6

Question 1 / 6

Quelle balise HTML5 représente le contenu principal d'une page ?

La balise <main> identifie le contenu principal et unique de la page. Il ne doit y en avoir qu'une seule par document et elle ne doit pas être imbriquée dans <article> ou <aside>.

Question 2 / 6

Quelle propriété CSS active le modèle Flexbox ?

On active Flexbox en appliquant display: flex (ou display: inline-flex) sur le conteneur parent. Les enfants directs deviennent alors des « flex items ».

Question 3 / 6

Quel sélecteur a la plus haute spécificité ?

#header .nav a a une spécificité de (1,1,1) grâce à l'id (#header), la classe (.nav) et l'élément (a). Les autres ont au maximum (0,2,1).

Question 4 / 6

Que fait box-sizing: border-box ?

Avec border-box, la propriété width inclut le contenu, le padding et la bordure. Sans cette règle (valeur content-box), seul le contenu est compté, ce qui complique les calculs de mise en page.

Question 5 / 6

Quelle propriété CSS Grid permet de nommer des zones de la grille ?

grid-template-areas permet de dessiner visuellement la grille en nommant les zones. Chaque enfant utilise ensuite grid-area pour s'y placer.

Question 6 / 6

Quelle approche consiste à concevoir d'abord pour les petits écrans ?

L'approche mobile-first écrit les styles de base pour mobile, puis utilise @media (min-width: ...) pour enrichir progressivement l'affichage sur les grands écrans.

Exercices

Mettez en pratique

Exercice 1 — Page personnelle sémantique

Créez une page HTML utilisant uniquement des balises sémantiques (<header>, <nav>, <main>, <article>, <footer>). La page doit contenir un titre, une barre de navigation avec 3 liens, un article avec un titre et deux paragraphes, et un pied de page.

Voir la solution 
<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <title>Mon portfolio</title>
</head>
<body>
  <header>
    <h1>Jean Dupont</h1>
    <nav>
      <a href="#accueil">Accueil</a>
      <a href="#projets">Projets</a>
      <a href="#contact">Contact</a>
    </nav>
  </header>
  <main>
    <article>
      <h2>Mon premier projet</h2>
      <p>Description du projet.</p>
      <p>Technologies utilisées.</p>
    </article>
  </main>
  <footer>
    <p>&copy; 2025 Jean Dupont</p>
  </footer>
</body>
</html>

Exercice 2 — Navbar Flexbox

Créez une barre de navigation horizontale avec Flexbox : le logo à gauche, les liens au centre et un bouton de connexion à droite. Le tout doit être centré verticalement.

Voir la solution 
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 1rem 2rem;
  background: #1a1a2e;
}
.navbar .logo { font-weight: bold; font-size: 1.5rem; }
.navbar .links { display: flex; gap: 1.5rem; }
.navbar .links a { text-decoration: none; color: #eee; }
.navbar .btn { padding: 0.5rem 1rem; background: #2980b9; border-radius: 4px; }

Exercice 3 — Galerie Grid responsive

Créez une galerie d'images en CSS Grid qui affiche automatiquement autant de colonnes que possible, avec une largeur minimale de 200px par carte. Utilisez auto-fit et minmax().

Voir la solution 
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 1rem;
  padding: 1rem;
}
.gallery img {
  width: 100%;
  height: 200px;
  object-fit: cover;
  border-radius: 8px;
}
/* auto-fit crée autant de colonnes que possible
   minmax(200px, 1fr) garantit 200px minimum et étire si possible */

Exercice 4 — Carte avec ombre et transition

Créez une carte (.card) avec un padding, un arrière-plan, un border-radius et une ombre (box-shadow). Au survol, l'ombre doit s'agrandir avec une transition douce de 0.3s.

Voir la solution 
.card {
  padding: 1.5rem;
  background: #16213e;
  border-radius: 12px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.2);
  transition: box-shadow 0.3s ease, transform 0.3s ease;
}
.card:hover {
  box-shadow: 0 8px 24px rgba(0,0,0,0.4);
  transform: translateY(-4px);
}

Exercice 5 — Layout complet responsive

Créez une mise en page avec CSS Grid : header en haut (pleine largeur), sidebar à gauche (250px), contenu principal au centre, footer en bas. Sur mobile (max-width: 768px), tout doit passer en colonne unique.

Voir la solution 
.page {
  display: grid;
  grid-template-columns: 250px 1fr;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
  min-height: 100vh;
}
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

@media (max-width: 768px) {
  .page {
    grid-template-columns: 1fr;
    grid-template-areas:
      "header"
      "main"
      "sidebar"
      "footer";
  }
}
/* Sur mobile, le sidebar passe sous le contenu principal */
